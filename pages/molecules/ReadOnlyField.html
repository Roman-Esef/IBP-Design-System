ReadOnlyField

Поле для отображения данных без возможности редактирования. Показывает значение — текст, чипы или ссылку — с опциональными лейблом и хэлпером (см. Label / Helper), иконками слева/справа и префиксом/постфиксом. Не имеет собственного контейнера: без рамки и фона, это пара «подпись → значение» для карточек сущностей и детальных панелей.

Конструктор

Соберите поле: тип значения, цвет, слоты и обрезание текста включаются независимо друг от друга и сочетаются в любой комбинации.

Использование

ReadOnlyField используется исключительно для отображения данных — в карточках сущностей и детальных панелях, когда значение приходит из системы и не редактируется пользователем напрямую в этом месте экрана.

  • Раскладку в сетку (например, 2 колонки) задаёт контейнер-карточка — сам компонент отвечает только за пару «подпись → значение», не за грид.
  • Действия уровня карточки — редактировать, перейти к деталям — это отдельные IconButton/Button в шапке карточки, а не часть самого поля.
  • Значение может быть интерактивным точечно: иконка копирования у значения или предупреждающий индикатор с тултипом — остальной текст не кликабелен.
  • Не путайте с задизейбленным полем ввода — см. раздел «Правила».
Информация об активе
Плановые платежи

Дата платежа — с предупреждающей иконкой и тултипом; комментарий обрезан до 2 строк с тултипом полного текста при наведении.

Анатомия

Поле — вертикальный стек из трёх зон: подпись, строка значения и пояснение. Строка значения может нести до пяти слотов одновременно; всё, кроме самого значения, — опционально.

  • Labelподпись поля — .ds-label, Body XS, Text-Secondary
  • Иконка слевалюбая иконка из библиотеки — источник/категория значения, декоративная, не интерактивна
  • Префикскороткий текст перед значением — символ, аббревиатура
  • Значениетекст / чипы / ссылка — единственный обязательный слот
  • Постфикскороткий текст после значения — единица измерения, валюта
  • Иконка справалюбая иконка из библиотеки; действие настраивается отдельно — нет, копирование по клику или тултип при наведении
  • Helperпояснение под значением — .ds-helper, Body XS, Text-Inactive

Размеры

У ReadOnlyField нет собственной шкалы размеров (M/S/…) — типографика каждого слота фиксирована токенами: Label и Helper — Body XS, значение — Body M. Единственная зависимая переменная — размер вложенного Chip для значений-чипов — всегда XS. Значения ниже измерены на живом экземпляре.

Типы значения

Три типа значения. Текст — произвольная строка, может обрезаться или переноситься. Чипы — список меток размера XS (см. компонент Chip, тип ReadOnly). Ссылка — значение-переход, набирается компонентом Link.

Опции

Дополнительные слоты включаются независимо друг от друга и сочетаются в любой комбинации: иконка слева, префикс/постфикс, иконка справа (любой глиф из библиотеки иконок; действие — копирование, тултип или без действия), скрытие лейбла, выравнивание значения.

Правила для префикса и постфикса

Сверились с практиками GOV.UK, USWDS, Ant Design и Vaadin — во всех четырех аффиксы ограничены общепонятным символом/аббревиатурой и никогда не становятся интерактивными. Обобщили это в собственные правила для ReadOnlyField:

  • Только общепонятный символ или аббревиатура. Валюта (₽, USD), единица измерения (%, кг, м²), приблизительность (≈), порядковый знак (№). Если нужна расшифровка словами — это Helper, а не аффикс.
  • Не дублируйте лейбл. Если значение и так однозначно понятно из подписи, лишний аффикс — визуальный шум.
  • Аффикс не интерактивен. Он не кликается и не открывает действие — для действия предназначена иконка справа (Icon-right).
  • Ограничение длины — ориентировочно 10–12 символов. При превышении текст аффикса обрезается многоточием и на наведении показывает тултип с полным содержанием — так же, как переполнение значения. Это подстраховка на случай ошибки конфигурации, а не штатный сценарий.
Так

Короткий, общепонятный символ — курс валюты не нужно объяснять словами.

Не так

Целая фраза вместо символа — обрежется многоточием и потребует наведения, чтобы её прочитать. Перенесите пояснение в Helper.

Переполнение и перенос

Тип «Текст» может течь свободно, обрезаться до одной строки или до N строк — во всех случаях обрезанный текст уходит в многоточие, а при наведении курсора показывается тултип с полным содержанием. Тип «Чипы» переносится по-другому: чипы как целый список уходят на следующую строку, а при ограниченной высоте лишние могут сворачиваться в счётчик.

Цвет значения

Текст значения может быть окрашен для передачи динамики — например, положительного (зелёный) или отрицательного (красный) изменения показателя. Остальные слоты (лейбл, хэлпер, аффиксы) цвет не наследуют.

Контент

Текстовое содержимое компонента — четыре независимых слота: подпись, значение, пояснение и опциональные аффиксы.

  • Label — короткое существительное или номинативная фраза, называющая атрибут («Внешняя АС», «ИНН контрагента»); sentence case, без двоеточия и точки в конце;
  • Значение — тип «Текст»: произвольная строка данных как пришла из системы, без искусственного форматирования; тип «Чипы» — метки компонента Chip (ReadOnly, XS); тип «Ссылка» — текст перехода, набирается по правилам Link;
  • Helper — короткое пояснение к значению, а не инструкция к заполнению (как в редактируемых полях) — значение уже показано; опционален;
  • Префикс / постфикс — только общепонятный символ или аббревиатура, не слово-предложение (см. правила в разделе «Опции»);
  • при обрезке — многоточие и тултип с полным текстом (см. «Переполнение и перенос»); пустое значение — плейсхолдер «—», а не пустая строка (см. «Состояния»).

Правила

ReadOnlyField сообщает, что значение принципиально недоступно для правки в этом месте интерфейса — в отличие от задизейбленного инпута, который лишь временно не активен.

Так

Системный ID — используйте ReadOnlyField: рамка поля ввода не нужна, значение никогда не станет редактируемым.

Не так

Задизейбленный InputText для того же значения — визуально обещает, что поле может стать редактируемым, хотя это не так.

Так

Длинное значение — ограничьте 2–3 строками с многоточием и тултипом на переполнении.

Не так

Нерастянутое значение произвольной длины ломает сетку карточки и мешает соседним полям.

Так

Индикатор с тултипом — точечная интерактивность внутри значения, без превращения всего поля в кнопку.

Не так

Не превращайте значение в акцентную кнопку действия — для этого есть Button/Link.

Доступность

ReadOnlyField — пара «подпись → значение» из статичного текста; сам по себе он не интерактивен и не участвует в таб-порядке, кроме собственных точечных интерактивных элементов (иконка-копирование, значение-ссылка).

  • Оборачивайте пару в <dl>/<dt>/<dd> либо связывайте значение с подписью через aria-labelledby — иначе скрин-ридер читает подпись и значение как два несвязанных фрагмента;
  • иконка-копирование (iconRightAction="copy") — фокусируемый элемент с role="button" и aria-label="Скопировать значение"; после копирования — временное визуальное подтверждение (см. «Опции»);
  • предупреждающая иконка с тултипом — фокусируема (tabindex="0"), текст пояснения доступен и с клавиатуры (:focus показывает тот же тултип, что и :hover);
  • значение типа «Ссылка» — обычный фокусируемый <a>, наследует доступность компонента Link;
  • не полагайтесь только на цвет для Positive/Negative значения — дублируйте смысл знаком (+/−) в самом значении, как в примерах раздела «Цвет значения»;
  • Loading-скелетон помечайте aria-busy="true" на контейнере поля, чтобы скрин-ридер не пытался озвучить псевдо-текст плейсхолдера.

Цвета

Компонент использует только семантические токены текста; для положительной/отрицательной динамики — ситуативные токены Success/Error.

Поведение · псевдокод (framework-agnostic) 
// 1. Обрезка/тултип: когда scrollWidth/scrollHeight элемента значения > clientWidth/clientHeight
//    (текст обрезан ellipsis или line-clamp) — на hover/focus показать tooltip с полным текстом.
//    Проверка выполняется заново при каждом наведении — обрезка зависит от текущей ширины layout.

// 2. Копирование: клик/Enter/Space по rof__icon--interactive → navigator.clipboard.writeText(value) →
//    иконка временно меняется на «check», всплывает тултип «Скопировано» (~1.3с), затем возврат

// 3. Переполнение чип-листа (chipsMaxRows): после layout измерить offsetTop чипов, сгруппировать
//    по строкам; если строк больше лимита — скрыть лишние чипы и добавить чип-счётчик «+N»
//    с тултипом, перечисляющим скрытые значения

// 4. Empty: если данных нет — показать value = «—» приглушённым цветом вместо пустой строки

// 5. Loading: aria-busy="true" на корне; вместо label/value — шиммер-плейсхолдеры фиксированной высоты

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функция makeROF в read-only-field.page.js).

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не обязательный контракт; адаптируйте под конвенции вашей кодовой базы.

ReadOnlyField.types.ts 
type ROFValueType = 'text' | 'chips' | 'link';
type ROFTone      = 'default' | 'positive' | 'negative' | 'empty';

interface ReadOnlyFieldProps {
  label: React.ReactNode;
  showLabel?: boolean;                // по умолчанию true
  helper?: React.ReactNode;
  helperStatus?: 'default' | 'error';
  type?: ROFValueType;                     // по умолчанию 'text'
  value?: React.ReactNode;                 // text / link
  chips?: string[];                     // type='chips'
  chipsMaxRows?: number;                // свёртка в «+N» (Предложение)
  linkHref?: string;                    // type='link'
  iconLeft?: React.ReactNode;
  prefix?: string;
  postfix?: string;
  iconRight?: React.ReactNode;
  iconRightAction?: 'none' | 'copy' | 'tooltip';
  iconRightTip?: string;                 // обязателен при iconRightAction='tooltip'
  tone?: ROFTone;                           // 'empty' — плейсхолдер «—» (Предложение)
  clampMode?: 'none' | '1' | '2' | '3';    // число строк перед line-clamp
  align?: 'left' | 'right';
  state?: 'default' | 'loading';      // (Предложение)
}
