InputAutocomplete

Поле с выпадающим списком: пользователь вводит запрос, а под полем раскрывается DropdownList с подходящими опциями. Список содержит как текстовые опции (одиночный выбор), так и опции с чекбоксами (множественный выбор). Выбранные значения показываются в поле сводкой, чипами внутри поля или чипами во внешнем стеке под полем. Построен на той же базе Input_Content, что и InputText и InputDate.

Конструктор

Соберите поле автокомплита: способ показа выбора (сводка / чипы в поле / чипы во внешнем стеке), тип списка (текст / чекбоксы), размер, состояние и раскрытие списка.

Использование

InputAutocomplete — выбор значения из справочника, когда вариантов много и нужен поиск по вводу. Одиночный выбор показывается сводкой в поле; множественный — чипами. Основной размер — M; размер S используется только внутри таблиц (Table Edit).

Одиночный выбор — выбранное значение подставляется в поле текстом.
Множественный · сводка — «Value 1, +4»; компактно при большом числе выборов.
Множественный · чипы — каждое значение видно и удаляется отдельно.

Когда использовать

  • выбор из большого справочника с поиском по вводу: контрагент, валюта, продукт;
  • множественный выбор из перечня — с чипами или сводкой;
  • ввод с подсказками, где допустимо и свободное значение (action-строка «создать»).

Если вариантов мало и поиск не нужен — Select. Для свободного текста без списка — InputText. Устройство самого списка (опции, состояния, системные строки) описано на странице Select · DropdownList.

Анатомия

Поле-триггер + раскрывающийся под ним DropdownList. Триггер отличается от InputText показом выбора (сводка / чипы) и обязательным шевроном справа.

  • Label — метка поля, .ds-label
  • Поле-триггер (Input_Content) — с сводкой или чипами выбранных значений
  • Действия — крестик очистки · шеврон раскрытия списка
  • DropdownList — список опций: текст или чекбоксы, с подсветкой совпадения
  • Внешний стек чипов — вариант, где чипы выносятся под поле (Chips Ext)

Варианты

Две основные оси: как показан выбор (сводка / чипы в поле / чипы во внешнем стеке) и чем наполнен список (текст / чекбоксы).

Показ выбора

Три способа отобразить выбранные значения в поле.

Чипы · в поле

Выбранные значения — чипами прямо в поле. Компактный размер чипа (S для поля M, XS для поля S). При переполнении последние сворачиваются в чип-счётчик «+N».

Чипы · внешний стек

Когда выборов много и они не должны сжимать поле, чипы выносятся в стек под полем (Chips Ext). Поле остаётся чистым для ввода следующего запроса.

Наполнение списка

DropdownList под полем: одиночный выбор — текстовые опции с подсветкой совпадения; множественный — опции с ведущим чекбоксом. Подробнее — на странице Select · DropdownList.

Table Edit

Вариант поля в таблице: размер S, никогда не содержит метку и хелпер. Показ выбора — сводкой (чипы в узкой ячейке не помещаются).

Размеры

Два размера. M — основной; S — только для Table Edit. Чип внутри поля берётся на размер меньше поля: S для поля M, XS для поля S.

Контент

Правила контента:

  • плейсхолдер пустого поля — приглашение к поиску («Начните вводить название»);
  • сводка множественного выбора — «первое значение, +N»; N — число остальных;
  • подсветка совпадения (.ddl__match) показывает, почему опция найдена;
  • текст чипа обрезается многоточием при переполнении, полный текст — в тултипе;
  • системные строки списка (загрузка, «ничего не найдено», ошибка) — по правилам DropdownList.

Поведение

Раскрытие списка, фильтрация по вводу и работа с чипами.

Раскрытие и фильтрация

Список раскрывается при фокусе на поле или клике по шеврону; ширина списка равна ширине поля. По мере ввода опции фильтруются, совпавшая подстрока подсвечивается. Пример живой — печатайте в поле.

Работа с чипами

Выбор опции с чекбоксом добавляет чип; крестик на чипе или повторный выбор опции — удаляет. Живой пример: выбирайте опции в списке и удаляйте чипы.

Состояния

Восемь состояний поля-триггера, общих со всеми типами поля. Тултип с текстом ошибки или предупреждения появляется в ErrorFocus и WarningFocus. Состояния опций списка — на странице DropdownList.

Доступность

Поле-триггер — role="combobox" с aria-expanded и aria-controls на список; список — role="listbox" (множественный — aria-multiselectable="true"), опции — role="option" с aria-selected / aria-checked. Активная опция подсвечивается через aria-activedescendant. Каждый чип и его крестик озвучиваются; шеврон — кнопка с aria-label.

Клавиатура

↓ / ↑открыть список и перемещать активную опцию
Enterвыбрать активную опцию (в multiselect — переключить чекбокс)
Backspaceв пустом поле — удалить последний чип
Escзакрыть список, фокус остаётся в поле

Типографика

Ввод и сводка набираются гарнитурой SB Sans Text. Чипы — по своим токенам (Body S / XS). Опции списка — Body M, helper и группа — Body XS.

Цвета

Поведение · псевдокод (framework-agnostic) 
// 1. Раскрытие: фокус поля или клик по шеврону → открыть список;
//    ширина списка = ширине поля; авто-разворот вверх при нехватке места.

// 2. Фильтрация: по вводу — дебаунс, фильтр опций, подсветка совпадения
//    (.ddl__match). Нет результатов → системная строка ddl__state--empty.

// 3. Одиночный выбор: клик по опции → значение в поле, список закрыть.

// 4. Множественный: клик по опции → переключить чекбокс и чип;
//    список НЕ закрывать. Показ выбора: сводка / чипы в поле / внешний стек.

// 5. Чипы: крестик или повторный выбор опции удаляет чип;
//    Backspace в пустом поле — последний чип. Переполнение поля → «+N».

// 6. Клавиатура: ↓/↑ — активная опция (aria-activedescendant),
//    Enter — выбор, Esc — закрыть.

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не обязательный контракт; адаптируйте под конвенции вашей кодовой базы.

InputAutocomplete.types.ts 
interface Option { id: string; label: string; helper?: string; disabled?: boolean; }

type Selection = 'summary' | 'chips' | 'chips-external';

interface InputAutocompleteProps {
  options: Option[];
  value: string[];               // id выбранных (одиночный — длина 0..1)
  onChange(ids: string[]): void;
  onSearch?(query: string): void; // async-фильтрация
  multiple?: boolean;            // список с чекбоксами
  selectionDisplay?: Selection;    // по умолчанию multiple? 'chips' : 'summary'
  size?: 'm' | 's';              // s — только Table Edit
  label?: string;
  helper?: string;
  placeholder?: string;
  status?: 'default' | 'error' | 'warning';
  statusMessage?: string;
  loading?: boolean;             // системная строка загрузки в списке
  allowCreate?: boolean;         // action-строка «создать значение»
  disabled?: boolean;
}

Что предлагаем добавить

Кейсы, которых нет в текущей документации:

  • action-строка «создать значение» при отсутствии совпадений (свободный ввод в список);
  • асинхронная загрузка опций с системными строками loading / error (есть в DropdownList — связать явно);
  • группировка опций с липкими заголовками в автокомплите;
  • лимит выбора (max N чипов) с блокировкой остальных опций.
