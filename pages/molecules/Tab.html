Tab

Таб — инструмент переключения между разными вьюхами одного экрана. По умолчанию горизонтальный таб подстраивается под размер контента, вертикальный — под размер контейнера. Размеры M и S, опциональные иконка и бейдж-счётчик.

Конструктор

Соберите таб: ориентация, размер, иконка, бейдж и состояние — превью и код обновляются мгновенно.

Использование

Табы переключают вьюхи в пределах одной страницы или модалки, не меняя контекст целиком. Горизонтальные — для разделов страницы и табов в шапке таблицы; вертикальные — для длинных списков разделов в модалках (фильтры, формы).

  • Горизонтальные табы. Разделы страницы сделки, переключение вкладок над таблицей, табы внутри модалки.
  • Вертикальные табы. Навигация по разделам формы или фильтра в модалке; бейдж показывает число применённых параметров.
  • Бейдж. Опциональный счётчик: целое число с разбиением на разряды. На горизонтальных табах прижат к тексту, на вертикальных — к правому краю.
  • Один таб в группе всегда выбран. Группа из одного таба не имеет смысла.

Анатомия

Таб — это интерактивная зона с текстом и опциональными слотами, плюс индикатор выбора. Группа табов разделяет общий рейл: снизу — у горизонтальных, слева — у вертикальных.

  • Контейнер / рейлобщая линия группы (1px); индикатор выбранного таба лежит поверх неё
  • Иконкаведущий слот слева — опционально
  • ТекстSB Sans Text, Regular во всех состояниях
  • Бейджсчётчик-целое с разбиением на разряды — опционально

Ориентация

Horizontal — табы в ряд с общим нижним рейлом; ширина каждого таба тянется по контенту. Vertical — табы в столбец с общим левым рейлом; ширина таба равна ширине контейнера, бейдж прижат к правому краю. Попробуйте переключить — табы рабочие, можно ходить стрелками с клавиатуры.

Размеры

Два размера. M — базовый для страниц и модалок. S — для плотных мест и узких модалок. Высота таба и кегль текста зависят от размера.

Состояния

Невыбранные и выбранные табы проходят через Default и Hover = Focus, плюс Disabled — отдельного состояния нажатия (Active) нет. Hover и Focus визуально совпадают — единая заливка --primary-bg (Active · PrimaryBG), одинаковая для выбранного и невыбранного таба. Клик переводит таб в Selected, а заливка остаётся как при наведении, пока курсор не покинет таб — отдельного pressed-стейта не требуется. Бейдж при этом цвет не меняет: он одинаковый у выбранного и невыбранного таба во всех состояниях.

Бейдж

Бейдж отображается опционально. В нём может быть только целое число, и оно показывается с разбиением на разряды (1 234). Минимальная ширина бейджа — 20 / 16px для размеров M / S. На горизонтальных табах бейдж прижат к тексту, на вертикальных — к правому краю таба.

ПереполнениеПредложение

Два паттерна для горизонтальных табов, когда их больше, чем помещается по ширине контейнера. Скролл — ряд прокручивается по горизонтали, скроллбар скрыт; по краям появляются стрелки, когда в эту сторону ещё есть куда скроллить, а выбранный таб подскролливается в зону видимости. Fit + меню «Ещё» — показываются только табы, помещающиеся целиком, остальные уходят в выпадающий список за иконкой из трёх точек. У вертикального таба слишком длинный текст обрезается многоточием с тултипом.

Равная ширина (Fitted)Предложение

Режим, в котором табы делят ширину контейнера поровну — для сегмент-переключателей с коротким набором опций (период, режим отображения). Это альтернатива контентной ширине горизонтальных табов.

Типографика

Текст таба набирается гарнитурой SB Sans Text, начертание Regular во всех состояниях, включая выбранный. Кегль зависит от размера; letter-spacing 0.

Контент

Текстовое содержимое таба — короткая метка и опциональный числовой бейдж-счётчик.

  • Метка (.tab__label) — короткое существительное или номинативная фраза, называющая раздел («Сделки», «История»), не предложение и не глагол-действие — для действий есть Button;
  • регистр — sentence case, как в остальном интерфейсе; точку в конце не ставим;
  • Бейдж (.tab__badge) — только целое число с разбиением на разряды (1 234), без единиц измерения и текста;
  • таб только с иконкой (без видимого текста) обязан иметь aria-label с названием раздела — иконка сама по себе не самодостаточна (см. «Доступность»);
  • при обрезке длинного текста вертикального таба полное значение дублируется тултипом (см. «Переполнение»).

Правила

Таб переключает вьюхи в пределах одного экрана. Каждая пара «так / не так» ниже раскрывает одну конкретную ситуацию — не смешивайте разные правила в одном сравнении.

Так

Короткие метки разделов одного экрана.

Не так

Длинные фразы-действия растягивают табы — это задача ссылок/кнопок.

Так

Группа из двух и более разделов — табам есть что переключать.

Не так

Один таб в группе бессмыслен — уберите табы или добавьте разделы.

Дополнительно — без пары «не так»

Так

Бейдж-счётчик показывает число элементов раздела — целое число, без парного антипаттерна.

Доступность

Группа табов — role="tablist", каждый таб — role="tab" с aria-selected. В таб-порядке участвует только выбранный таб (roving tabindex) — у остальных tabindex="-1", они достижимы стрелками.

  • Стрелки ← → у горизонтальной группы и ↑ ↓ у вертикальной переключают фокус на соседний таб и сразу выбирают его; Home / End — на первый/последний;
  • :focus-visible показывает обводку 2px --primary с отступом −2px внутрь таба. Нажмите Tab, затем стрелки, чтобы увидеть:
  • Disabled-таб исключён из таб-порядка (tabIndex=-1, aria-disabled="true", pointer-events:none);
  • таб только с иконкой обязателен aria-label — без него скрин-ридер озвучит только «кнопка» без контекста;
  • панель, которую открывает таб, свяжите через aria-controls с id панели, а панель — aria-labelledby с id таба.

Цвета

Невыбранный таб по умолчанию окрашен в Text_Secondary (текст и иконка), при наведении переходит на Text_Primary; бейдж — на StSystem_MidLight. Выбранный таб — текст на Text_Primary, индикатор и иконка на Primary (Emerald). Disabled — на рампе StDisabled.

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

<div class="tabs tabs--horiz" role="tablist">

    <span class="tab__label">Обзор</span>
  </button>

  <button type="button" class="tab tab--m tab--horiz tab--selected" role="tab" aria-selected="true" tabindex="0">

  </button>
</div>
Поведение · псевдокод (framework-agnostic) 
// 1. Клик/Enter/Space по табу → снять tab--selected/aria-selected со всех, поставить на текущий,
//    tabIndex текущего = 0, у остальных = -1 (roving tabindex)

// 2. Клавиатура на [role="tablist"]:
//    ArrowRight/ArrowDown (horiz/vert) → следующий таб (зациклено), сразу активировать
//    ArrowLeft/ArrowUp     → предыдущий таб (зациклено), сразу активировать
//    Home / End            → первый / последний доступный (non-disabled) таб

// 3. Hover = Focus — единый визуальный стейт-слой (см. «Состояния»); клик не добавляет
//    отдельного pressed-состояния — заливка держится как hover, пока курсор не покинет таб

// 4. Overflow (Предложение): измерить ширину контейнера и суммарную ширину табов —
//    лишние уходят в «⋯»-меню, либо контейнер скроллится с появляющимися стрелками
//    (рабочий референс — makeOverflowTabs / makeScrollArrows в tab.page.js)

// 5. Disabled: pointer-events: none, tabIndex = -1, aria-disabled = true

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функции wireTablist, makeOverflowTabs в tab.page.js).

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не обязательный контракт; адаптируйте под конвенции вашей кодовой базы.

Tab.types.ts 
type TabOrientation = 'horizontal' | 'vertical';
type TabSize = 'm' | 's';

interface TabItem {
  value: string;
  label?: React.ReactNode;        // опустить для icon-only — тогда обязателен ariaLabel
  icon?: React.ReactNode;
  badge?: number;                 // целое число, рендерится с разбиением на разряды
  disabled?: boolean;
  ariaLabel?: string;
}

interface TabsProps {
  items: TabItem[];
  value: string;                                // controlled-компонент
  onChange: (value: string) => void;
  orientation?: TabOrientation;                  // по умолчанию 'horizontal'
  size?: TabSize;                                // по умолчанию 'm'
  fitted?: boolean;                            // равная ширина сегментов (Предложение)
  overflow?: 'scroll' | 'menu';              // паттерн переполнения (Предложение)
  'aria-label': string;                       // обязателен на tablist
}
