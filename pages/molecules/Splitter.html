Splitter

Функциональный разделитель контейнера: пользователь тащит сплиттер и регулирует размеры соседних панелей. При наведении курсор меняется на col-resize / ew-resize, а сам сплиттер переходит в состояние Hover. Перемещается по одной оси; в момент перетаскивания вид и курсор сохраняются.

Конструктор

Соберите сплиттер: ориентация и состояние. Выберите режим «Живой», чтобы перетащить разделитель и изменить размеры панелей.

Использование

Используйте сплиттер, когда нужно:

  • дать пользователю самому распределить пространство между двумя областями — список и карточку, дерево и контент, редактор и превью;
  • сохранить обе области видимыми, но позволить расширить ту, что важнее в текущей задаче.

Сплиттер — интерактивный элемент, в отличие от Divider, который только визуально отделяет блоки и не двигается. Если менять размеры не нужно — используйте обычный разделитель.

Пример использования

Раздел «Пользователи»: слева — таблица-мастер, справа — карточка деталей. Сплиттер позволяет расширить нужную сторону. Потяните разделитель внутри макета.

Рекомендации

Сплиттер должен читаться как перетаскиваемый: оставляйте ручку-грип и состояние Hover. Невидимый разделитель без аффорданса пользователь не найдёт.

Не так

Нет ручки и подсветки — выглядит как обычная граница, перетаскивание неочевидно.

Так

Грип и подсветка зоны явно сообщают, что границу можно тащить.

Анатомия

Сплиттер — это узкая зона захвата (11px) между двумя панелями. По центру проходит тонкая линия 1px, в середине — ручка-грип из точек, подсказывающая, что элемент можно тащить.

  • 1Зона захвата — 11px, курсор col-resize
  • 2Линия — 1px, темнеет в Hover/Move
  • 3Ручка-грип — точки-аффорданс по центру

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Ориентация

Вертикальный сплиттер регулирует ширину колонок (курсор col-resize) — основной сценарий. Горизонтальный регулирует высоту строк (курсор row-resize) — например, делит контент и панель деталей по высоте. Грип разворачивается вдоль оси перетаскивания.

Вертикальный — .spl

Горизонтальный — .spl--h

Размеры

У сплиттера нет масштабной линейки размеров (M/S/…) — геометрия задана фиксированными токенами — одними и теми же для любой ориентации. Значения ниже измерены на живом экземпляре компонента (getComputedStyle).

Контент

У сплиттера нет видимого текста — единственный «контент» это описание в aria-label, которое скрин-ридер озвучивает вместо визуальной подписи:

  • aria-label называет обе стороны или сам факт изменения размера — например «Изменить ширину панелей» или «Пользователи / Карточка пользователя»;
  • не дублируйте в подписи слова «сплиттер» или «разделитель» — роль уже озвучивается через role="separator";
  • если по обе стороны — узнаваемые области (таблица и карточка деталей, дерево и контент), называйте их напрямую: «Пользователи / Карточка пользователя»;
  • текущее значение озвучивается через aria-valuenow при перетаскивании — отдельный текстовый лейбл значения не нужен.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Поведение и ограничения

Сплиттер перемещается только по своей оси. Размеры панелей ограничены минимумом и максимумом (здесь 25% / 75%), чтобы ни одна панель не схлопнулась полностью. Перетащите разделитель — соотношение обновляется в реальном времени.

Соотношение панелей Лево / Право: 50% / 50%

Состояния

Исходная пара Default / Hover дополнена состояниями Move (перетаскивание), Focus и Disabled. В Move сплиттер сохраняет вид Hover и курсор col-resize на всём документе, пока кнопка мыши зажата.

Доступность

Сплиттеру присваивается role="separator", aria-orientation и значения aria-valuemin / aria-valuemax / aria-valuenow (в процентах). Он фокусируется по Tab; стрелки ←/→ (или ↑/↓ для горизонтального) сдвигают границу с шагом 2%, с Shift — 10%, Home / End — в минимум / максимум. Сфокусируйте сплиттер в примерах выше и подвигайте стрелками.

Типографика

Раздел требует ревизии: токены типографики компонента — в redline-таблице раздела «Для разработчиков».

Цвета

Компонент использует только семантические токены границ, текста и подсветки. Собственных цветов нет.

Токены

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

// 3. pointerup / pointercancel → dragging = false, снять курсор с body
