InputDateRange

Поле ввода диапазона дат: два InputDate с префиксами «От» / «До», размещённых горизонтально и соединённых линией Range_Line. При фокусе на поле или по кнопке-календарю поднимается DatePicker в режиме диапазона: первая выбранная дата пишется в «От», вторая — в «До». Общая метка сверху и общий хелпер снизу. Стандартная высота поля 40px; размера S у InputDateRange нет.

Конструктор

Соберите диапазон дат: состояние левого и правого поля независимо, наполнение, метка и хелпер. Ввод живой — маска форматирует цифры по шаблону.

Использование

InputDateRange задаёт диапазон дат «с … по …» в фильтрах и формах: период сделки, срок действия, окно отчётности. Единственный размер — M. Одно из полей может остаться пустым — тогда граница периода открыта.

В форме — общая метка и хелпер, оба поля заполнены.
Пустой период — оба поля показывают маску-плейсхолдер.
Открытая граница — заполнено только «От».

Когда использовать

  • задать период дат: срок сделки, окно отчётности, интервал платежей;
  • фильтр «с» / «по» с возможностью открытой границы (одно поле пустое).

Для одиночной даты — InputDate. Для числового диапазона — InputAmountRange.

Анатомия

Компонент состоит из общей метки, двух полей InputDate, размещённых горизонтально, с компонентом Range_Line между ними и хелпера (опционально). У каждого поля есть минимальная ширина (186px) из расчёта ширины контента (даты) и иконок крестика и календаря (вызов DatePicker). В полях всегда присутствует префикс «От» / «До».

  • Label — общая метка диапазона, .ds-label (Body XS)
  • Input × 2 — два поля InputDate с префиксом «От» / «До», мин. ширина 186px, высота 40px (M)
  • Range_Line — линия между полями; толщина и цвет аналогичны бордеру инпута (1px, --border-primary)
  • Helper — общий хелпер под диапазоном, .ds-helper (Body XS)

С хелпером и без

Наполнение

Оба поля могут быть пустыми, заполнено одно (открытая граница) или оба.

Размеры

Единственный размер — M, высота поля 40px. Размера S у InputDateRange нет. Минимальная ширина каждого поля — 186px; общая ширина задаётся контейнером.

Контент

Правила контента:

  • префикс «От» / «До» присутствует всегда — обозначает начало и конец диапазона, неизменяем;
  • маска — ММ.ДД.ГГГГ; шаблон одинаков в обоих полях экрана;
  • пустое поле показывает маску как плейсхолдер — отдельный плейсхолдер-текст не пишется;
  • метка — существительное: «Период сделки», не «Введите даты»;
  • хелпер — правило заполнения; в состоянии Error заменяется текстом ошибки.

Поведение

Плейсхолдер отображается в поле до тех пор, пока поле не перейдёт в состояние Focus. Если пользователь очистил поле — оно возвращается в пустое состояние по потере фокуса. В полях InputDateRange всегда присутствует префикс (От или До) для обозначения начала и конца диапазона.

Маска ввода

Вводятся только цифры — точки-разделители подставляются автоматически. Каждое поле поднимает DatePicker при фокусе и по кнопке-календарю; выбор даты записывает значение и закрывает календарь. Пример живой — печатайте.

Независимость полей

С каждым полем пользователь взаимодействует отдельно: ховер или фокус на левом поле не вызывают реакции на правом и наоборот. То же касается состояния ошибки — оно распространяется только на то поле, в котором находится невалидное значение. При этом оба поля могут находиться в состоянии ошибки одновременно, если у каждого содержится невалидное значение.

Состояния

Восемь состояний, общих для базового поля. Состояние показано на левом поле «От»; правое поле «До» — в Default. Текст ошибки или предупреждения по умолчанию не выносится в хелпер (хелпер остаётся правилом заполнения) — сообщение показывается только в тултипе в состояниях ErrorFocus и WarningFocus. Тултип не смещает хелпер: лежит поверх потока, z-index выше поля. Исключение из правила допустимо, если оно намеренно заложено при разработке.

Доступность

Общая метка связывается с полями через label / aria-label, хелпер — через aria-describedby, ошибка каждого поля — aria-invalid="true" только на невалидном поле. Кнопка-календарь — настоящая кнопка с aria-haspopup="dialog"; Range_Line декоративен (aria-hidden="true"). Каждое поле озвучивается отдельно.

Цвета

Токены совпадают с InputDate. Range_Line использует тот же токен рамки, что и поле: --border-primary; в состоянии Disabled — --border-light.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Каждое поле — экземпляр базового .inp (см. InputDate); здесь только обвязка диапазона.

// 1. Каждое поле — независимый InputDate со своим состоянием.
//    Hover/Focus/Error одного поля НЕ влияют на другое.

// 2. Маска: пропускать цифры; после 2-го и 4-го знака вставлять «.»;
//    Backspace удаляет цифру вместе с разделителем.

// 3. DatePicker: открывать при фокусе поля и по кнопке-календарю;
//    выбор → записать значение, закрыть, вернуть фокус в поле.

// 4. Валидация диапазона: from <= to (если обе даты заданы) →
//    иначе помечать ошибкой то поле, где значение невалидно.
//    Оба поля могут быть в ошибке одновременно.

// 5. Плейсхолдер-маска до Focus; очищенное поле по blur → пустое.
//    Префикс «От» / «До» всегда присутствует.
