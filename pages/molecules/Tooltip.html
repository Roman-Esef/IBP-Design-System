Tooltip

Всплывающая подсказка при наведении или фокусе на объект — иконку, IconButton, обрезанный текст или поле с ошибкой. Контент — строка, тултип не может быть пустым. Хинт может быть со стрелкой или без неё, в любом из 12 краевых положений.

Конструктор

Соберите тултип: тип, размещение, выравнивание стрелки и текст — превью обновляется мгновенно.

Использование

Три типичных сценария — пояснение иконки, полное значение обрезанного текста, сообщение об ошибке рядом с полем. Наведите курсор на каждый пример.

IconButton без подписи — тултип поясняет действие
Обрезанный текст — тултип показывает полное значение
Поле с ошибкой — тултип типа Error уточняет причину

Когда использовать

Используйте тултип, когда нужно:

  • пояснить назначение иконки или IconButton без подписи;
  • показать полное значение обрезанного текста;
  • дать короткое уточнение или сообщение об ошибке рядом с полем.

Тултип — вспомогательный, временный слой. Не размещайте в нём контент, без которого нельзя выполнить задачу, и интерактивные элементы (ссылки, кнопки) — для этого подходят поповер или модальное окно. Текст должен быть коротким; тултип не может быть пустым.

Анатомия

Тултип состоит из контейнера с фоном, текстового контента и опциональной стрелки, направленной на объект подсказки.

  • Контейнер — фон, радиус 8px, тень
  • Контент — строка, стиль Body S (14/16)
  • Стрелка — опционально, указывает на объект

Паддинги контентной области — 8 / 12 / 8 / 12. Расстояние от стрелки до края тултипа при краевом размещении — 6px. Стрелка выносится за контейнер на 7px.

Размеры

В общем случае размер тултипа определяется размером контента. Задаётся максимальная ширина контейнера (--tip-max = 280px). Если ширина контента с паддингами меньше максимальной — тултип сжимается до обтекания контента.

Короткий контент — обтекание
Длинный контент — перенос по max-width Новое

Однострочный текст не переносится — тултип растёт по ширине до максимума. Текст длиннее максимальной ширины переносится на несколько строк (.tip--multiline), сохраняя паддинги 8/12.

Контент

Текст тултипа — единственное содержимое компонента; пустым он не бывает. Правила:

  • одна короткая фраза — идеально в одну строку (см. «Размеры» выше); длинный текст — признак того, что нужен поповер или другой компонент, а не тултип;
  • тип Main — нейтральное пояснение без точки в конце, если это не полное предложение (например, полное значение обрезанного текста);
  • тип Error — конкретная причина и (по возможности) что сделать для исправления — не общее «Ошибка!»;
  • не дублируйте видимую рядом подпись дословно — тултип добавляет смысл, а не повторяет его;
  • без интерактивных элементов внутри (см. «Когда использовать») — ссылки и кнопки недоступны внутри тултипа.

Типы

Виды тултипа отличаются между собой цветом фона. Main — нейтральные подсказки, Error — сообщения об ошибке рядом с полем или контролом.

Размещение и стрелка

Тултип размещается с одной из 4 сторон объекта (top · bottom · left · right), а стрелка выравнивается вдоль края по одному из 3 положений (start · center · end) — всего 12 комбинаций. Стрелка всегда направлена на объект подсказки.

12 положений стрелки

Стрелку можно отключить — тогда тултип просто отступает от объекта на 8px. Без стрелки уместен для подсказок, не привязанных к конкретной точке.Новое

Авто-разворот и смещениеНовое

Если у предпочтительной стороны не хватает места, тултип разворачивается на противоположную сторону, а вдоль края смещается, чтобы не выходить за границы экрана — при этом стрелка остаётся направленной на объект. Перетащите цель к краям области, чтобы увидеть поведение.

Триггеры и таймингиНовое

Тултип появляется при наведении курсора и при фокусе с клавиатуры, скрывается при уходе курсора, потере фокуса или нажатии Esc. Чтобы случайные пересечения курсора не вызывали мелькания, появление задерживается на ~400 мс, скрытие — мгновенное. На тач-устройствах тултип вызывается долгим нажатием.

Наведите курсор или перейдите Tab’ом на иконку

Тайминги

ДоступностьНовое

Контейнеру присваивается role="tooltip" и уникальный id, а объект-триггер ссылается на него через aria-describedby. Триггер должен быть фокусируемым (кнопка, ссылка или элемент с tabindex="0"). Тултип не перехватывает фокус и закрывается по Esc.

Не так

Критичная информация и действия спрятаны в тултипе — недоступны с клавиатуры и на тач-устройствах.

Так

Короткое пояснение, продублированное в aria-describedby; объект остаётся доступным.

Рекомендации

Держите текст коротким и по делу. Тултип поясняет объект, а не дублирует видимую подпись и не содержит длинных инструкций.

Не так

Длинный абзац перегружает временную подсказку.

Так

Одна короткая фраза — суть подсказки.

Типографика

Текст тултипа набирается гарнитурой SB Sans Text токеном --type-body-s.

Цвета

Компонент использует только семантические токены. Стрелка наследует фон контейнера через переменную --tip-bg.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

Разметка · HTML (эталонная реализация ДС) 
<!-- триггер ссылается на тултип через aria-describedby -->
<button type="button" class="iconbtn" aria-label="Удалить" aria-describedby="tip-1">…</button>

<!-- сам тултип: позиционируется JS относительно триггера, скрыт до показа -->
<span class="tip tip--main tip--top tip--center tip--floating" role="tooltip" id="tip-1">
  Удалить
  <span class="tip__arrow"></span>
</span>

<!-- без стрелки / с переносом -->
<span class="tip tip--error tip--bottom tip--start tip--no-arrow tip--multiline" role="tooltip">…</span>
Поведение · псевдокод (framework-agnostic) 
// 1. Триггеры: hover (задержка ≈400мс перед показом, чтобы случайное пересечение курсора
//    не вызывало мелькание), focus (мгновенно, без задержки); скрытие — мгновенно по mouseleave/blur/Esc

// 2. Позиционирование (вызывается при показе и при resize):
//    взять getBoundingClientRect() цели и тултипа относительно позиционирующего родителя;
//    top/bottom — y = край цели ± gap(8px); left/right — аналогично по X;
//    align center/start/end сдвигает вторую ось относительно центра цели (start/end — отступ 13px для стрелки)

// 3. Авто-разворот: проверить свободное место по предпочтительной стороне относительно viewport/контейнера;
//    не хватает — пробовать противоположную, затем перпендикулярные стороны; выбрать первую подходящую;
//    затем зажать позицию внутри границ контейнера и пересчитать позицию стрелки, чтобы она всё ещё указывала на цель

// 4. Перенос: если ширина одной строки > --tip-max — включить white-space:normal (.tip--multiline)

// 5. Доступность: триггер должен быть фокусируем элементом; тултип не перехватывает фокус и закрывается по Esc без сброса фокуса с триггера

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функции makeTip / placeTip в tooltip.page.js).

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не обязательный контракт; адаптируйте под конвенции вашей кодовой базы. В React позиционирование обычно выносится в useLayoutEffect, чтобы избежать мелькания перед первым показом.

Tooltip.types.ts 
type TooltipType = 'main' | 'error';
type TooltipPlacement = 'top' | 'bottom' | 'left' | 'right';
type TooltipAlign = 'start' | 'center' | 'end';

interface TooltipProps {
  content: React.ReactNode;         // обязательно; тултип не рендерится без содержимого
  children: React.ReactElement;     // триггер — должен принимать ref и быть фокусируемым
  type?: TooltipType;               // по умолчанию 'main'
  placement?: TooltipPlacement;     // по умолчанию 'top'; авто-разворачивается при нехватке места
  align?: TooltipAlign;             // по умолчанию 'center'
  arrow?: boolean;                // по умолчанию true
  delay?: number;                 // задержка появления при hover в мс, по умолчанию 400
  disabled?: boolean;             // полностью отключить тултип
}
