Toast

Тёмная пилюля-уведомление, всплывающая у верхней границы рабочей области по центру, — индикатор запуска фонового или блокирующего процесса. Два режима: ToastBar (фоновый, авто-скрытие, стек до трёх) и ToastLoader (блокирующий, со спиннером и затемнением). Ведущий слот — спиннер или статус-иконка (взаимоисключающие); тоны Neutral · Success · Error · Info.

Конструктор

Соберите тост: режим, тон, ведущий слот, текст и показ (single / stack). Превью и код обновляются мгновенно. Кнопка «Проиграть» в превью показывает анимацию появления и скрытия.

Использование

Toast сообщает о запуске серверного процесса, результат которого отложен во времени, либо подтверждает инициированное пользователем действие. Он не требует реакции и не содержит действий — только информирует.

  • Фоновый процесс (ToastBar). На сервере запущен процесс, результат которого придёт позже: «Процесс запущен», «Запущена маршрутизация».
  • Подтверждение действия. Индикация того, что действие пользователя принято системой.
  • Блокирующий процесс (ToastLoader). На сервере запущен процесс, который блокирует работу системы, — с затемнением и спиннером: «Выполняется маршрутизация».
  • Не используем для сообщений, требующих решения или действия пользователя (это модалка / снэкбар с кнопкой), и для постоянных статусов (это Chip / Badge).

ToastBar — фоновый процесс

Слой под снэкбаром, без затемнения. Скрывается сам по таймауту.

ToastLoader — блокирующий процесс

Слой над снэкбаром, затемнение 25%, спиннер. Скрывается по событию с сервера.

ToastBar и ToastLoader — в чём разница

Один компонент, два режима. Выбор — по тому, блокирует ли процесс работу пользователя.

ToastBar фоновый

Процесс идёт в фоне, система доступна. Слой под снэкбаром, без затемнения. Показывается до трёх штук стеком. Скрывается по таймауту 3 сек (без лоадера) или по событию с сервера (с лоадером). Текст — глагол совершенного вида.

ToastLoader блокирующий

Процесс блокирует работу системы. Слой над снэкбаром, затемнение 25%, спиннер. Одновременно — только один. Если появляется ToastLoader, он замещает весь ToastBar. Скрывается по событию с сервера. Текст — глагол несовершенного вида.

Анатомия

Тост — тёмная пилюля с необязательным ведущим слотом (спиннер или статус-иконка — взаимоисключающие) и текстом сообщения. Ведущий слот выровнен по верхней строке текста — при переносе остаётся сверху слева.

  • Пилюля (контейнер)тёмная плашка StGrey (CGrey_700), радиус 8px, мягкая тень Elevation_2
  • Ведущий слот (опционально)спиннер (0.7 с linear) или статус-иконка (check-circle / alert, тон Success / Error / Info) — 20 px, не оба сразу
  • Текст сообщениябелый, SB Sans Text; при переносе выравнивается по левому краю под первой строкой

Максимальная ширина тоста — 60% от ширины рабочей области. При её достижении текст переносится на строку ниже и выравнивается с иконкой слева сверху.

Ширина и перенос текста

Ширина зависит от контента, высота фиксирована для одной строки. Максимум — 60% от ширины рабочей области; при достижении предела текст переносится, а ведущий слот прижимается к верхней строке.

Короткий текст — ширина по контенту

Длинный текст — упирается в 60% и переносится (слот сверху слева)

Ведущий слот

Слева от текста — необязательный слот: спиннер, статус-иконка или ничего. Спиннер и иконка взаимоисключают друг друга — одновременно показывается только одно. Переход «процесс → результат» — это замена спиннера на иконку во времени (см. «Переход Loading → Success»), а не оба глифа сразу.

Без слота — просто индикация запуска

Спиннер — процесс идёт

Статус-иконка — результат процесса

Состояния и тоны

Тост не реагирует на наведение и фокус — он информативный, не интерактивный. «Состояние» здесь — это тон результата процесса. Neutral и Loading были в исходной документации; Success и Error добавлены как недостающие финальные состояния фонового процесса.

Переход Loading → Success Добавлено

Тост с лоадером по завершении процесса не исчезает резко: спиннер сменяется галочкой Success, тост держится ещё ~1 сек и уходит фейдом вниз. Так пользователь успевает считать результат.

Стек и переполнение

Тосты добавляются по одному — новый появляется сверху. Одновременно показывается не более трёх. Что происходит с четвёртым при лимите — в исходной документации не описано; ниже — предлагаемое поведение.

Single

Один тост.

Stack — до трёх

Новые добавляются сверху, самый ранний — снизу.

Переполнение стека Добавлено

Когда в стеке уже три тоста, а приходит четвёртый — он добавляется сверху, а самый ранний (нижний) уходит фейдом вниз (принцип FIFO, «окно» из трёх). Тосты с лоадером не вытесняются по таймауту — они ждут события с сервера, поэтому в очередь на вытеснение попадают только тосты без лоадера.

Позиционирование и затемнение

Тост появляется в отдельном слое, горизонтально — по центру рабочей области, по вертикали — у верхней границы. Вокруг всего компонента — охранное поле 24 px (сверху и по бокам), чтобы тост не касался краёв. ToastBar живёт в слое под снэкбаром и не затемняет подложку; ToastLoader — в слое над снэкбаром с затемнением 25%.

ToastBar — без затемнения

Слой с ToastBar не затемняем. Пользователь продолжает работать.

ToastLoader — затемнение 25%

Охранное поле 24 px, подложка затемнена на 25%.

Анимация

Тосты добавляются в бар по одному — линейной анимацией сверху. Удаляются фейдом вниз. При prefers-reduced-motion: reduce остаётся только смена прозрачности, спиннер замедляется до 1.6 с.

Тайминги Добавлено

Явные значения длительностей и кривых — чтобы поведение было воспроизводимо в коде.

СобытиеДлительностьКриваяПримечание

Текстовка

Название короткое и понятное, однозначно описывает суть процесса. Формулируйте так, чтобы вид глагола отражал режим тоста.

Не так

ToastBar без лоадера — глагол несовершенного вида читается как «ещё не сделано».

Так

ToastBar без лоадера — глагол совершенного вида: действие принято, процесс запущен.

Не так

ToastLoader — совершенный вид противоречит идущему процессу.

Так

ToastLoader — глагол несовершенного вида: процесс в моменте.

  • ToastBar без лоадера → совершенный вид: «Процесс запущен», «Сделка ООО «Ромашка» маршрутизирована».
  • ToastLoader (со спиннером) → несовершенный вид: «Выполняется маршрутизация».
  • Success → совершенный вид результата: «Маршрутизация завершена».
  • Error → указывает на неуспех прямо: «Не удалось запустить процесс».

Доступность Добавлено

Тост не фокусируется и не участвует в порядке табуляции — он появляется и исчезает сам. Чтобы скринридер озвучил сообщение, тост объявляется через ARIA live-region.

Режим / тонrolearia-liveПочему
  • Спиннер и статус-иконка декоративны — aria-hidden="true"; состояние передаёт текст сообщения.
  • Смысл не кодируется одним цветом: Success и Error различаются иконкой (галочка / знак ошибки), а не только тоном.
  • ToastLoader затемняет и блокирует подложку — на время блокировки фокус не должен уходить в задизейбленную область под затемнением.
  • Тост не должен нести критическую информацию, которую нельзя пропустить: он исчезает сам и недоступен после скрытия.

Цвета

Пилюля — тёмный фон StGrey (CGrey_700, #333F48), текст белый (Text_OnDark). Тень — Elevation_2. Статус-иконки берут семантические токены раздела Situative (светлые рампы, читаемые на тёмном фоне): Success — Success_Light (LightGreen_400), Error — Error_Light (Red_200), Info — Info_Light (LightBlue_400). Затемнение ToastLoader — StGrey 25%.

Поведение · псевдокод (framework-agnostic) 
// 1. Очередь и лимит: одновременно видно не более MAX = 3 тостов.
//    ToastBar — стек; ToastLoader — единичный и замещает весь ToastBar.
MAX = 3

function pushToast(toast):
  if toast.kind == 'loader':            // блокирующий
    clearAllBars()                     // ToastLoader замещает весь ToastBar
    showScrim(0.25); showLoader(toast) // затемнение + слой над снэкбаром
    return
  // bar: новый добавляем СВЕРХУ; при переполнении вытесняем самый ранний (нижний) без лоадера
  visibleBars.unshift(toast); animateIn(toast)  // новый сверху, linear
  if visibleBars.length > MAX:
    dismiss(oldestDismissableBar())    // нижний уходит фейдом вниз, FIFO-окно из трёх

// 2. Авто-скрытие: тост БЕЗ лоадера — по таймауту 3с; тост С лоадером —
//    только по приходящему с сервера событию (успех / ошибка).
function onShown(toast):
  if !toast.hasLoader: setTimeout(() => dismiss(toast), 3000)

// 3. Loader → Success: по событию сервера спиннер меняется на галочку,
//    тост держится ~1с и уходит фейдом вниз.
function onServerDone(toast, result):
  toast.setTone(result.ok ? 'success' : 'error')
  toast.swapSpinnerToIcon()
  setTimeout(() => dismiss(toast), 1000)

// 4. dismiss = класс .toast--leave (фейд вниз), удаление узла по animationend.
// 5. reduced-motion: только opacity, без translateY; спиннер 1.6с.

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не жёсткий контракт; адаптируйте под свою кодовую базу.

Toast.types.ts 
type ToastKind = 'bar' | 'loader';      // фоновый | блокирующий
type ToastTone = 'neutral' | 'success' | 'error' | 'info';

interface ToastOptions {
  message: string;             // текст, вид глагола по kind/tone
  kind?: ToastKind;             // по умолчанию 'bar'
  tone?: ToastTone;             // по умолчанию 'neutral'
  loader?: boolean;            // показать спиннер (у kind='loader' — всегда)
  durationMs?: number | null;  // null = ждать событие сервера; bar без лоадера — 3000
}

// императивный контроллер очереди/стека (провайдер на уровне приложения)
interface ToastController {
  show(opts: ToastOptions): ToastHandle;   // поставить в очередь / показать
  update(h: ToastHandle, patch: Partial<ToastOptions>): void; // loader→success
  dismiss(h: ToastHandle): void;
  // invariants: одновременно ≤ 3 bar; loader единичен и замещает bar
}
