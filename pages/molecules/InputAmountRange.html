InputAmountRange

Поле ввода числового диапазона: два InputAmount с префиксами «От» / «До», размещённых горизонтально и соединённых линией Range_Line. Общая метка сверху и общий хелпер снизу. Каждое поле — самостоятельный экземпляр со своими состояниями. Стандартная высота поля 40px; размера S у InputAmountRange нет.

Конструктор

Соберите диапазон: состояние левого и правого поля независимо, наполнение, метка и хелпер. Ввод живой — крестик очищает поле.

Использование

InputAmountRange задаёт числовой диапазон «от … до …» в фильтрах и формах: сумма сделки, лимит, объём. Единственный размер — M. Одно из полей может остаться пустым — тогда граница диапазона открыта.

В форме — общая метка и хелпер, оба поля заполнены.
Пустой диапазон — оба поля показывают плейсхолдер «Amount».
Открытая граница — заполнено только «От».

Когда использовать

  • задать числовой диапазон: сумма, лимит, количество, ставка;
  • фильтр «от» / «до» с возможностью открытой границы (одно поле пустое).

Для одиночного числа — InputAmount (числовой InputText). Для диапазона дат — InputDateRange. Для выбора из списка — InputAutocomplete.

Анатомия

Компонент состоит из общей метки, двух полей InputAmount, размещённых горизонтально, с компонентом Range_Line между ними и хелпера (опционально). В полях всегда присутствует префикс «От» / «До» для обозначения начала и конца диапазона.

  • Label — общая метка диапазона, .ds-label (Body XS)
  • Input × 2 — два поля InputAmount с префиксом «От» / «До», высота 40px (M)
  • Range_Line — линия между полями; толщина и цвет аналогичны бордеру инпута (1px, --border-primary)
  • Helper — общий хелпер под диапазоном, .ds-helper (Body XS)

Варианты

Оси конфигурации: наличие хелпера и наполнение полей.

С хелпером и без

Наполнение

Оба поля могут быть пустыми, заполнено одно (открытая граница) или оба.

Размеры

Единственный размер — M, высота поля 40px. Размера S у InputAmountRange нет. Ширина диапазона задаётся контейнером, поля делят её поровну.

Контент

Правила контента:

  • префикс «От» / «До» присутствует всегда — обозначает начало и конец диапазона, неизменяем;
  • пустое поле показывает плейсхолдер «Amount» — отдельный плейсхолдер-текст не пишется;
  • метка — существительное с единицей: «Сумма, ₽», не «Введите сумму»;
  • хелпер — правило заполнения; в состоянии Error заменяется текстом ошибки.

Поведение

Для компонента действуют те же правила, что и для InputAmount: фильтр ввода [0-9.,-], группировка целой части по три разряда, обрезка нулей на blur. Крестик очистки — только у заполненного поля.

Независимость полей

С каждым полем пользователь взаимодействует отдельно: ховер или фокус на левом поле не вызывают реакции на правом и наоборот. То же касается состояния ошибки — оно распространяется только на то поле, в котором находится невалидное значение. При этом оба поля могут находиться в состоянии ошибки одновременно, если у каждого содержится невалидное значение.

Состояния

Восемь состояний, общих для базового поля. Состояние показано на левом поле «От»; правое поле «До» — в Default. Текст ошибки или предупреждения по умолчанию не выносится в хелпер (хелпер остаётся правилом заполнения) — сообщение показывается только в тултипе в состояниях ErrorFocus и WarningFocus. Тултип не смещает хелпер: лежит поверх потока, z-index выше поля. Исключение из правила допустимо, если оно намеренно заложено при разработке.

Доступность

Общая метка связывается с полями через label / aria-label, хелпер — через aria-describedby, ошибка каждого поля — aria-invalid="true" только на невалидном поле. Range_Line — декоративный, помечен aria-hidden="true". Оба поля доступны с клавиатуры и озвучиваются отдельно.

Цвета

Токены совпадают с InputText / InputAmount. Range_Line использует тот же токен рамки, что и поле: --border-primary; в состоянии Disabled — --border-light.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Каждое поле — экземпляр базового .inp (см. InputText); здесь только обвязка диапазона.
