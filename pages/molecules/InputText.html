InputText

Базовое поле ввода произвольного текста. Собирается из компонентов подкачки LabelHelper (метка и хелпер) и Input_Content (поле с рамкой: префикс / текст / постфикс, иконки и действия). На той же базе построены InputDate и InputAutocomplete, а также числовой вариант InputAmount.

Конструктор

Соберите поле: размер, состояние, наполнение текстового слоя (префикс, постфикс, иконка поиска), действия и многострочный режим.

Использование

InputText — поле для свободного текста: наименования, комментарии, реквизиты. Основной размер — M; размер S используется только внутри таблиц (Table Edit).

В форме — с меткой и хелпером; ширина поля задаётся сеткой формы.
Table Edit — размер S внутри ячейки таблицы, без метки и хелпера.
InputAmount — числовой вариант с разрядами и постфиксом валюты.

Когда использовать

  • ввод произвольного текста, который нельзя выбрать из перечня: наименование, комментарий, реквизит;
  • ввод чисел и сумм — вариант InputAmount (разряды, запятая, децималы);
  • редактирование значения прямо в ячейке таблицы — размер S (Table Edit).

Если значение выбирается из справочника — возьмите InputAutocomplete или Select. Для даты — InputDate. Показ значения без редактирования — ReadOnlyField.

Анатомия

Поле состоит из метки, контейнера Input_Content с текстовым слоем и действиями, и хелпер-текста. Все части, кроме самого поля, опциональны.

  • Label — метка поля, компонент подкачки .ds-label (Body XS)
  • Поле (Input_Content) — рамка 1px, радиус 8px, высота 40px (M)
  • Текстовый слой (Text_MASTER) — иконка поиска · префикс · текст · постфикс
  • Действия (Input_Action) — информер и крестик очистки, справа (информер левее крестика)
  • Helper — подсказка под полем, .ds-helper (Body XS)

Варианты

Оси конфигурации: наполнение текстового слоя, многострочный режим, числовой InputAmount и табличный Table Edit.

Текстовый слой

Слой Text_MASTER комбинирует префикс, текст и постфикс с иконкой поиска слева. Все элементы независимы; лишние не показываются.

Многострочный

Для длинного текста (комментарии, описания) поле растёт по высоте. Действия прижаты к верхней строке.

InputAmount

Вариант для ввода числовых значений; полностью наследует стили InputText. Правила ввода:

  • вводятся только цифры, точка, запятая и минус — остальные знаки игнорируются;
  • точка при вводе заменяется на запятую;
  • целая часть разделяется на разряды по 3 знака;
  • по умолчанию после запятой два знака; длина децималов настраивается;
  • незначащие нули убираются при потере фокуса («01» → «1», «…,010» → «…,01»); запись «1,00» с двумя нулями допустима.

Table Edit

Вариант поля в таблице: размер S, никогда не содержит метку и хелпер. В остальном полностью идентичен обычному полю — со всеми состояниями и видами контента.

Размеры

Два размера. M — основной размер поля в формах и фильтрах. S — только для вариантов, используемых в таблицах (Table Edit). Ширина поля задаётся контейнером.

Контент

Правила текста в поле и вокруг него:

  • метка — существительное без двоеточия: «ИНН контрагента», не «Введите ИНН:»;
  • плейсхолдер не дублирует метку — это формат или пример значения («Например, 7707083893»);
  • хелпер — короткое правило заполнения; в состоянии Error заменяется текстом ошибки;
  • префикс и постфикс — единицы и неизменяемые части значения (₽, %, «От»), не вводятся пользователем;
  • текст длиннее поля прокручивается внутри однострочного поля; для заведомо длинного текста берите многострочный вариант.

Поведение

Плейсхолдер, очистка и адаптивность. Все примеры живые — печатайте в полях.

Плейсхолдер и очистка

Пустое поле по умолчанию не отображает ничего; при необходимости в него помещается плейсхолдер. Плейсхолдер виден, пока поле не перейдёт в Focus и не начнётся ввод. Крестик появляется только у заполненного поля; если пользователь очистил поле — по потере фокуса оно возвращается в пустое состояние.

Адаптивность

Собственной ширины у поля нет — её задаёт контейнер (сетка формы, ячейка таблицы). Высота фиксирована по размеру; растёт только многострочный вариант.

Состояния

Восемь состояний, общих для всех типов поля (InputText, InputDate, InputAutocomplete). Текст ошибки или предупреждения по умолчанию не выносится в хелпер — хелпер остаётся правилом заполнения — и показывается только в тултипе в состояниях ErrorFocus и WarningFocus. Тултип не смещает хелпер: лежит поверх потока, z-index выше поля. Исключение из правила допустимо, если оно намеренно заложено при разработке.

Доступность

Метка связывается с контролом через label[for], хелпер и текст ошибки — через aria-describedby. Ошибка помечается aria-invalid="true" и передаётся не только цветом: текст ошибки выводится в хелпере или тултипе. Крестик очистки — настоящая кнопка с aria-label.

Клавиатура

Tabфокус в поле; следующий Tab — на крестик очистки (если есть)
Escотменяет незафиксированный ввод (если реализовано формой)
Enterподтверждает значение / отправляет форму

Типографика

Текст поля набирается гарнитурой SB Sans Text. Метка и хелпер — Body XS.

Цвета

Компонент использует только семантические токены. Рамка — --border-primary, фокус — --primary, ошибка — --error, предупреждение — --warning.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки.

Точные размеры (redline)

Значения измерены на живом экземпляре компонента (getComputedStyle), источник — токены в styles/input.css.
