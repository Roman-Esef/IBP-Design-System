InputDate

Компонент ввода даты: поле с маской и кнопкой-календарём. При фокусе на поле или по кнопке-календарю поднимается DatePicker. Построен на той же базе Input_Content, что и InputText и InputAutocomplete, — состояния и размеры общие.

Конструктор

Соберите поле даты: размер, состояние, наполнение, информер. Ввод живой — маска форматирует цифры по шаблону.

Использование

InputDate — единственный способ ввода даты в формах: с клавиатуры по маске или выбором в календаре. Основной размер — M; размер S используется только внутри таблиц (Table Edit).

В форме — с меткой и хелпером; пустое поле показывает маску-плейсхолдер.
Заполненное поле — появляется крестик очистки перед календарём.
Table Edit — размер S внутри ячейки таблицы, без метки и хелпера.

Когда использовать

  • ввод конкретной даты: подписание, транш, плановая дата;
  • редактирование даты в ячейке таблицы — размер S (Table Edit).

Для произвольного текста — InputText; для выбора из справочника — InputAutocomplete. Диапазон дат («с … по …») собирается из двух InputDate с префиксами «От» / «До».

Анатомия

Отличие от InputText — фиксированное действие-календарь в правой части. Крестик очистки появляется у заполненного поля, информер опционален (обычно его нет).

  • Label — метка поля, .ds-label (Body XS)
  • Поле (Input_Content) — рамка 1px, радиус 8px, высота 40px (M)
  • Значение / маска — дата по шаблону; пустое поле показывает маску как плейсхолдер
  • Действия — информер (опционально) · крестик очистки · календарь (информер всегда левее крестика)
  • Helper — подсказка под полем, .ds-helper (Body XS)

Варианты

Оси конфигурации: хелпер, информер и табличный Table Edit.

С хелпером и без

Информер

Иконка информера опциональна, обычно её нет. Ставится между крестиком и календарём.

Table Edit

Вариант поля в таблице: размер S, с иконкой поиска слева, никогда не содержит метку и хелпер. В остальном идентичен обычному полю со всеми его состояниями.

Размеры

Два размера. M — основной размер в формах и фильтрах. S — только для вариантов, используемых в таблицах (Table Edit). Ширина поля задаётся контейнером.

Контент

Правила контента:

  • маска — ММ.ДД.ГГГГ; шаблон настраивается на уровне продукта и одинаков во всех полях экрана;
  • пустое поле показывает маску как плейсхолдер — отдельный плейсхолдер-текст не пишется;
  • метка — существительное: «Дата подписания», не «Введите дату»;
  • хелпер — правило заполнения («Не раньше даты договора»); в состоянии Error заменяется текстом ошибки.

Поведение

Маска, календарь и плейсхолдер.

Маска ввода

Вводятся только цифры — точки-разделители подставляются автоматически. Плейсхолдер-маска отображается, пока поле не перейдёт в состояние Focus и не начнётся ввод. Если пользователь очистил поле — по потере фокуса оно возвращается в пустое состояние. Пример живой — печатайте.

DatePicker

При фокусе на поле под ним поднимается календарь DatePicker; клик по кнопке-календарю открывает его явно. Выбор даты в календаре записывает значение в поле (маска ММ.ДД.ГГГГ) и закрывает календарь. Календарь — компонент DatePicker; здесь он подключён рантаймом ds-datepicker.js — кликните по кнопке-календарю в примерах ниже.

Состояния

Восемь состояний, общих для всех типов поля. Текст ошибки или предупреждения по умолчанию не выносится в хелпер — хелпер остаётся правилом заполнения — и показывается только в тултипе в состояниях ErrorFocus и WarningFocus. Тултип не смещает хелпер: лежит поверх потока, z-index выше поля. Исключение из правила допустимо, если оно намеренно заложено при разработке.

Пусто · Заполнено

Доступность

Метка связывается с контролом через label[for], хелпер — через aria-describedby, ошибка — aria-invalid="true". Кнопка-календарь — настоящая кнопка с aria-label="Открыть календарь" и aria-haspopup="dialog"; дата вводима полностью с клавиатуры по маске, без обязательного календаря.

Tabфокус в поле → крестик → календарь

Enter / Spaceна кнопке-календаре — открыть DatePicker

Цвета

Токены совпадают с InputText: рамка — --border-primary, фокус — --primary, ошибка — --error, предупреждение — --warning, маска-плейсхолдер — --text-inactive.
