Breadcrumbs

Хлебные крошки — трейл ссылок в шапке страницы, показывающий путь до текущего раздела. Состоят из ссылок Link (Breadcrumbs_1 — родительские страницы, Breadcrumbs_2 — текущая, некликабельная) и разделителя «/». Показываются на всех страницах системы, кроме Дашборда; отступ до контента страницы — 0.

Конструктор

Соберите трейл: число уровней и ширина контейнера — превью схлопывает срединные звенья в «…» и обрезает текущую крошку, когда не помещается. Размер один — S.

Использование

Показывайте хлебные крошки, когда нужно:

  • дать понять, где находится страница в иерархии разделов, и позволить быстро вернуться на любой уровень выше;
  • заменить собой заголовок предыдущего уровня, когда страница вложена на 2+ уровня от корня.

Крошки показываются на всех страницах системы, кроме Дашборда — там нет вышестоящего раздела, показывать нечего. Правило распространяется на любую страницу первого уровня: если у страницы нет родителя, компонент не выводится вовсе (см. «Контент»). Крошки размещаются первым элементом шапки, отступ до контента страницы — 0.

Анатомия

Крошки — это список ссылок (Breadcrumbs_1) с текущей неинтерактивной крошкой в конце (Breadcrumbs_2). Между звеньями — разделитель «/» того же цвета, что и родительские ссылки. Когда трейл не помещается по ширине, срединные звенья схлопываются в кликабельное «…».

  • Родительское звено — ссылка тона Muted (Breadcrumbs_1), обрезается многоточием при длинном названии
  • Разделитель «/» — цвет совпадает с родительскими звеньями, не читается скринридером
  • Свёрнутое звено «…» — раскрывает список скрытых страниц по клику
  • Текущая крошка — тон Neutral (Breadcrumbs_2), обычное начертание (как и у Breadcrumbs_1) — отличается только цветом, не кликабельна, aria-current="page"

Размеры

У компонента единственный размер — S (--type-body-s, 14/16). Второго размера нет: крошки — вспомогательная навигация над заголовком страницы и не должны конкурировать с ним по весу, поэтому размер не варьируется.

Контент

Правила именования и корнер-кейсы:

  • показывайте только те разделы пути, что имеют собственную страницу — служебные/технические сегменты URL в крошки не попадают;
  • называйте звено так же, как называется сам раздел (заголовок страницы) — не сокращайте и не переформулируйте;
  • текущая страница выводится текстом, без ссылки — aria-current="page";
  • если у страницы нет родителя (страница первого уровня), компонент не показывается — крошка из одного звена не несёт смысла.Новое
Не так

Единственная крошка «Главная» не несёт навигационной пользы — на странице первого уровня компонент не нужен.

Так

Длинное родительское звено обрезается многоточием (title показывает полное название), а не переносится на вторую строку.Новое

Состояния

Кроме базового состояния из спецификации, документированы наведение/фокус на родительской ссылке, авто-схлопывание по ширине и обрезка текущей крошки — с раскрывающимся списком скрытых страниц.Новое

Доступность

Контейнер — ориентир <nav aria-label="Хлебные крошки"> со списком <ol>/<li> внутри (паттерн WAI-ARIA APG для breadcrumb). Текущая страница — не ссылка и не фокусируется, помечена aria-current="page". Разделитель «/» — CSS-псевдоэлемент (::after), поэтому не входит в дерево доступности и не зачитывается скринридером. Focus у ссылок оформлен идентично Hover — единое цветовое решение для мыши и клавиатуры, без отдельной обводки. Кнопка «…» — обычная <button> с aria-haspopup / aria-expanded, раскрывающееся контекстное меню закрывается по Esc и клику вне.

  • Tab проходит только по кликабельным звеньям (ссылки, «…», если он есть) — текущая крошка и разделители фокус не получают;
  • обрезанное длинное название доступно через тултип по фокусу/наведению, а не только по hover — важно для клавиатурной навигации;
  • цветовой контраст Breadcrumbs_1 (--text-inactive) на --bg-tile/--bg-page соответствует WCAG AA для текста этого размера.

Цвета

Компонент использует только семантические токены — те же, что определены для тонов Muted и Neutral компонента Link.

Поведение · псевдокод (framework-agnostic) 
// 1. Рендер полного трейла: все звенья, кроме последнего, — ссылки (Breadcrumbs_1),
//    последнее — текст с aria-current="page" (Breadcrumbs_2).

// 2. Измерить: scrollWidth контейнера > clientWidth?
//    Если да и звеньев больше 2 — заменить звенья [1 .. N-2] на одно
//    звено-кнопку «…» (оставляя первое и текущее звено видимыми).
//    Измерение лучше делать в скрытом клоне той же ширины — тогда
//    пользователь не увидит промежуточного «полного» кадра.

// 3. Текущая крошка ВСЕГДА однострочная и обрезается многоточием по CSS
//    (overflow:hidden + text-overflow:ellipsis + white-space:nowrap) —
//    независимо от того, схлопнулись ли срединные звенья. Если это
//    реально обрезало текст (scrollWidth > clientWidth) — показать
//    полное название в тултипе по hover/focus.

// 4. Клик по «…»: открыть контекстное меню (см. Context Menu) со списком скрытых
//    страниц; клик по строке — переход; Esc/клик вне — закрыть.
//    aria-expanded кнопки синхронизируется с открыт/закрыт.

// 5. Пересчитывать схлопывание при ресайзе контейнера (ResizeObserver)
//    или ширины окна — не только при первом рендере.

// 6. Если у страницы нет родителей — компонент не рендерится вовсе.

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функция renderCrumbs в breadcrumbs.page.js этой страницы).

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не обязательный контракт; адаптируйте под конвенции вашей кодовой базы.

Breadcrumbs.types.ts 
interface BreadcrumbItem {
  label: string;   // текст звена — совпадает с заголовком страницы
  href?: string;   // не указывается у последнего элемента (текущая страница)
}

interface BreadcrumbsProps {
  items: BreadcrumbItem[];        // от корня к текущей странице; <2 элементов → не рендерить
  onNavigate?: (href: string) => void; // клик по звену/по пункту меню «…»
  // единственный размер — S; size-пропа нет
}
