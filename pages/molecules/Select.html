Select · DropdownList

Выпадающий список — всплывающая поверхность со списком опций, которая раскрывается под полем Select или InputAutocomplete. Служит для выбора значения из перечня: одиночного (подсветка выбранной строки), множественного (ведущие чекбоксы) или с поиском по мере набора (подсветка совпадения). В отличие от Context Menu — это выбор данных, а не список команд.

Конструктор

Соберите список: вариант (текст / чекбокс), helper-текст, ведущие иконки, подсветку совпадения, группировку и системные состояния. Список закреплён открытым — меняйте настройки и смотрите результат.

Использование

Триггер — поле Select или InputAutocomplete. По клику под полем раскрывается список; его ширина равна ширине поля. Нажмите на поле в каждом примере.

Select · одиночный выбор — выбранная опция подсвечена заливкой строки.
Multiselect — ведущие чекбоксы, список не закрывается после выбора.
Autocomplete — фильтрация по вводу, совпавшая подстрока выделена жирным.

Когда использовать

Выпадающий список — способ выбрать значение из перечня, когда вариантов много или место ограничено. Используйте, когда нужно:

  • выбрать одно значение из 5+ вариантов (валюта, контрагент, счёт) — вместо радиокнопок;
  • выбрать несколько значений — вариант с ведущими чекбоксами;
  • найти значение в длинном справочнике по мере набора — в паре с InputAutocomplete.

Для 2–4 взаимоисключающих вариантов на виду возьмите SegmentControl или Radiobutton. Для списка команд-действий над объектом — Context Menu: у него hover нейтрально-серый, выбор — галочка справа, а здесь выбор — заливка строки и (для multiselect) чекбокс.

Анатомия

Список — контейнер-поповер с вертикальным перечнем опций. Опция состоит из ведущего слота (иконка или чекбокс), тела (подпись + опциональный helper) и опциональной правой части. Опции можно группировать липким заголовком.

  • Контейнер — фон, радиус 12px, тень, паддинг 4px по вертикали
  • Заголовок группы — липкий (sticky), Body XS uppercase
  • Опция — ведущий слот + подпись, высота 40px
  • Helper — вторая строка, компонент подкачки .ds-helper (Body XS, Text-Inactive)
  • Выбранная опция — заливка строки + усиленная подпись

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Варианты опции Dropdown_Item

Четыре варианта опции × пять состояний. Ведущий слот — либо иконка, либо чекбокс (для множественного выбора); helper-текст добавляет вторую строку. Матрица повторяет исходный компонент.

Размеры

У опции один размер. Высота строки фиксирована для одной строки; helper-текст даёт вторую строку и увеличивает высоту. Ширина контейнера равна ширине поля-триггера (не меньше 200px).

Контент

Подпись опции — существительное-значение, коротко и однозначно. Правила:

  • по умолчанию подпись в одну строку с многоточием при переполнении; перенос (.ddl__item--wrap) — только когда полный текст критичен для выбора;
  • helper-текст — короткое уточнение (код, ISO, счёт), не дублирует подпись;
  • в автокомплите подсвечивайте совпавшую подстроку жирным (.ddl__match), а не меняйте цвет;
  • порядок: сначала популярное/выбранное, затем — по алфавиту; группируйте длинные списки заголовками.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Позиционирование и авто-разворот

Список выравнивается по левому краю поля, его ширина равна ширине поля. Если снизу не хватает места, список разворачивается вверх. Перетащите поле к нижнему краю области.

Состояния

Опция меняет фон при наведении и фокусе, темнеет при нажатии. Выбранная (одиночный выбор) — залита фоном «выбранной строки» и усилена начертанием. Отключённая — приглушена и не реагирует на курсор. Фокус клавиатуры (активная опция) визуально отделён от hover.

Системные состояния

Когда показывать нечего, вместо списка опций выводится одна системная строка. Три случая — загрузка, пустой результат и ошибка — плюс корнер-кейс автокомплита со свободным вводом («создать»).

Доступность

Контейнеру присваивается role="listbox" (для множественного выбора — aria-multiselectable="true"), опциям — role="option" с aria-selected. Поле-триггер получает role="combobox", aria-expanded и aria-controls; активная опция помечается через aria-activedescendant без потери фокуса из поля. Список полностью управляется с клавиатуры и закрывается по Esc.

Клавиатура

Активная опция (фокус клавиатуры) визуально отделена от выбранной

Не полагайтесь только на цвет: выбранная опция несёт не только заливку, но и усиленное начертание (одиночный выбор) либо заполненный чекбокс (множественный). Ошибка в системной строке помечается иконкой Error, а не только красным.

Типографика

Подписи набираются гарнитурой SB Sans Text. Helper и заголовок группы — Body XS.

Цвета

Компонент использует только семантические токены. Фон списка — --bg-popup, hover опции — --tertiary-light, выбранная строка — --bgtable-row-focus, заливка чекбокса — --primary.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки.

// 1. Открытие: клик/стрелка вниз/ввод в поле → показать список, aria-expanded=true.
//    Фокус остаётся в поле; активная опция помечается aria-activedescendant.

// 2. Ширина и позиция: ширину списка приравнять к ширине поля; открыть под полем
//    (gap 6px), выровнять по левому краю. Не хватает места снизу → развернуть вверх.

// 3. Клавиатура: ↑/↓ двигают активную опцию (пропуская disabled), Home/End — край,
//    Enter выбирает (multiselect — переключает), Space — переключает чекбокс,
//    typeahead по первой букве, Esc закрывает и возвращает фокус в поле.

// 4. Одиночный выбор: клик по опции → aria-selected на ней (снять с прочих),
//    записать значение в поле, закрыть список.

// 5. Множественный выбор: клик переключает aria-checked, список НЕ закрывается;
//    в поле — сводка («N выбрано» / чипы). Закрытие по клику вне и Esc.

// 6. Автокомплит: по вводу фильтровать опции, подсвечивать совпавшую подстроку
//    (.ddl__match), дебаунсить запрос; показать loading → results / empty / error.

// 7. Скролл: при переполнении контейнер прокручивается; активная опция всегда
//    подкручивается в видимую область; заголовки групп липкие (sticky).
