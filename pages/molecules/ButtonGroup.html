ButtonGroup

Объединяет связанные кнопки в единый визуальный блок с общим фоном или обводкой. Поддерживает 2–5 кнопок, типы Accent / Outline, все три размера, вариант Split Button и сегментированный выбор. Собирается из компонентов Button — кнопки наследуют свои свойства.

Конструктор

Соберите группу: переключайте вариант, тип, размер, количество кнопок, ориентацию и состояние — превью обновляется мгновенно.

Использование

ButtonGroup группирует кнопки, связанные общим контекстом или относящиеся к одному объекту: переключение режимов отображения, набор действий над записью, парные действия «Применить / Отмена». Группа делает связь между кнопками явной и экономит горизонтальное пространство.

Рекомендации

Группируйте только связанные действия. Не смешивайте типы и размеры внутри одной группы.

Делайте

Один тип, один размер, 2–5 связанных действий. Disabled — на всю группу.

Не делайте

Не смешивайте типы и размеры и не блокируйте отдельную кнопку внутри группы.

Анатомия

Собирается из компонентов Button, объединённых общим фоном (Accent) или обводкой (Outline). Соседние кнопки разделены вертикальной линией: у Accent — окрашенный шов, у Outline — общая обводка соседних кнопок. Внешние углы скруглены по радиусу кнопки, внутренние — прямые.

Split Button

Отдельный вариант группы из двух кнопок: текстовой и стрелки. Клик по текстовой кнопке выполняет действие по умолчанию, клик по стрелке открывает контекстное меню с другими доступными действиями. Нажмите, чтобы открыть.

Левая кнопка — с иконкой или без неё

Позиционирование меню определяется на этапе проектирования — справа или слева

Ориентация

Помимо горизонтальной, группа поддерживает вертикальную раскладку — для компактных меню и боковых панелей, где действия выстраиваются в столбец. Скругление переносится на верхнюю и нижнюю кнопки, шов становится горизонтальным.

Сегментированный выбор

Группа может работать как переключатель: кнопки сохраняют выбранное состояние, выбранный сегмент заливается акцентным цветом. Поддерживается одиночный выбор (как радиокнопки — всегда выбран ровно один) и множественный (как чекбоксы). Управляется атрибутом aria-pressed. Попробуйте нажать.

Fullwidth и равная ширина

Группа со свойством Fullwidth заполняет контейнер целиком; кнопки получают равную ширину. Удобно для нижних панелей действий в модальных окнах и на мобильных макетах.

Размеры

Все кнопки группы используют один размер, унаследованный от Button. Высота фиксирована, ширина зависит от контента.

Спецификация

Геометрия наследуется от Button; группа управляет только скруглением внешних углов и швом между кнопками.

Контент

Текстовые правила наследуются от Button, но внутри группы есть дополнительные ограничения:

  • текст кнопки — короткий глагол в повелительном наклонении («Сохранить», «Неделя»), без точки в конце;
  • держите метки одного типа сопоставимыми по длине — сильно разные по ширине кнопки делают группу визуально неровной;
  • не смешивайте в одной группе кнопки с текстом и чисто icon-only — выбирайте один формат для всей группы;
  • у каждой icon-only кнопки в группе должен быть aria-label, описывающий действие;
  • в Split Button текстовая кнопка называет действие по умолчанию, пункты меню стрелки — альтернативные действия, связанные с тем же объектом.

Поведение

Логика выбора в режиме Toggle.

Одиночный и множественный выбор

В режиме Toggle группа работает по одной из двух схем: одиночный выбор (radio-like) — клик по кнопке снимает выбор со всех соседей и ставит только на неё, всегда ровно один нажатый сегмент; множественный выбор (checkbox-like) — каждая кнопка переключается независимо, допустимы любое число нажатых от нуля до всех. Схема задаётся на уровне группы, а не смешивается внутри одной группы.

Состояния

Кнопки внутри группы наследуют все свои свойства и ведут себя так же, как вне группы. Исключение — Outline: в группе у него не меняется цвет обводки на Hover / Active (иначе соседний шов «прыгал» бы).

Disabled — на всю группу целиком

Состояние Disabled распространяется на всю группу. В группе не может быть заблокирована только одна или часть кнопок — это нарушило бы единство блока.

Доступность

Контейнер группы получает role="group" и осмысленный aria-label (например «Режим отображения»). В сегментированном выборе используется aria-pressed на кнопках; одиночный выбор удобно навигировать стрелками с roving tabindex. Split Button: текстовая кнопка — основное действие, у стрелки aria-haspopup="menu" и aria-expanded.

Типографика

Не применимо: группа не вводит собственной типографики — кнопки внутри используют токены компонента Button.

Цвета

Группа наследует цвета от Button и добавляет собственные токены только для шва между кнопками.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

Значения измерены на живом экземпляре компонента (getComputedStyle), источник — токены в styles/button-group.css и styles/button.css.

Поведение · псевдокод (framework-agnostic) 
// 1. Кнопки внутри группы — обычные <button>, события click обрабатываются как обычно

// 2. Toggle / сегментированный выбор:
//    одиночный (radio-like) — клик по кнопке сбрасывает aria-pressed у всех соседей,
//    ставит true только нажатой; всегда ровно один выбранный
//    множественный (checkbox-like) — клик инвертирует только свой aria-pressed, независимо от соседей

// 3. Split Button:
//    клик по текстовой кнопке — выполняет действие по умолчанию напрямую
//    клик по стрелке — переключает aria-expanded и показывает/скрывает меню;
//    клик вне группы или Esc — закрывает меню

// 4. Disabled — только на всю группу целиком: класс .btn-group--disabled (pointer-events: none)
//    + aria-disabled="true" на контейнере; нельзя блокировать отдельную кнопку

// 5. Клавиатура: Tab переходит между кнопками внутри группы как обычные фокусируемые элементы
//    (без roving tabindex); :focus-visible — обычное поведение Button, поверх соседей (z-index)

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функции makeGroup / makeSplit в коде этой страницы).

Бэклог

Кейсы, которых недостаёт документации, — на существующих токенах и компонентах.

  • Группа с бейджем — счётчик на отдельной кнопке группы (например, число непрочитанных).
  • Адаптивное схлопывание — при нехватке ширины лишние кнопки уходят в оверфлоу-меню.
