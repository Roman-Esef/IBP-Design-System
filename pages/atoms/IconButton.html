IconButton

Безрамочная кнопка с одной иконкой и круглым стейт-слоем (риплом). Цвет рипла соответствует цвету иконки. Тоны Neutral / Primary / Danger / Contrast, размеры L / M / S, состояния Default / Hover / Pressed / Focus / Disabled / Loading, режим-переключатель и встроенный вариант для полей ввода.

Конструктор

Соберите IconButton: переключайте тон, размер, иконку, состояние, форму и модификаторы — превью обновляется мгновенно.

Использование

IconButton подходит, когда:

  • действие очевидно из иконки и не требует подписи — закрыть, удалить, редактировать, открыть меню;
  • места мало: тулбары, заголовки карточек, строки таблиц, поля ввода;
  • нужно собрать ряд однотипных действий, не перегружая интерфейс текстом.

Если действие основное или его смысл не считывается с иконки — используйте обычную Button с текстом. Иконка без подписи всегда должна сопровождаться aria-label и, как правило, Tooltip.

В тулбаре

Типовой паттерн: ряд IconButton одного тона и размера, сгруппированный с разделителями. Деструктивное действие выделяется тоном Danger.

Анатомия

Компонент состоит из иконки и круглого стейт-слоя вокруг неё. Стейт-слой появляется на Hover / Pressed и заполняет всю область компонента; его цвет — это полупрозрачный оттенок цвета иконки. Кликабельная область равна размеру компонента.

  • Иконка — задаёт тон; размер привязан к размеру кнопки
  • Стейт-слой (рипл) — круг, оттенок цвета иконки, виден на Hover / Pressed
  • Область клика — равна размеру компонента (L 40 · M 32 · S 24)

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Тоны

Тон задаёт цвет иконки, а стейт-слой автоматически берёт полупрозрачный оттенок этого же цвета. Neutral — нейтральные действия, иконка --secondary (Active · Secondary, Emerald_200); Primary — акцентные (--primary); Danger — деструктивные, например удалить (--error); Contrast — для тёмных поверхностей (--text-on-dark).

Режим-переключатель

IconButton может работать как переключатель (toggle): нажатие фиксирует состояние Selected. В выбранном состоянии иконка окрашивается в Primary и остаётся лёгкий стейт-слой; чаще всего контурная иконка заменяется на залитую. Подходит для «в избранное», «закрепить», «показать пароль». С бейджем

IconButton может нести индикатор-Badge — счётчик непрочитанного или точку-индикатор. Бейдж прижимается к правому верхнему углу иконки и не перехватывает клики.

Контент

У IconButton нет видимого текста — контент компонента это иконка и её текстовое описание:

  • иконка — только из библиотеки ДС (24×24, заливка currentColor); метафора должна быть общепринятой: корзина — удалить, карандаш — редактировать;
  • aria-label — глагол действия, 1–3 слова: «Удалить», «Скачать PDF»; не название иконки («Корзина»);
  • текст Tooltip совпадает с aria-label;
  • в режиме-переключателе подпись отражает текущее состояние: «В избранное» ↔ «В избранном».

Иконография

Иконки берутся из библиотеки ДС (24×24, заливка currentColor) и масштабируются под размер кнопки. Никаких эмодзи или юникод-символов.

Поведение

Независимость режима-переключателя и блокировка в Loading.

Режим-переключатель

В отличие от ButtonGroup Toggle, где сегменты одной группы могут быть взаимоисключающими, отдельный IconButton в режиме toggle всегда переключается независимо от других кнопок рядом — своё Selected не влияет на соседние IconButton, даже если они выглядят как ряд (например, панель инструментов). Если нужна взаимоисключающая группа — используйте ButtonGroup Toggle или SegmentControl.

Loading блокирует повторный клик

Пока идёт процесс, кнопка недоступна для повторного нажатия (pointer-events: none) — это предотвращает повторный запуск того же действия до получения ответа. Иконка временно скрыта под спиннером и возвращается по завершении.

Состояния

Интерактивные состояния по горизонтали, размеры по вертикали. К исходной матрице Default / Hover / Disabled добавлены Pressed, Focus и Loading.

Доступность

У иконки нет видимого текста, поэтому каждому IconButton обязателен aria-label, описывающий действие, и, как правило, Tooltip по наведению. Управление с клавиатуры: Tab ставит фокус, Space / Enter активируют, фокус показывается обводкой --primary.

Минимальная рекомендованная область нажатия — 44×44px. Размеры S (24) и M (32) меньше неё, поэтому в тач-сценариях либо берите размер L, либо расширяйте кликабельную зону невидимым отступом, оставляя видимый стейт-слой прежнего размера.

Не так

Иконка без подписи и без aria-label / тултипа — пользователь не понимает действие.

Так

aria-label + тултип раскрывают значение; область нажатия не меньше 44px.

Типографика

Не применимо: компонент не содержит текста.

Цвета

Компонент использует только семантические токены. Стейт-слой — это color-mix цвета иконки (currentColor) с прозрачным: 12% на Hover, 22% на Pressed.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая верстает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

Точные размеры (redline)

Значения измерены на живом экземпляре компонента (getComputedStyle), источник — токены в styles/icon-button.css.

Поведение · псевдокод (framework-agnostic)

Поведение · псевдокод (framework-agnostic) 
// 1. Стейт-слой — чистый CSS, отдельного DOM-элемента нет:
//    background = color-mix(currentColor 12%, transparent) на :hover
//    background = color-mix(currentColor 22%, transparent) на :active
//    цвет рипла всегда следует за цветом иконки (тоном)

// 2. Toggle (режим-переключатель):
//    click → selected = !selected
//    aria-pressed = selected; тон neutral → primary;
//    контурная иконка заменяется на залитую; aria-label отражает состояние
// React: состав рендерится декларативно:
glyph = selected ? iconSelected : icon

// 3. Loading: иконка заменяется на .ibtn__spinner (размер = размеру иконки);
//    .ibtn--loading даёт pointer-events: none; добавить aria-busy="true"

// 4. Embedded (поля ввода): стейт-слоя нет, hover меняет цвет иконки
//    text-inactive → primary (на :active → primary-dark)

// 5. Тач-сценарии: при размерах S/M расширять зону клика до 44×44
//    невидимым псевдоэлементом, не увеличивая видимый стейт-слой

// 6. Состояния hover/pressed — чистый CSS (:hover/:active);
//    классы .is-hover/.is-pressed/.is-focus — только для спецификаций в документации

Это не готовый сниппет для вставки — алгоритм, который нужно реализовать поверх вашего стека (рабочий референс на vanilla JS — функция makeIB в коде этой страницы).
