Avatar

Аватар — компактное визуальное представление пользователя или сущности. Содержит инициалы (Text), запасную иконку (Icon) или фотографию (Image). Две формы — Circular и Rounded — четыре размера и опциональный бейдж: счётчик или индикатор статуса.

Конструктор

Соберите аватар: форма, размер, тип содержимого и бейдж. Для типа «Изображение» перетащите фото в область превью.

Использование

Может использоваться как самостоятельный компонент, так и в составе другого — например, в сущности (Entity): строка пользователя, автор комментария, участник в ячейке таблицы. Группа аватаров показывает несколько участников компактным стеком.

Анатомия

Аватар — это фигура-контейнер фиксированного размера с содержимым по центру и опциональным бейджем поверх. В группе у каждого аватара появляется белое кольцо-разделитель.

  • Контейнерквадрат фиксированного размера; форма Circular (50%) или Rounded
  • Содержимоеинициалы, иконка-фолбэк или изображение, центрированы
  • Бейджопционально — счётчик (сверху-справа) или статус-точка (снизу-справа)

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Типы содержимого

Тип определяется доступными данными. Приоритет — изображение; если его нет, показываем инициалы; если нет и имени — нейтральную иконку.

Text
Инициалы — одна или две заглавные буквы имени/названия. Базовый вариант для пользователей.
Icon
Нейтральный глиф — когда нет ни фото, ни имени. Для сущностей используется тематическая иконка.
Image
Фотография, обрезанная по форме (object-fit: cover). Перетащите изображение в область выше.

Форма

Circular — для людей (инициалы, фото). Rounded (скруглённый квадрат) — для сущностей, организаций, ботов. Радиус скругления у Rounded пропорционален размеру: 28 / 12 / 10 / 8 px для XL / L / M / S.

Бейдж на аватаре

Аватар может нести дочерний Badge. Counter ставится в правый верхний угол — непрочитанные сообщения, количество задач. Dot ставится в правый нижний угол как индикатор присутствия. На фото у бейджа всегда белая обводка (badge--bordered).

Группа аватаров

Несколько участников показываются перекрывающимся стеком: левый аватар лежит поверх следующего, между ними — белое кольцо. Если участников больше лимита, остаток сворачивается в нейтральный чип «+N». Рекомендуемый лимит — 3–5 видимых аватаров.

Не так

Длинный нестриженный ряд — занимает место и не читается.

Так

3–4 аватара и «+N» — компактно и считываемо.

Размеры

Четыре размера. Выбирайте по контексту: XL — карточка профиля, L — строки списков и шапки, M — таблицы и навигация, S — плотные перечисления и инлайн-упоминания.

Контент

Содержимое аватара определяется доступными данными, а не выбирается произвольно — см. цепочку деградации в разделе «Заглушки и фолбэк» выше. Правила текста:

  • Инициалы — 1–2 заглавные буквы. Для человека — первая буква имени и первая буква фамилии («Игорь Белов» → «ИБ»); для однословного имени или организации — одна буква;
  • инициалы вычисляются автоматически из имени — не задавайте их произвольным текстом и не используйте цифры, эмодзи или более двух знаков;
  • Иконка (Icon) — нейтральный глиф user для людей без имени; тематический глиф (например, entity) — для организаций и ботов;
  • Группа «+N» — только целое число оставшихся участников, без знака «людей» или подписи;
  • полное имя выносите рядом текстом (см. Entity) — сам аватар не заменяет подпись, а дополняет её.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Заглушки и фолбэк

Содержимое выбирается по цепочке деградации: если есть фотография — показываем её; если фото нет или не загрузилось — инициалы; если нет и имени — нейтральную иконку. Это гарантирует, что аватар никогда не остаётся пустым.

Состояния

Аватар может быть интерактивным — вести на профиль. В этом случае у него есть состояния Hover, Focus (обводка --primary) и Pressed. Неактивный аватар приглушается.

Доступность

Сам по себе аватар — визуальный, декоративный элемент. Доступность определяется контекстом использования.

  • Интерактивный аватар (av--button, ведёт на профиль) — фокусируемый элемент с aria-label, называющим человека или сущность («Открыть профиль Игоря Белова»);
  • Аватар рядом с видимым именем (Entity, комментарий, ячейка таблицы) — сам аватар помечается aria-hidden="true", чтобы скрин-ридер не озвучивал инициалы отдельно от имени;
  • Изображение — если фото несёт смысл сверх имени, добавьте содержательный alt; в остальных случаях alt="", так как имя уже есть рядом текстом;
  • Индикатор присутствия (Dot) — сам по себе не имеет текстовой альтернативы; статус («В сети», «Не беспокоить») дублируйте в aria-label родителя;
  • Группа участников — контейнеру .av-group присваивается role="group" и aria-label со списком имён или их количеством; отдельные аватары внутри — aria-hidden;
  • :focus-visible у интерактивного аватара — обводка 2px --primary, отступ 2px.

Типографика

Инициалы набираются гарнитурой SB Sans Text начертанием Semibold. Кегль пропорционален размеру аватара (около ⅓ стороны). Используются 1–2 заглавные буквы.

Цвета

Аватар по умолчанию использует нейтральную «болотную» подложку. Для сущностей доступен акцентный тон. Индикаторы статуса наследуют семантические токены из Badge.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.
