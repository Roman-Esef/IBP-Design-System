Checkbox

Чекбокс используется, когда доступен список опций и пользователю нужно выбрать одну или несколько из них. Три состояния выбора — Unselected / Selected / Indeterminate — и интерактивные состояния Default / Hover / Focus / Pressed / Disabled / Error.

Конструктор

Соберите чекбокс: переключайте состояние выбора, интерактивное состояние и состав — превью обновляется мгновенно.

Использование

Используйте чекбокс для обозначения и управления состоянием или свойством параметра, которое требует дополнительного подтверждения (например, кнопкой «Отправить» в формах).

Используйте группу чекбоксов, если нужно дать пользователю возможность выбрать один или несколько пунктов из короткого списка.

Если действие необходимо выполнить сразу, предпочтительнее использовать Switch.

Если пунктов слишком много (от 5 до 15) или они не помещаются в ограниченном пространстве — используйте InputAutocomplete. Исключением являются группы в модальном окне.

Если нужно дать пользователю возможность выбрать только один вариант из нескольких — используйте Radiobutton.

Анатомия

Чекбокс состоит из обязательного бокса-метки и опциональных текстовых частей; в группе сверху может быть заголовок.

  • Заголовок группы — опционально, H6 Strong
  • Чекбокс — бокс-метка 16px в зоне выравнивания 20px
  • Название чекбокса — Body M, справа от бокса
  • Хелпер — опционально, Body XS, общий компонент подкачки — см. Label / Helper

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Состав

Чекбокс рендерится с названием, без названия (icon-only — например, в шапке таблицы) и с хелпером. Бокс всегда выровнен по первой строке текста.

Размеры

У чекбокса один размер. Геометрия бокса привязана к интерлиньяжу Body M (20px): видимый бокс — 16px в зоне выравнивания 20px, чтобы бокс центрировался по первой строке названия. Значения ниже измерены на живом экземпляре.

  • Зона выравнивания 20px совпадает с интерлиньяжем Body M — бокс ровно ложится на строку.
  • Видимый бокс 16px с рамкой 2px и радиусом --radius-2xs.
  • Галочка и «минус» (indeterminate) — 14px, цвет --text-on-dark.
  • Зазор бокс ↔ название — 8px; название ↔ хелпер — 2px.

Контент

Текстовый контент чекбокса — название, опциональный хелпер и заголовок группы. Краткий свод правил (подробно — в разделах ниже):

  • Название (Body M, справа от бокса) — с заглавной буквы, без пунктуации в конце, в положительном ключе («Получать…», не «Не получать…»);
  • Хелпер (Body XS под названием) — поясняет неочевидный результат включения; общий компонент — см. Label / Helper;
  • Заголовок группы (H6 Strong) — добавляется для ясности или чтобы вынести повторяющиеся в пунктах слова;
  • Обязательность и ошибка — у обязательного названия ставится звёздочка *; текст ошибки — Body XS цвета --error;
  • не обрезайте название многоточием — при нехватке места переформулируйте его короче.

Название чекбокса

Название пишется с заглавной буквы, без знаков пунктуации в конце, и всегда располагается справа от чекбокса.

Формулируйте название в положительном ключе: отрицательная формулировка заставляет пользователя выполнять в голове логическую операцию с двумя «не».

Не так

Отрицательная формулировка — «Не получать…» — усложняет восприятие.

Так

Положительная формулировка читается однозначно.

Если пользователю может быть не очевидно, что произойдёт при включении чекбокса, объясните это дополнительным текстом-хелпером. Старайтесь делать названия короткими; при нехватке места меняйте формулировку, но не обрезайте название многоточием.

Заголовок группы

Добавляйте заголовок к группе для ясности, если взаимосвязь пунктов неочевидна, или чтобы вынести в него повторяющиеся в пунктах слова. Заголовок пишется с заглавной буквы, начертанием Strong того же кегля, что и название.

Не так

«Статус сделки» повторяется в каждом пункте.

Так

Общее слово вынесено в заголовок группы.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Описание работы

Клик по названию или по самому чекбоксу приводит к его выбору или снятию выбора.

Чекбоксы могут объединяться в группы:

  • заголовок группы для дополнительной ясности, если необходимо;
  • один или несколько чекбоксов с названием (название всегда справа от чекбокса);
  • выравнивание группы чекбоксов всегда по левому краю.

Группа чекбоксов может зависеть от родительского чекбокса. Когда родительский чекбокс выбран, все дочерние так же выбраны. Когда с родительского снят выбор — все дочерние сняты. Если в дочерних выбрана только часть пунктов, родительский чекбокс находится в состоянии Indeterminate — частично выбран. Поиграйте с примером:

Ошибка и валидация

Состояние Error появляется, когда обязательный чекбокс не выбран при отправке формы, или когда группа не прошла валидацию. Бокс и хелпер окрашиваются в --error; рядом с обязательным названием ставится звёздочка.

Состояния

Три состояния выбора по вертикали и интерактивные состояния по горизонтали. Состояния Hover, Focus, Pressed и Error добавлены к исходной матрице Default / Disabled

Доступность

Видимый бокс — 16px, но кликабельная область шире: она включает название и хелпер. В группе активная область чекбокса вытягивается на всю высоту строки, чтобы её было легче нажать. Управление возможно с клавиатуры: фокус показывается обводкой --primary, переключение — пробелом.

Типографика

Название набирается гарнитурой SB Sans Text токеном --type-body-m, хелпер — --type-body-xs, заголовок группы — --type-h6-strong.

Цвета

Компонент использует только семантические токены. Производные оттенки (hover) и state-слои вычисляются из --primary через color-mix.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

Поведение · псевдокод (framework-agnostic) 
// 1. Состояние выбора — три значения, отражаются на нативном input:
//    unselected → checked=false, indeterminate=false
//    selected   → checked=true
//    indeterminate → input.indeterminate=true (визуально «минус»)
//    indeterminate — только промежуточное состояние, не самостоятельное значение

// 2. Клик по названию ИЛИ боксу переключает выбор (<label> оборачивает input)

// 3. Родитель ↔ дети (select all):
//    parent selected → все дети selected; parent unselected → все дети unselected
//    часть детей выбрана → parent = indeterminate
//    после клика по ребёнку пересчитать состояние родителя

// 4. Интерактивные состояния — чистый CSS (:hover / :active / :focus-visible);
//    hover: рамка primary + halo 4px; pressed: halo 6px
//    классы .cb--hover / .cb--focus / .cb--pressed — только для документации

// 5. Клавиатура: Tab фокусирует input, Space переключает;
//    :focus-visible → outline 2px --primary, offset 2px

// 6. Error: класс .cb--error (required не выбран при сабмите или группа
//    не прошла валидацию). Добавить aria-invalid="true";
//    сообщение об ошибке группы — .cb-group__error (Body XS, --error)

// 7. Disabled: input.disabled + .cb--disabled (pointer-events: none)

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функции makeCb / setCbState в коде этой страницы).

Бэклог

Кейсы, которых недостаёт документации, — на существующих токенах и компонентах.

  • Валидация всей группы — сообщение об ошибке на уровне .cb-group, а не только отдельного пункта.
  • Чекбокс-карточка — крупный кликабельный вариант с описанием и иконкой (как RadioCard), не только строка с меткой.
