Radiobutton

Радиокнопка позволяет выбрать всего один пункт из списка взаимоисключающих вариантов. Используется только в составе группы. Два состояния выбора — Unselected / Selected — и интерактивные состояния Default / Hover / Focus / Pressed / Disabled / Error.

Конструктор

Соберите радиокнопку: переключайте состояние выбора, интерактивное состояние и состав — превью обновляется мгновенно.

Использование

Используется только в составе группы радиокнопок и не может использоваться как единственная доступная опция — в этом случае используйте Чекбокс.

Группу радиокнопок используют, когда вариантов выбора немного — 2–5. Может использоваться в тайлах, таблицах, модальных окнах, в формах и на стандартных страницах.

Используется для переключения с одной настройки на другую. Часто может выступать в качестве механизма фильтрации.

Если пунктов слишком много (от 5 до 15) или они не помещаются в ограниченном пространстве — используйте Выпадающий список.

Если нужно дать возможность выбрать один или несколько пунктов — используйте Чекбокс.

Анатомия

У иконки радиокнопки есть охранное поле: компонент иконки имеет размер 20×20 px, а сам круглый шейп внутри него — 16×16 px. В группе сверху может быть заголовок.

  • Заголовок группы — опционально, H6 Strong
  • Радиокнопка — круглый шейп 16px в охранном поле 20px
  • Название радиокнопки — Body M, справа от шейпа
  • Хелпер — опционально, Body XS, общий компонент подкачки — см. Label / Helper

Группы радиокнопок могут быть вертикальными или горизонтальными. По умолчанию группы вертикальные. Используйте горизонтальную группу, когда пространство по вертикали ограничено.

Состав

Вспомогательные части — название и хелпер — опциональны. Радиокнопка рендерится без названия (icon-only — например, в ячейке таблицы), с названием и с хелпером. Шейп всегда выровнен по первой строке текста.

Размеры

У радиокнопки один размер. Геометрия шейпа привязана к интерлиньяжу Body M (20px): видимый шейп — 16px в охранном поле 20px, чтобы центрироваться по первой строке названия. Значения ниже измерены на живом экземпляре.

  • Охранное поле 20px совпадает с интерлиньяжем Body M — шейп ровно ложится на строку.
  • Видимый круг 16px с рамкой 2px; внутренняя точка (Selected) — 8px.
  • Зазор шейп ↔ название — 8px; название ↔ хелпер — 2px.
  • Горизонтальная группа: зазор между пунктами — 24px.

Контент

Текстовый контент радиокнопки — название, опциональный хелпер и заголовок группы. Краткий свод правил (подробно — в разделах ниже):

  • Название (Body M, справа от шейпа) — с заглавной буквы, без пунктуации в конце; чётко и кратко описывает эффект выбора;
  • Хелпер (Body XS под названием) — поясняет неочевидный эффект выбора; общий компонент — см. Label / Helper;
  • Заголовок группы (H6 Strong) — добавляется для ясности или чтобы вынести повторяющиеся в пунктах слова; формулируйте так, чтобы было понятно: выбрать можно только один вариант;
  • Обязательность и ошибка — у обязательного заголовка группы ставится звёздочка *; текст ошибки группы — Body XS цвета --error;
  • не давайте пересекающиеся варианты и не обрезайте название многоточием — при нехватке места переформулируйте его короче.

Название радиокнопки

Название пишется с заглавной буквы, без знаков пунктуации в конце, и всегда располагается справа от радиокнопки. Название каждой радиокнопки должно чётко и кратко описывать эффект от её выбора — короткие названия позволяют делать выбор быстро и уверенно.

Убедитесь, что варианты не пересекаются. Например, при выборе суммы «0–100», «100–1000» неясно, что выбрать, если сумма равна 100.

Не так

Длинные названия и пересекающиеся диапазоны замедляют выбор.

Так

Короткие непересекающиеся названия читаются однозначно.

Если пользователю может быть не очевидно, что произойдёт при выборе, объясните это дополнительным текстом-хелпером.

Заголовок группы

Добавляйте заголовок к группе для ясности, если взаимосвязь пунктов неочевидна, или чтобы вынести в него повторяющиеся в пунктах слова. Заголовок начинается с заглавной, начертанием Strong того же кегля, что и название. Допустимо использование H6 Strong, если группа входит в состав другой группы. Формулируйте заголовок так, чтобы было понятно: можно выбрать только один вариант.

Не так

«Статус сделки» повторяется в каждом пункте.

Так

Общее слово вынесено в заголовок группы.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Описание работы

Клик по названию или по самой радиокнопке приводит к её выбору. Повторное нажатие не снимает выбор. Когда пользователь выбирает новый элемент, предыдущий выбор автоматически отменяется.

Радиокнопки всегда образуют группы:

  • заголовок группы для дополнительной ясности, если необходимо;
  • несколько радиокнопок с текстом подписи (подпись всегда справа от радиокнопки);
  • выравнивание группы всегда по левому краю.

В группе радиокнопок только один пункт может быть выбран. Группа не может состоять из одного пункта. В группе активная область радиокнопки вытягивается на всю высоту строки. Поиграйте с примером:

Если группа используется для настройки параметров и показывает текущее состояние системы, одна из радиокнопок всегда выбрана — так видно, в каком состоянии система находится по умолчанию. Выбранный по умолчанию пункт ставьте первым в списке.

Сортируйте пункты списка по определённой логике:

  • от наиболее подходящего в данном контексте до наименее подходящего;
  • по возрастанию или убыванию;
  • хронологически;
  • следуя другой понятной системе.

Ошибка и валидация

Состояние Error появляется, когда обязательная группа не имеет выбранного пункта при отправке формы. Поскольку у радиокнопки нет промежуточного состояния, ошибка всегда относится к группе целиком: шейпы и сообщение окрашиваются в --error, рядом с заголовком обязательной группы ставится звёздочка.

Состояния

Два состояния выбора по вертикали и интерактивные состояния по горизонтали. Состояния Hover, Focus, Pressed и Error добавлены к исходной матрице Default / Disabled.

Доступность

Видимый шейп — 16px, но кликабельная область шире: она включает название и хелпер. В группе активная область вытягивается на всю высоту строки. Клавиатура работает по правилам radio group: Tab переводит фокус на группу, стрелками ↑↓ ←→ выбор перемещается между пунктами (и сразу применяется), фокус показывается обводкой --primary.

Типографика

Название набирается гарнитурой SB Sans Text токеном --type-body-m, хелпер — --type-body-xs, заголовок группы — --type-h6-strong.

Цвета

Компонент использует только семантические токены. Производные оттенки (hover) и state-слои вычисляются из --primary через color-mix.

Поведение · псевдокод (framework-agnostic) 
// 1. Состояние выбора — бинарное на каждом input:
//    unselected → checked=false; selected → checked=true
//    у радиокнопки нет indeterminate — всегда ровно один выбранный пункт в группе

// 2. Взаимоисключающий выбор: все input в группе делят один name —
//    браузер снимает выбор с остальных автоматически; повторный клик по выбранному не снимает выбор

// 3. Клик по названию ИЛИ шейпу переключает выбор (<label> оборачивает input)

// 4. Клавиатурная навигация (native radiogroup semantics):
//    Tab входит/выходит из группы одним шагом — фокусируется только checked-пункт (или первый, если ничего не выбрано);
//    ↑↓←→ перемещают фокус МЕЖДУ пунктами группы И сразу применяет выбор (без Enter/Space)

// 5. Интерактивные состояния — чистый CSS (:hover / :active / :focus-visible);
//    hover: рамка primary + halo 4px; pressed: halo 6px
//    классы .rb--hover / .rb--focus / .rb--pressed — только для документации

// 6. Error: обязательная группа без выбора при сабмите → класс .rb--error на всех пунктах группы;
//    сообщение .rb-group__error (Body XS, --error), связано через aria-describedby;
//    выбор любого пункта снимает ошибку со всей группы

// 7. Disabled: input.disabled + .rb--disabled (pointer-events: none) — может быть у отдельных
//    пунктов внутри доступной группы

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функции makeRb / wireGroup в коде этой страницы).
