Link

Текстовая ссылка для навигации внутри лайаута и модальных окон, а также в Хлебных крошках. Тоны Accent / Muted / Neutral, размеры M / S, состояния Default и Hover = Focus (подчёркивание) — кроме тона Neutral (Breadcrumbs_2): у него только состояние Default, ссылка не кликабельна. Цветовое решение едино для всех видов и размеров.

Конструктор

Соберите ссылку: переключайте тон, размер, состояние и модификаторы — превью обновляется мгновенно.

Использование

Используйте ссылку, когда нужно:

  • перейти на другую страницу или экран — в лайауте, шапке, Хлебных крошках;
  • открыть документ, профиль или внешний ресурс прямо из текста.

Ссылка ведёт к навигации, а Button запускает действие (отправить, сохранить, удалить). Не оформляйте ссылку как кнопку и наоборот — например, действие «Отмена» в диалоге должно быть прозрачной кнопкой (Transparent Button), а не ссылкой.

Анатомия

Ссылка — это текстовая метка, окрашенная в цвет тона. Подчёркивание работает как маркер интерактивности: у standalone-ссылки оно появляется на Hover / Focus, у инлайн-ссылки включено всегда. Опциональная иконка масштабируется относительно текста (1em). При навигации с клавиатуры добавляется обводка фокуса.

  • Текстовая метка — цвет по тону; размер наследуется от токена (M / S) или окружающего текста
  • Подчёркивание — на Hover / Focus у standalone; у инлайн-ссылки включено всегда
  • Иконка (опц.) — 1em, слева (скачивание) или справа (внешняя ссылка)
  • Focus = Hover — клавиатурный фокус оформлен точно так же, как Hover — без отдельной обводки

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Ссылки лайаута и модалок

Основной тип ссылок (тон Accent). Цветовое решение едино для всех видов и размеров. В покое — без подчёркивания, на Hover и Focus появляется подчёркивание.

Тона состояний

Семантические тона Info · Warning · Error · Success — для ссылок внутри цветных Alert и SnackBar: цвет совпадает с тоном компонента. Hover — на ступень темнее (--<тон>-dark), Pressed — ещё темнее. Классы .link--info / .link--warning / .link--error / .link--success.

Ссылки Хлебных крошек

Ссылки используются в Хлебных крошках (шапка страницы). Есть два цветовых решения: Breadcrumbs_1 (тон Muted — родительские страницы) — кликабельна, подчёркивается на Hover и Focus; и Breadcrumbs_2 (тон Neutral — выделенная текущая страница) — не кликабельна, доступно только состояние Default, без подчёркивания. Полная спецификация компонента (анатомия, авто-схлопывание, доступность) — на странице Breadcrumbs.

Готовая цепочка крошек с разделителем «/». Текущая страница — не ссылка (aria-current="page"), родительские — ссылки тона Muted.

При длинной цепочке средние крошки сворачиваются в кнопку «…». Клик по ней открывает выпадающий список с названиями скрытых страниц — сами крошки не разворачиваются.

Ссылка внутри текста

Инлайн-ссылка живёт в потоке текста и наследует его размер. Чтобы её можно было отличить не только по цвету, инлайн-ссылка всегда подчёркнута (мягким подчёркиванием), а на Hover подчёркивание становится насыщенным. Это требование доступности (WCAG 1.4.1).

С иконкой и внешние ссылки

Иконка может стоять слева (например, ссылка-скачивание) или справа. Внешняя ссылка ведёт на другой ресурс — она открывается в новой вкладке (target="_blank", rel="noopener") и помечается иконкой со стрелкой. Иконка масштабируется относительно текста (1em).

Размеры

Два размера, привязанных к токенам текста: M — --type-body-m (16/20), S — --type-body-s (14/16). Инлайн-ссылка наследует размер окружающего текста. Размер S также используется в таблицах — весь текстовый контент внутри таблицы набирается размером S.

Контент

Контент ссылки — её текст (и опциональная иконка). Текст должен считываться вне контекста:

  • называйте цель перехода, а не механику: «Открыть договор», а не «нажмите здесь» или «подробнее»;
  • держите ссылку короткой (1–3 слова, где возможно); длинные названия переносятся или обрезаются многоточием (см. «Перенос и обрезка»);
  • у внешней ссылки поясните переход на другой ресурс — иконка-стрелка и aria-label «(открывается в новой вкладке)»;
  • регистр — как в окружающем интерфейсе (sentence case); точку в конце standalone-ссылки не ставим.

Текст ссылки не называет цель — вне предложения и в списке ссылок он бесполезен.

Так
Выгрузите отчёт за квартал.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Перенос и обрезка

Длинная ссылка по умолчанию переносится по словам вместе с текстом. В узких контейнерах (ячейка таблицы, чип) ссылку можно обрезать многоточием — полное значение остаётся в title / тултипе.

Состояния

Hover и Focus оформлены одинаково (подчёркивание) — как в спецификации. Дополнительно добавлены Pressed и Disabled, а на клавиатурный фокус показывается обводка для доступности. Тон Neutral (Breadcrumbs_2) не кликабелен — для него доступно только состояние Default.

Доступность

Не полагайтесь только на цвет: смысл ссылки должен считываться и без него. В потоке текста ссылка всегда подчёркнута; standalone-ссылки подчёркиваются на Hover / Focus. Клавиатурный фокус оформлен точно так же, как Hover, без отдельной обводки — единое оформление для мыши и клавиатуры. Для внешних ссылок добавляйте подпись «открывается в новой вкладке» в aria-label.

Не так

Инлайн-ссылка различима только по цвету — её не видно при дальтонизме и в ч/б.

Так

Подчёркивание делает ссылку различимой независимо от цвета.

Типографика

Ссылка набирается гарнитурой SB Sans Text. Размер задаётся токеном текста; начертание — Regular.

Цвета

Компонент использует только семантические токены. Hover берёт затемнённый оттенок того же цвета.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

Точные размеры (redline)

Значения измерены на живом экземпляре компонента (getComputedStyle), источник — токены в styles/link.css и токены текста в styles/typography.css.

Поведение · псевдокод (framework-agnostic)

Поведение · псевдокод (framework-agnostic) 
// 1. Подчёркивание — маркер интерактивности (отдельного DOM-слоя нет):
//    standalone (link--m/s): в покое без подчёркивания;
//    :hover / :focus-visible / .is-hover / .is-focus → text-decoration: underline
//    inline (link--inline): подчёркнута ВСЕГДА — 45% currentColor,
//    на hover/focus подчёркивание → 100% currentColor

// 2. Цвет задаёт тон:
//    accent  → --link, hover/focus --link-dark, pressed --c-emerald-900
//    muted   → --text-inactive, hover --text-secondary (Breadcrumbs_1)
//    neutral → --text-primary (Breadcrumbs_2); только Default, не кликабельна (pointer-events: none)

// 3. Фокус = Hover: :focus-visible применяет ТЕ ЖЕ правила, что и :hover
//    (цвет + подчёркивание), без отдельной обводки — оформление одинаковое для мыши и клавиатуры

// 4. Disabled: aria-disabled="true" (или .link--disabled) →
//    цвет --text-inactive, pointer-events: none, подчёркивание снято

// 5. Внешняя ссылка: target="_blank" + rel="noopener";
//    aria-label дополняется «(открывается в новой вкладке)»; справа иконка-стрелка 1em

// 6. Обрезка: .link--truncate = inline-block + ellipsis; полное значение — в title/тултипе

// 7. Хлебные крошки: текущая страница — НЕ ссылка (span[aria-current=page]);
//    средние звенья схлопываются в кнопку «…», раскрывающую цепочку.
//    Форс-состояния .is-hover/.is-pressed/.is-focus — только для документации.
//    Полная спецификация компонента Хлебные крошки (авто-схлопывание по ширине,
//    обрезка текущей крошки, доступность) — см. Breadcrumbs.html

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функция makeLink в коде этой страницы).
