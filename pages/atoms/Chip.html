Chip

Чип — компактный интерактивный элемент для фильтрации и группировки данных. Применяется как тег, маркер и статус, входит в состав чиплиста и InputAutocomplete. Два типа — Edit и ReadOnly, два стиля — Fill и Outline.

Конструктор

Соберите чип: тип, стиль, размер, тон, дополнительный элемент, удаление и скругление — превью и код обновляются мгновенно.

Использование

Чипы используются в составе чиплиста для отображения параметров фильтра, в InputAutocomplete при множественном выборе, а также как статус-метки в таблицах и карточках.

  • Фильтры. Чиплист отображает применённые параметры фильтрации; удаляемые чипы (с крестиком) сбрасывают параметр.
  • Множественный выбор. В InputAutocomplete выбранные значения показываются чипами внутри поля.
  • Статусы и теги. Цветной тон передаёт состояние сущности (активна, в работе, просрочена).
  • Удалённый чип повторно не отображается в чиплисте.

Чиплист фильтра

Удаляемые чипы сбрасывают параметр фильтра. Порядок удаления зависит от выравнивания чиплиста.

ReadOnlyField · значение-чипы

Регионы присутствия

Тип ReadOnly, размер XS — так чип используется в ReadOnlyField (значение типа «Чипы»); как и любое значение в ReadOnlyField, оно всегда идёт с лейблом; без рамки и фона у самого поля, без крестика.

Правила

Чип несёт короткую метку или статус. Не превращайте его в кнопку и не помещайте длинные предложения — для этого есть другие компоненты.

Так

Статус или стадия сущности — скруглённый чип (Rounded, 999px). Так — в 90% случаев.

Не так

Квадратный чип для статуса. Квадратные цветные чипы оставляйте для категорий сущности (типы продуктов, типы активов и т.п.), не для статусов.

Так

Короткая метка; длинный текст обрезается многоточием с тултипом.

Не так

Целое предложение растягивает чип — это не его задача.

Так

Тон передаёт статус; крестик — только когда чип удаляемый.

Не так

Не используйте чип как основную кнопку действия.

Анатомия

Чип — это капсула-контейнер с текстом и опциональными слотами. Ширина зависит от контента, высота фиксирована по размеру. Максимальная ширина — 320px; при переполнении текст уходит в многоточие.

  • Контейнеркапсула с заливкой и/или обводкой; высота фиксирована, ширина тянется по контенту до 320px
  • Дополнительный элементиконка-маркер, иконка, бейдж или аватар — опционально
  • ТекстSB Sans Text; обрезается многоточием при переполнении
  • Действиекрестик удаления или шеврон выпадающего списка — опционально

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Типы

Компонент имеет два типа. Edit — интерактивный чип, реагирует на наведение и поддерживает удаление. ReadOnly — для показа значения без редактирования (например, в ReadOnlyField) и на наведение не реагирует.

Edit

Интерактивный. Меняет фон при наведении, поддерживает крестик удаления. Размеры M / S (+ XS для компактных мест).

ReadOnly

Показ значения без редактирования. Не реагирует на наведение. Размеры L / M / S / XS — ReadOnlyField использует XS для значений-чипов.

Стиль

Fill (Border + Fill) — заливка плюс обводка, базовый стиль для чиплистов и инпутов. Outline (Border) — только обводка, для более лёгкого вида на плотных поверхностях; при наведении появляется заливка.

Ведущие элементы

Слева к тексту можно добавить дополнительный элемент. .Chip_Content — иконка-маркер; Иконка — смысловая пиктограмма, не завязанная на тон/статус; Аватар — компонент Avatar с выбираемым содержимым (текст-инициалы, иконка или фото). Отдельный слот «бейдж» убран — он объединён с аватаром. Справа — действие: крестик удаления или шеврон выпадающего списка (.Chip_Action) — взаимоисключающие, вместе не ставятся.

Тон и статусы

Текущая ДС описывает только нейтральный системный тон (StSystem). Поскольку чипы используются и как статусы, предлагаем тональную шкалу на базе локальных статус-рамп: фон Light, текст Dark, иконка — базовый цвет статуса. Это даёт семантику «активно / в работе / ошибка» без отдельной иконки.

Скругление (Rounded)

Новые чипы объединили с маркером и добавили свойство Rounded для включения полного скругления (капсула 999px). В дизайн-системе пока не перешли на них — будем переводить после внутренних тестов нового компонента.

Правило применения. В 90% случаев для статусов и стадий (частный вид статуса) используются преимущественно скруглённые чипы (Rounded, 999px). Квадратные цветные чипы (скругление 6–8px) допустимы для отображения категорий сущности — например, разных продуктов, разных типов активов и т.п., — но не для статусов. Исключения возможны и фиксируются дизайнером индивидуально для конкретного кейса.

Default · скругление 6–8px — для категорий сущности

Размеры

Четыре размера. L — крупный ReadOnly-показ (зарезервирован). M и S — базовые для чиплистов и фильтров. XS — для компактных мест (плотные таблицы, инлайн-метки, значения-чипы в ReadOnlyField).

Контент

Текстовое содержимое чипа — метка, опциональный числовой суффикс и текст в альтернативах (title/aria-label). Краткий свод правил (подробно — в «Переполнение текста» и «Правила» выше/ниже):

  • Метка (.chip__label) — короткая номинативная фраза или существительное, не предложение и не глагол-действие — для действий есть Button;
  • регистр — как в остальном интерфейсе (sentence case); точку в конце не ставим;
  • максимум ширины — 320px; при превышении текст обрезается многоточием, полное значение — в title;
  • Счётчик (.chip__count) — только число, без единиц измерения и без текста;
  • крестик удаления не несёт текста визуально — сопровождайте aria-label с именем чипа (см. «Доступность»);
  • для чипов-статусов используйте краткие существительные состояния («Активна», «Просрочена»), не глаголы.

Переполнение текста

Размер чипа зависит от контента, максимальная ширина — 320px. Если длина контента превышает максимум, текст уходит в многоточие, а при наведении показываем тултип с полной версией. Это работает и для Disabled-чипа с длинной текстовкой.

Переполнение чиплиста — свёртка в «+N»

Когда чипов больше, чем помещается, лишние сворачиваются в чип-счётчик «+N», открывающий полный список.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Удаление

Если нужно дать пользователю управлять набором чипов, добавляется крестик. Крестик — единственная интерактивная зона помимо самого чипа; на наведении у него появляется собственный стейт-слой. Удаление доступно мышью и с клавиатуры (Backspace / Delete на сфокусированном чипе).

Состояния

Базовые состояния чипа и их цветовые токены. При наведении меняется цвет фона. Чип в состоянии Disabled на наведение не реагирует — кроме случаев с длинными текстовками, когда показываем тултип.

Доступность

Чип Edit — фокусируемый элемент (tabindex="0"); крестик удаления — вторая интерактивная зона внутри него. ReadOnly-чип неинтерактивен и из таб-порядка исключён.

  • Фокус на самом чипе удаляет его при Backspace / Delete (см. «Удаление» выше) — мышь и клавиатура дают одинаковый результат;
  • крестик (.chip__remove) помечайте role="button" и aria-label="Удалить {метка}" — без этого скрин-ридер озвучит только иконку без контекста;
  • :focus-visible показывает обводку 2px --primary с отступом 2px (состояние Focus — Предложение). Нажмите Tab, чтобы увидеть:
  • Disabled-чип исключён из таб-порядка (pointer-events:none), но если текст обрезан — класс .chip--has-tooltip возвращает наведение для тултипа; полный текст дублируйте в title;
  • не полагайтесь только на тон (цвет) для критичных статусов (например, Error против Success) — для пользователей с нарушениями цветовосприятия дублируйте смысл маркером/иконкой;
  • для чипов-фильтров с выбором (chip--selected, Предложение) рекомендуем aria-pressed на каждом чипе и role="group" на чиплисте.

Типографика

Текст чипа набирается гарнитурой SB Sans Text. Кегль зависит от размера и совпадает с токенами Body. Letter-spacing — 0.

Цвета

Компонент использует только семантические и локальные токены статусов. Базовый системный тон — на рампе StSystem; статусные тона — на локальных рампах.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.
