Badge

Бейдж — небольшой дочерний элемент, который накладывается на родительский компонент или ставится рядом с ним. Тип Counter показывает количество (счётчик), тип Dot — статус без числа. Пять размеров — M / S / XS / XXS / Dot — и семантические тона.

Конструктор

Соберите бейдж: переключайте тип, тон, размер и значение — превью и наложение на родителя обновляются мгновенно.

Использование

Встраивается в компонент в качестве дочернего:

  • применяется как счётчик (counter) для отображения количественных значений;
  • или в качестве уведомления о состоянии материнского компонента (dot).

Правила

Бейдж дополняет родителя, а не заменяет его. Не помещайте в счётчик длинные строки и не используйте его как самостоятельную метку-«пилюлю».

Не так

Текст в счётчике растягивает капсулу — для текста есть Tag/Chip.

Так

Счётчик несёт только число; переполнение сворачивается в «99+».

Не так

Dot со значением — противоречие: точка не несёт числа.

Так

Нужно число — берём Counter; нужен статус — Dot.

Анатомия

Бейдж — это контейнер-капсула со значением внутри. Ширина зависит от контента, высота фиксирована по размеру. При наложении на родителя бейдж позиционируется в правом верхнем углу и выступает за его границу.

  • Контейнеркапсула со скруглением 999px; высота фиксирована, ширина тянется по контенту
  • Значениечисло, SB Sans Text Semibold, tabular-nums; центрировано
  • Индикатор переполнения«+» при значении больше максимума

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Типы

Два типа по содержимому. Counter несёт число — точное количество непрочитанного, элементов в табе, товаров в корзине. Dot не несёт числа — это компактный индикатор наличия события или статуса.

Counter

Числовой счётчик. Поддерживает переполнение «99+». Скрывается при значении 0.

Dot

Точка 8px без числа. Сигнализирует о новом событии или статусе родителя.

Тон

Тон передаёт смысл, а не просто цвет. Neutral — нейтральный счётчик без эмоциональной окраски (количество в табах, списках). Accent — привлекает внимание к новому (непрочитанное). Статусные тона — для индикации состояния родителя.

Размеры

Четыре размера счётчика и фиксированный Dot. Размер выбирается по габаритам родителя и плотности интерфейса. Однозначное число даёт круг (min-width = высота), многозначное — капсулу с горизонтальными отступами.

Контент

Содержимое бейджа — число (Counter) или его отсутствие (Dot). Краткий свод правил (подробно — в «Переполнение и формат» выше):

  • Counter несёт только целое неотрицательное число — никакого текста, букв или знаков препинания внутри (для этого есть Tag/Chip);
  • у каждого счётчика есть порог переполнения (max, по умолчанию 99) — при превышении показывается max+ (например, «99+»), точное значение — в тултипе;
  • для компактных второстепенных счётчиков («ещё N») используется безфоновый текстовый вариант badge--text в формате «+99»;
  • Dot не несёт никакого текста или числа — это только индикатор наличия события/статуса; если нужно число — используйте Counter;
  • при value = 0 счётчик скрывается по умолчанию — не показывайте пустой бейдж вместо того, чтобы его скрыть.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Переполнение и формат

У счётчика есть максимум (по умолчанию 99). Если значение его превышает, бейдж показывает max+ — например «99+», чтобы капсула не растягивалась бесконтрольно. В компактных местах (счётчик скрытых элементов, «ещё N») используется безфоновый текстовый вариант badge--text в формате «+99».

Позиционирование

Counter накладывается на правый верхний угол родителя и центрируется на его границе. Dot для статуса ставится в правый нижний угол (например, индикатор «онлайн» на аватаре). На цветном или фото-фоне у бейджа появляется белая обводка (badge--bordered), чтобы он читался отдельно от подложки.

Состояния

Поведение бейджа определяется значением и состоянием родителя.

Значение 0 — бейдж скрывается

При count = 0 счётчик не отображается; исключение — явный режим «показывать ноль».

Родитель disabled — бейдж приглушается

Тон сбрасывается в серый --disabled, чтобы не конкурировать с неактивным родителем.

Тултип при переполнении — наведите, чтобы увидеть точное значение

Доступность

Бейдж — неинтерактивный визуальный элемент: он сам по себе не получает фокус и не имеет собственной семантики вне контекста родителя. За доступность отвечает компонент, к которому бейдж прикреплён.

  • Counter: число дублируйте в aria-label родителя (например, aria-label="Входящие, 12"), а сам бейдж помечайте aria-hidden="true" — иначе число прочитается дважды;
  • Dot: без сопровождающего текста статус недоступен для скрин-ридера — добавляйте aria-label на родителе (например, «Онлайн» или «Есть новые уведомления»);
  • не полагайтесь только на тон (цвет) для критичных различий (например, Error против Success) — для пользователей с нарушениями цветовосприятия различие должно дублироваться текстом или иконкой в самом родителе;
  • title при переполнении — mouse-only тултип; если точное число важно, продублируйте его в aria-label родителя;
  • в состоянии muted бейдж только следует за aria-disabled родителя, а не сигнализирует самостоятельно.

Типографика

Значение набирается гарнитурой SB Sans Text начертанием Semibold с моноширинными цифрами (font-variant-numeric: tabular-nums), чтобы счётчик не «прыгал» при смене цифр. Кегль зависит от размера.

Цвета

Компонент использует только семантические токены. Текст на цветных тонах — --text-on-dark; на amber для контраста — тёмный --c-cgrey-800.

Для разработчиков
