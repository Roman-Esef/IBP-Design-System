Switch

Свитч мгновенно переключает состояние — например, включает или отключает определённую опцию. Действие применяется сразу, без подтверждения. Два состояния — Off / On — и интерактивные состояния Default / Hover / Focus / Pressed / Disabled / Loading.

Конструктор

Соберите свитч: переключайте состояние, интерактивное состояние и состав — превью обновляется мгновенно.

Использование

Используйте свитч, когда нужно:

  • дать пользователю возможность мгновенно включить / выключить или показать / скрыть что-либо;
  • дать мгновенный ответ без подтверждения действия.

Свитчи всегда мгновенно выполняют действие, поэтому не стоит использовать их в сценариях с кнопкой отправки данных или кнопкой сохранения — это может запутать пользователя. В этих случаях используйте Radiobutton или Checkbox.

Свитч нельзя использовать для выбора пунктов в списке из нескольких вариантов. Для этого используйте группу Checkbox.

Анатомия

Контрол свитча имеет размер 20×36 px: трек 20px высотой (равен интерлиньяжу Body M) с круглым ползунком 16px и отступом 2px со всех сторон. В группе сверху может быть заголовок.

  • Заголовок группы — опционально, H6 Strong
  • Свитч-контрол — трек 20×36px, ползунок 16px
  • Название свитча — Body M, справа от контрола
  • Хелпер — опционально, Body XS, общий компонент подкачки — см. Label / Helper

У ползунка свитча есть охранное поле: компонент контрола имеет размер 20×36 px, а круглый ползунок внутри него — 16×16 px с отступом 2px. Ход ползунка — 16px.

Варианты

Оси конфигурации компонента — каждая описана отдельным подразделом.

Состав

Вспомогательные части — название и хелпер — опциональны. Свитч рендерится без названия (например, на карточке), с названием и с хелпером. Контрол всегда выровнен по первой строке текста.

Размеры

У свитча один размер. Трек высотой 20px равен интерлиньяжу Body M, чтобы контрол центрировался по первой строке названия. Значения ниже измерены на живом экземпляре.

  • Трек — 36×20px, радиус --radius-pill.
  • Ползунок — 16×16px с отступом 2px от края трека; ход ползунка — 16px.
  • Зазор контрол ↔ название — 10px; название ↔ хелпер — 2px.
  • Pressed: ползунок временно расширяется до 19px (Material-style).

Контент

Текстовый контент свитча — название и опциональный хелпер. Краткий свод правил (подробно — в «Название свитча» ниже):

  • Название (Body M, справа от контрола; слева в строке настроек) — короткое, описывает действие в состоянии On, в положительном ключе, без указания текущего состояния в скобках;
  • Хелпер (Body XS под названием) — поясняет неочевидный эффект включения, может отражать текущий статус (например, «Применяем…» в Loading); общий компонент — см. Label / Helper;
  • в строке настроек название слева, свитч прижат к правому краю (см. «Свитч в строке настроек»);
  • не обрезайте название многоточием — оно может переноситься в несколько строк (см. «Описание работы»).

Название свитча

Название должно быть коротким и понятным. Название должно описывать, что делает свитч в активном (On) состоянии, и пишется в положительном ключе — без отрицаний и без указания текущего состояния.

Не так

«Не выключать…» — двойное отрицание; состояние «(включено)» дублирует сам свитч.

Так

Короткая положительная формулировка описывает действие в On-состоянии.

Если пользователю может быть не очевидно, что произойдёт при включении, объясните это дополнительным текстом-хелпером.

Поведение

Правила поведения: позиционирование, переполнение, анимация, адаптивность.

Описание работы

Клик по названию свитча или по самому контролу приводит к его переключению. Повторное нажатие переключает свитч обратно. Поиграйте с примером:

Свитч может использоваться как с названием, так и без него — когда понятен контекст использования (например, на карточке).

Название и хелпер (вспомогательный текст) могут располагаться в несколько рядов. Контрол всегда выравнивается по первой строке текста.

Свитч в строке настроек

Частый паттерн: название слева, контрол прижат к правому краю строки. Вся строка — активная область. Так свитчи легко сканировать в списке настроек, а контролы выровнены по правому краю.

В строке настроек контролы выравнивайте по правому краю — по общей вертикальной оси, а не по концу текста.

Не так

Выравнивание по тексту ломает вертикальную ось контролов.

Так

Контролы прижаты к правому краю — общая ось сохраняется.

Загрузка и асинхронное действие

Свитч выполняет действие мгновенно. Но если переключение запускает запрос к серверу, который может занять время, покажите промежуточное состояние Loading: в ползунке появляется индикатор, контрол временно недоступен для повторного нажатия. Когда ответ получен — свитч фиксируется в новом состоянии; при ошибке возвращается в исходное.

Состояние загрузки

Живой пример — переключение с задержкой ~1.2с

Состояния

Два состояния по вертикали — Off / On — и интерактивные состояния по горизонтали. Состояния Hover, Focus, Pressed и Loading добавлены к исходной матрице Default / Disabled.

Доступность

Видимый контрол — 20×36px, но кликабельная область шире: она включает название и хелпер. В строке настроек активная область вытягивается на всю строку. Управление с клавиатуры: Tab ставит фокус на свитч, Space / Enter переключают его, фокус показывается обводкой --primary. Контролу присваивается role="switch" с aria-checked.

Типографика

Название набирается гарнитурой SB Sans Text токеном --type-body-m, хелпер — --type-body-xs, заголовок группы — --type-h6-strong.

Цвета

Компонент использует только семантические токены. Производные оттенки (hover) и state-слои вычисляются из --primary через color-mix.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

Поведение · псевдокод (framework-agnostic)

Поведение · псевдокод (framework-agnostic) 
// 1. Состояние — бинарное на нативном input (checked=off/on),
//    отражается также в aria-checked для ассистивных технологий

// 2. Действие применяется МГНОВЕННО по клику, без подтверждения

// 3. Loading (асинхронное действие):
//    клик → оптимистично показать .sw--loading (спиннер вместо точки, ввод заблокирован)
//    → дождаться ответа сервера → успех: зафиксировать новое состояние;
//    ошибка: откатиться к предыдущему состоянию, снять .sw--loading

// 4. Клик по названию ИЛИ контролу переключает (<label> оборачивает input)

// 5. Интерактивные состояния — чистый CSS (:hover / :active / :focus-visible);
//    hover: halo 5px; pressed: halo 7px и ползунок расширяется до 19px
//    классы .sw--hover / .sw--focus / .sw--pressed — только для спецификации в документации

// 6. Клавиатура: Tab фокусирует input, Space или Enter переключают;
//    :focus-visible → outline 2px --primary, offset 2px

// 7. Disabled: input.disabled + .sw--disabled (pointer-events: none)

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функции makeSw / wireToggle в коде этой страницы).
