Label / Helper

Общие вспомогательные текстовые слоты, вынесенные из нескольких родительских компонентов в отдельную группу. Label подписывает поле ввода, Helper поясняет назначение или ограничение контрола. Оба переиспользуются в разных компонентах, но не имеют собственного визуального контейнера.

Не существуют самостоятельно — только внутри родительского компонента

Конструктор

Выберите родительский компонент и соберите его Label/Helper — состав и доступные настройки зависят от того, кто в данный момент выступает хозяином слота.

Использование

Label и Helper — два переиспользуемых текстовых слота, которые каждый компонент подключает по необходимости:

  • Label подписывает элемент управления — используется только в полях ввода (InputText, InputAutocomplete, InputDate, ReadOnlyField) — у Checkbox/Radiobutton/Switch эту роль играет собственный текст пункта;
  • Helper поясняет назначение, формат или ошибку заполнения — используется шире: во всех полях ввода, во всех контролах выбора (Checkbox, Radiobutton, Switch) и в опциях выпадающего списка (Select · DropdownList);
  • слоты включаются независимо друг от друга — компонент может использовать оба, только один или ни одного (см. таблицу ниже, раздел «Используется в компонентах»);
  • выравнивание (left/right) — общее свойство пары: Label и Helper одного поля всегда выровнены одинаково — не бывает так, чтобы Helper был справа, а Label слева;
  • Label и Helper не публикуются отдельно от родителя — см. «Правило зависимости» ниже.

Правило зависимости

Label и Helper — не самостоятельные компоненты дизайн-системы. У них нет собственного узнаваемого контейнера: вне контекста родителя их невозможно опознать на экране. Публикуются только как часть композиции Checkbox, Radiobutton, Switch или полей ввода.

Не так
Label Helper

Label/Helper сами по себе, без родительского контрола или поля — так они на экране не встречаются.

Так

Helper всегда встроен в конкретный компонент — здесь в Checkbox, в паре с его названием.

Каждый слот переиспользуется независимо — компонент может задействовать только Helper, только Label, или ни одного из них.

Композиция с родителем

Одни и те же слоты выглядят одинаково независимо от родителя — меняется только то, что их окружает.

Анатомия

Оба слота набираются одним и тем же типографским токеном Body XS и отличаются только цветом и составом; выравнивание задаётся одной общей настройкой на оба слота.

Label

  • Текст — подпись поля, без переноса, обрезается многоточием.
  • Иконки — опционально, одна или две, любого назначения — редкий кейс.
  • Выравнивание — left/right — общее с Helper (см. право зависимости).

Helper

  • Текст — пояснение или ограничение, может переноситься на несколько строк.
  • Выравнивание — left/right — общее с Label, по ориентации родителя.

Варианты

Не применимо: компонент существует в единственной конфигурации, отдельных вариантов нет.

Размеры

У Label и Helper нет собственной шкалы размеров — оба слота всегда набираются одним и тем же токеном Body XS, независимо от размера родительского компонента. Значения ниже измерены на живом экземпляре.

Контент

Текстовые правила для обоих слотов:

  • Label — короткое существительное или номинативная фраза, называющая поле («ИНН контрагента», а не «Введите ИНН контрагента»); без двоеточия в конце; не переносится — при нехватке ширины обрезается многоточием;
  • Helper — короткое пояснение к формату или ограничению значения («10 или 12 цифр»), не дублирует Label; может занимать несколько строк — переносится свободно, без обрезания;
  • Helper · Error — конкретная причина ошибки («Нужно выбрать хотя бы один вариант»), а не общее «Ошибка» без пояснения;
  • иконки в Label — редкий кейс: добавляйте одну или две иконки, только если одной подписи недостаточно для контекста; если иконка интерактивна (действие) — обязателен aria-label (см. «Доступность»).

Label

Подпись над полем ввода. Используется в InputText, InputAutocomplete, InputDate, ReadOnlyField. Может содержать одну или две дополнительные иконки любого назначения (информирование, действие, статус и т.п.) — редкий кейс, только когда одной подписи недостаточно для контекста.

Helper

Пояснительный текст под контролом или полем. Используется в Checkbox, Radiobutton, Switch, InputText, InputAutocomplete, InputDate, ReadOnlyField, а также как вторая строка опции в Select · DropdownList. Поведение

Не применимо: специфического поведения (позиционирование, анимация, адаптивность) у компонента нет.

Состояния

Оба слота синхронизированы с состоянием родительского компонента: Default и Disabled — у обоих слотов, Error — только у Helper: Label визуально не меняется при ошибке родителя — сообщение об ошибке несёт только Helper.

Доступность

Label и Helper — вспомогательный текст, который должен быть программно связан с элементом управления, а не просто визуально расположен рядом.

  • Label связывается с полем через <label for="…"> (либо оборачивая инпут) — клик по подписи должен фокусировать поле;
  • Helper связывается через aria-describedby, указывающий на id текста Helper — иначе скрин-ридер озвучит только значение поля без пояснения;
  • Helper в состоянии Error дополнительно получает role="alert" (или aria-live="polite"), чтобы сообщение об ошибке озвучивалось сразу при появлении, а не только при переходе фокуса;
  • иконка предупреждения в Helper — декоративна (aria-hidden="true"), сам смысл ошибки несёт текст;
  • иконки в Label (редкий кейс, до двух) — если иконка интерактивна (действие) — обязателен aria-label с названием действия («Удалить», «Изменить»), а не название иконки; если иконка информационная — связывается с Tooltip тем же способом, что и любой другой информационный тултип.
