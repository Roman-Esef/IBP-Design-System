ProgressBar

Линейный индикатор прогресса: заполнение трека отражает долю значения от максимума. Определённый режим (значение известно) и неопределённый (длительность неизвестна); шесть тональных вариантов на локальных статусных токенах; три размера.

Конструктор

Соберите прогресс-бар: тон, размер, значение, формат и состав — превью обновляется мгновенно.

Использование

Используйте ProgressBar, когда нужно:

  • показать долю выполнения известного процесса — загрузка файла, заполнение анкеты, прогресс к цели;
  • визуализировать величину в диапазоне 0–100 без бинарного статуса — например, вероятность сделки, скор, заполненность лимита;
  • показать, что процесс идёт, но точный процент неизвестен — режим Indeterminate;

Не используйте ProgressBar, чтобы просто показать факт статуса без величины («Активна», «В архиве») — для этого есть Chip или Badge. Для мгновенного действия внутри кнопки (сохранение, отправка) используйте .btn--loading в Buttons, а не отдельный ProgressBar. Для уведомления о фоновом процессе, который не привязан к конкретному месту на экране, используйте Toast.

Анатомия

Компонент состоит из трека и заливки; заголовок, значение и хелпер — опциональные вспомогательные слоты вокруг трека.

  • Заголовок (Label) — опционально, Body S, слева над треком
  • Значение (Value) — опционально, Body S Strong, справа над треком, синхронизировано с заливкой
  • Трек (Track) — фон на 100% ширины родителя, скругление --radius-pill
  • Заливка (Fill) — ширина = value / max, тот же радиус, что и трек
  • Хелпер — опционально, Body XS, под треком; общий компонент — см. Label / Helper

Размеры

Три размера различаются только высотой трека и заливки; типографика заголовка/значения/хелпера не меняется. Значения ниже измерены на живом экземпляре.

  • S — 4px: компактные места — таблицы, списки, карточки.
  • M — 6px: размер по умолчанию для форм и карточек.
  • L — 8px: акцентные показатели — дашборды, крупные карточки метрик.
  • Скругление трека и заливки — всегда --radius-pill, независимо от размера.

Состав

Заголовок, значение и хелпер добавляются независимо друг от друга. Плавающее значение — расширенный вариант, решающий частную задачу (величина без формы), добавлен поверх исходного материала.Новое

Состояния

Состояния описывают поведение самого индикатора — независимо от них выбирается тон (см. «Цвета»). Indeterminate и явная граница «сливера» на малых значениях добавлены поверх исходного материала.Новое

Тон — независимая от состояния настройка на локальных статусных токенах. Нейтральный тон system соответствует референсному материалу («Вероятность сделки»); остальные — расширение под ситуативную семантику success/warning/error/info.

Авто-подбор тона по порогамНовое

Опциональный helper вычисляет тон автоматически из значения по процентным порогам — вместо ручного выбора тона. Направление по умолчанию — «выше — лучше» (прогресс к цели); для метрик, где высокое значение означает риск (просрочка, нагрузка), передайте свои пороги с обратным порядком тонов — компонент не диктует направление, это делает конфигурация.

Корнер-кейсы

Случаи, не покрытые исходным референсом, но важные для корректной реализации.Новое

Контент

  • Заголовок — короткое существительное или словосочетание (1–4 слова), описывающее, что именно измеряется («Вероятность сделки», «Заполнено анкетой»); без указания текущего значения в тексте — значение выводится отдельным слотом.
  • Значение — всегда число: процент («50%») или дробь текущее/максимум («50/100»); формат выбирается один раз для всего экрана, не смешивайте оба на одной странице.
  • Хелпер — поясняет контекст или следующий шаг («Осталось 2 поля», «Обновляется каждую минуту»); опционален, не дублирует заголовок.
  • Не обрезайте значение и не переносите его на вторую строку — заголовок обрезается многоточием первым.

Доступность

  • Корневой элемент несёт role="progressbar" с aria-valuenow / aria-valuemin / aria-valuemax; в Indeterminate атрибут aria-valuenow не указывается вовсе (это предписывает WAI-ARIA), вместо него — опциональный aria-valuetext со словесным статусом («Загрузка»).
  • Если видимого заголовка нет — обязателен aria-label, описывающий, что именно измеряется.
  • Тон не должен быть единственным носителем смысла: если статус критичен (например, ошибка), продублируйте его текстом в заголовке/хелпере, а не только цветом заливки.
  • Анимация заливки и Indeterminate — уважают prefers-reduced-motion: переход ширины отключается, цикл Indeterminate замедляется вдвое (1.6с → 3.2с).
  • Компонент не интерактивен и не входит в таб-порядок — фокус на него не ставится.

Типографика

Заголовок и хелпер набираются гарнитурой SB Sans Text; значение — тем же семейством с моноширинными цифрами (tabular-nums) для устойчивой ширины при обновлении.

Цвета

Трек — всегда нейтральный --border-light независимо от тона. Заливка — базовый цвет локальной статусной рампы; Disabled переопределяет и трек, и заливку токенами --disabled-*.

Что предлагаем добавитьПредложение

Референс из исходной ДС покрывал только два статичных варианта (форма с Label/Value и вариант с плавающим значением). Ниже — идеи, сверенные с практиками Material Design, Ant Design, Carbon и Polaris, которые в этой версии не реализованы и требуют отдельного решения перед стартом.

  • Круговой / радиальный прогресс — кольцевой индикатор для аватаров с загрузкой, миниатюр файлов и виджетов дашборда (аналог Material CircularProgress, Ant Progress type="circle"). Другая геометрия и разметка — отдельный компонент, не модификатор линейного.
  • Составной Upload-прогресс — карточка с именем файла, размером/скоростью/ETA и действиями «Отменить» / «Повторить» (аналог Carbon FileUploader, Ant Upload). Требует состояний ошибки загрузки и повторной попытки — лучше собрать как отдельный организм поверх текущего ProgressBar, а не расширять его API.
  • Анимированный счётчик значения — плавный count-up текста значения синхронно с ростом заливки; микро-интеракция для отчётов и результатов, не критична для форм.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки, не читая код этой страницы построчно.

Поведение · псевдокод (framework-agnostic) 
// 1. Значение всегда клампится в границы [0, max] перед рендером —
//    отрицательные и превышающие max значения не должны ломать вёрстку
value = Math.max(0, Math.min(max, rawValue));
widthPercent = (value / max) * 100;

// 2. Сливер: при 0 < value < ~6% принудительно задаём min-width = высоте
//    трека (.pbar__fill--sliver), иначе скругление трека визуально «съедает»
//    заливку и прогресс выглядит пустым. При value === 0 сливер НЕ применяется —
//    пустой трек означает «ещё не начато».

// 3. Indeterminate: aria-valuenow не рендерится вовсе (не «0»), а не просто скрывается;
//    визуальная анимация — на отдельном элементе, не на реальной ширине

// 4. Плавающее значение (.pbar--floating): метка — потомок заливки; в CSS у метки
//    явный width:max-content — это критично, иначе при width:auto width решается через
//    равенство left+width+right=containerWidth, если заданы и left, и right, и метка
//    растягивается на ширину контейнера вместо ширины текста. После отрисовки
//    (requestAnimationFrame + повтор на document.fonts.ready) измерить:
//      trackW = track.clientWidth; fillW = fill.getBoundingClientRect().width; markW = marker.offsetWidth;
//    left = clamp(fillW - markW/2, 0, trackW - markW);  // центр метки — над краем заливки,
//                                                        // но не выходит за границы трека
//    Пересчитывать при каждом изменении value и при resize контейнера.

// 5. Плавная анимация ширины — переход CSS width .3s ease;
//    под prefers-reduced-motion переход отключается (значение появляется мгновенно),
//    а цикл Indeterminate замедляется вдвое, а не отключается полностью
//    (полное исчезновение анимации у Indeterminate убрало бы единственный сигнал «идёт процесс»)

Это не готовый сниппет для вставки — алгоритм, который реализуется поверх вашего стека (рабочий референс на vanilla JS — функции makePbar / positionFloatingMarkers в коде этой страницы).

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не обязательный контракт; адаптируйте под конвенции вашей кодовой базы.

ProgressBar.types.ts 
type ProgressBarTone = 'accent' | 'success' | 'warning' | 'error' | 'info' | 'system';
type ProgressBarSize = 's' | 'm' | 'l';
type ProgressBarValueFormat = 'percent' | 'fraction';

interface ProgressBarProps {
  value?: number;                    // текущее значение; игнорируется при indeterminate
  max?: number;                      // по умолчанию 100
  tone?: ProgressBarTone;             // по умолчанию 'accent'
  size?: ProgressBarSize;             // по умолчанию 'm'
  valueFormat?: ProgressBarValueFormat; // по умолчанию 'percent'
  label?: React.ReactNode;            // заголовок над треком; также источник aria-label, если floating
  showValue?: boolean;               // показывать значение в заголовочной строке (не в floating-режиме)
  helper?: React.ReactNode;           // подпись под треком
  floating?: boolean;                // значение плавает над краем заливки вместо заголовочной строки
  indeterminate?: boolean;           // длительность неизвестна — анимированная полоса вместо value
  disabled?: boolean;
  autoTone?: boolean;             // тон вычисляется из value по порогам, игнорируя ручной tone
  toneThresholds?: { upTo: number; tone: ProgressBarTone }[]; // пороги в % от max; по умолчанию 30/70 → error/warning/success
  stack?: { value: number; color: string; label?: string }[]; // Предложение — составной прогресс из нескольких величин
  'aria-label'?: string;
}
