PageHeader

Заголовок страницы рабочей области: идентифицирует открытую сущность и несёт основные действия страницы. Состоит из независимых частей — Title, Chips, кнопка возврата, Subtitle, Actions — и адаптируется вплоть до мобильной версии.

Ширина контейнера

На ширине Mobile компонент переключается в раскладку .phead--stack: actions переносятся вниз на всю ширину, Title переносится максимум в две строки.

Использование

PageHeader — заголовок страницы рабочей области. Один на страницу, всегда сверху, под строкой Breadcrumbs. Title заголовка — семантический h1 страницы и наименование открытой сущности: сделки, финансового инструмента, раздела.

Когда использовать

  • На каждой странице рабочей области — списке сущностей, карточке сделки, странице инструмента, отчёте.
  • Когда странице нужны основные действия (выгрузка, расчёт, создание) — они живут в Header_Actions, а не в контенте.
  • Когда нужно показать статус сущности — чипы после заголовка.
  • Когда со страницы есть возврат на уровень выше — кнопка Return после заголовка.

Когда не использовать

  • Заголовок плашки — TileHeader (компонент Tile), не PageHeader.
  • Шапка модального окна — Modal_Top (компонент Modal).
  • Заголовок Popover — собственный Header компонента Popover.

Сравнение с соседними компонентами

  • Breadcrumbs — путь до страницы; отдельный компонент строкой над PageHeader, в состав не входит.
  • TileHeader — заголовок блока внутри страницы: токен на два шага меньше (H5 Strong), действия — IconButton, а не Button M.
  • TableFilter — панель фильтра; ставится ниже PageHeader, над таблицей.

Примеры в интерфейсе

Типовые страницы продукта: список сущностей (Title + Actions), карточка сделки (Title + Edit + Chips + Actions + MenuButton), вложенная страница (Title + Return).

Контент

Title — наименование сущности как в системе: номер + название («D-007. ПАО «Газпром»»), без точки в конце. Подпись Return называет, куда ведёт возврат («В сделку»), а не абстрактное «Назад». Тексты кнопок Actions — глагол действия («Выгрузить», «Рассчитать ОСЗ»).

Так
«Плановые платежи по сделке» — Title называет содержимое страницы; Return «В сделку» говорит, куда вернёмся; в Actions максимум три кнопки, остальное в меню «⋮».
Не так
«Страница платежей» — канцелярит; Return «Назад» без адреса; пять кнопок в ряд — переполненный хэдер, действия не считываются.
Так
Длинное наименование усечено многоточием, полное значение — в Tooltip по наведению. Чипы показывают только актуальные статусы (1–3).
Не так
Title в две строки на десктопе; шесть чипов-статусов подряд; иконка слева используется как декоративная, без информационной роли.

Поведение

Переполнение действий

В Actions допускается не больше трёх кнопок. Если действий больше — сверх лимита они попадают в MenuButton: клик по «⋮» открывает контекстное меню со списком опций. Приоритетное действие остаётся кнопкой Accent, второстепенные уходят в меню первыми.

MenuButton открыт

Усечение заголовка

Title занимает одну строку и усекается многоточием, когда не помещается. Если текст усечён (scrollWidth > clientWidth) — по наведению показывается Tooltip с полным значением. Chips, Return и Actions при усечении не сжимаются.

Усечённый Title · контейнер 640px

Редактирование заголовка

Кнопка Edit присутствует на страницах сущностей типа «Сделка», где Title — редактируемое наименование. Клик переводит заголовок в режим переименования (инлайн-поле или модалка — по решению страницы); после подтверждения Title обновляется.

Адаптивность

Компонент адаптируется к ширине контентной области вплоть до мобильной версии: сначала второстепенные кнопки Actions сворачиваются в MenuButton, затем блок Actions переносится под заголовок на всю ширину, Title получает право на перенос максимум в две строки.

≥ 1024 · полный ряд
< 1024 · второстепенные действия в меню
< 720 · мобильная раскладка .phead--stack

Состояния

PageHeader собственных состояний не имеет — это статичный контейнер. Интерактивны и несут состояния только вложенные компоненты: Edit и MenuButton (IconButton/Button: hover, active, focus, disabled), Return и кнопки Actions (Button), информер чипа. Недоступное действие показывается disabled-кнопкой, а не скрывается — если оно в принципе применимо к сущности.

Доступность

  • Title — семантический h1 страницы; на странице он один.
  • Иконка слева и иконки сабтайтла декоративны — aria-hidden="true".
  • Edit — aria-label="Переименовать"; MenuButton — aria-label + aria-haspopup="menu" и aria-expanded.
  • Подпись Return называет, куда ведёт возврат; у icon-only варианта — aria-label с адресом.
  • Усечённый Title дублируется Tooltip'ом (роль tooltip, связь через aria-describedby) — полное значение доступно с клавиатуры.
  • Порядок фокуса: Edit → чипы (информеры) → Return → кнопки Actions → MenuButton.

Что предлагаем добавить

  • Sticky-режим — компактная закреплённая версия заголовка при прокрутке длинной страницы (Title XS + Actions).
  • Skeleton — состояние загрузки наименования сущности (шиммер-плашка вместо Title/Subtitle).
  • Слот табов — регламентированное место строки Tab под PageHeader (сейчас табы кладутся в контент страницы).
  • Переполнение чипов — чип-счётчик «+N» по образцу InputAutocomplete, если статусов больше трёх.
