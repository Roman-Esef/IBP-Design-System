Tile

Один из основных компонентов интерфейса — плашка на рабочей области страницы. Состоит из TileHeader и контентной области; наполнение контента индивидуально под задачу и тип тайла.

Ширина тайла привязана к колонкам контентной области (3–12). Контент здесь — типовой пример: наполнение всегда индивидуально под задачу тайла.

Использование

Tile — основная плашка рабочей области страницы: группирует связанные данные и действия по объекту (сделка, продукт, инструмент, контрагент). Описаны общие правила построения; для каждой конкретной плашки в проекте заводится локальный компонент, разработанный по этим правилам.

Дисклеймер

Описаны общие правила построения и использования тайла.

Для каждой конкретной плашки в проекте заводится локальный компонент, разработанный по правилам этой страницы.

Контент в контентной области отличается в зависимости от типа и задачи плашки и может быть любым.

Когда использовать

  • Для группировки связанных полей и действий по объекту на рабочей области страницы.
  • Когда блоку нужен заголовок, опциональный статус (чипы/бейдж) и локальные действия в шапке.
  • Когда блок раскрывается/сворачивается — вариант AccordionTile.
  • Когда заголовок не нужен и блок целиком состоит из контента — вариант TileHeadless.

Когда не использовать

  • Внутри модальных окон — там используется Card (собственная система отступов, состояния перетаскивания/выбора).
  • Для отдельного объекта в списке или строке — это Entity, а не тайл.
  • Как контейнер-обёртку без содержательного заголовка и контента — тайл не декоративен.

Размеры

Ширина плашки привязана к колонкам контентной области страницы и может занимать от 3 до 12 колонок. Высота в строке фиксированная — определяется плашкой с максимальным объёмом контента с учётом возможных алертов; при изменении состояния/наполнения высота может меняться.

Расстояние между колонками — 16px, между строками контента — 16 или 24px.

Контент

Наполнение контентной области индивидуально под задачу тайла и может быть любым. Разрешённые элементы — текст, дивайдеры, ReadOnly-инпуты, кнопки, ссылки; в отдельных случаях внутри тайла размещаются плашки-аккордеоны.

Внутренние отступы и ритм

  • Отступы контентной области — 10 / 20 / 24 / 20 (top / right / bottom / left). TileHeadless — 24 / 24 / 32 / 24.
  • Расстояние между строками — 16 или 24px.
  • Расстояние между колонками — 16px.
  • TileHeader размещается над контентом вплотную, без отступа; Alert (если есть) — между ними полноширинным блоком.

Правила наполнения

Хорошо
Группировать поля в 2–3 колонки по 16px, длинные значения переносить или усекать с Tooltip, действия по строке — в ReadOnly-инпутах и ссылках.
Плохо
Смешивать несвязанные данные в одном тайле, дублировать заголовок внутри контента, задавать произвольные отступы вне сетки 16/24.

Поведение

Сворачивание AccordionTile

Клик по шеврону (IconButton в Actions) переключает состояние. Полный коллапс скрывает весь контент, частичный — оставляет заранее заданную сводку и сворачивает только вложенный блок. Шеврон поворачивается на 180°.

Высота в строке

Высота плашки в строке одинаковая и выставляется по тайлу с наибольшим объёмом контента (с учётом возможных алертов). При изменении состояния или наполнения одной плашки высота всей строки может пересчитаться.

Адаптивность

Ширина пересчитывается по сетке колонок контейнера. Длинный заголовок усекается в одну строку с многоточием; полное значение — по Tooltip.

Состояния

Tile собственных состояний не имеет — нет hover / active / selected у самой плашки. Интерактивны только вложенные элементы: IconButton в хэдере, а также Button и Link в алертах и контентной области. Состояния перетаскивания/выбора есть только у родственного компонента Card.

  • IconButton в хэдере — собственные состояния Default / Hover / Focus / Disabled (см. IconButton).
  • Button / Link в контенте и алертах — состояния своих компонентов.
  • AccordionTile — не состояние, а переключаемая конфигурация (collapsed / expanded) через .tile--collapsed.

Доступность

  • Заголовок тайла — семантический heading (h2–h4 по уровню страницы), а не просто жирный текст.
  • Иконка-предупреждение в тайтле декоративна: aria-hidden="true"; смысл дублируется текстом Alert или чипа.
  • IconButton без текста — обязателен aria-label.
  • Шеврон аккордеона — aria-expanded на кнопке + aria-controls на сворачиваемый блок; сам блок при коллапсе скрыт (display:none), а не только визуально.
  • Alert-слот — role="status"/alert по тону (см. Alert); озвучивается при появлении.
  • Усечённый заголовок — полное значение доступно через Tooltip и title.
  • prefers-reduced-motion — анимация поворота шеврона отключается.
