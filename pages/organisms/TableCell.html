TableCell

Ячейка таблицы и ячейка шапки (TableHeader). Универсальный контейнер для любого типа контента строки: текст, дерево, чипы, контролы и редактируемые поля. Высота строки фиксирована и растёт только там, где ячейка сама разрешает перенос текста — плотностей строки нет.

Каждая строка начинается и заканчивается ячейкой-разделителем 8px. Первая строка — TableHeader. Наведите курсор, чтобы увидеть hover, скрытые действия и Tooltip у усечённого текста.

Использование

TableCell — атом строки таблицы. Из ячеек разных типов собирается любая колонка: реестр сделок, финансовая сводка, дерево иерархии, редактируемая форма-таблица.

Анатомия

Ячейка — flex-контейнер с горизонтальным паддингом, нижним бордером и слотом контента. Ячейка шапки (TableHeader) построена по той же сетке, но с подписью, кнопкой сортировки и действием. Строка всегда начинается и заканчивается ячейкой-разделителем шириной 8px.

Варианты

Не применимо: компонент существует в единственной конфигурации, отдельных вариантов нет.

Размеры

Плотностей строки нет — высота ячейки фиксирована (48px). Ячейка вправе вырасти выше только когда сама разрешает перенос текста вместо усечения в три точки (толкает высоту строки вверх — например, длинное название без Truncate).

Правила по типам

  • Текст — по левому краю; по умолчанию усекается в одну строку (.tc__text--truncate) с Tooltip по наведению; если важно показать значение целиком — выключите усечение, ячейка перенесёт текст и вырастет по высоте.
  • Числа, суммы — включите настройку «Числовое значение»: выравнивание справа + табулярные цифры (.tc--numbers).
  • Вторая строка — опциональная мелкая строка под значением (.tc__subtext), доступна для любого типа ячейки с текстом: название + ИНН, сумма + валюта и т.п.
  • Prefix / Postfix — независимые настройки, включаются по отдельности в любом сочетании.
  • Пустое значение — прочерк «—» (.tc__empty, цвет inactive). Не оставляйте ячейку визуально пустой и не пишите «нет»/«null».
  • Действий в ячейке — максимум 2–3; больше — сворачивайте в кебаб-меню.

Поведение

Усечение текста, раскрытие дерева и навигация по сетке.

Усечение текста

Тултип с полным значением показывается только когда текст реально обрезан (scrollWidth > clientWidth), а не всегда у типа «Текст» — короткие значения тултип не получают. Проверка идёт и по hover, и по клавиатурному фокусу ячейки.

Дерево (TreeNode)

Твисти (шеврон) переключает aria-expanded и разворачивает/сворачивает дочерние строки — с поворотом глифа на 90°. Отступ строки считается от уровня вложенности (--tc-level × 20px), у листьев (без потомков) твисти не рендерится — вместо него зарезервировано пустое место той же ширины, чтобы значение не съезжало по горизонтали.

Навигация по сетке

Внутри таблицы фокус перемещается между ячейками стрелками (grid-паттерн), а не табом по каждому интерактивному элементу — Tab уводит из таблицы целиком. Фокус-ячейка получает видимую обводку; если в ячейке несколько интерактивных элементов (например, кнопка + чип), первую активацию Enter/Space получает первый из них.

Corner cases

  • Фон + состояние комбинируются: hover/focus у accent- и pinned-колонок имеют собственные тона (--bgtable-accent-hover, --bgtable-pinned-hover).
  • EditMark — угловой маркер в правом верхнем углу ячейки, виден до сохранения/сброса правки.
  • Скрытые действия (.tc__hidden) появляются только по наведению/фокусу строки; сочетаются с усечённым текстом и Tooltip в той же ячейке.
  • Disabled-ячейка не перехватывает клики (pointer-events:none), контент — цвет inactive.

Доступность

  • Семантика таблицы — использовать нативные <table>/<th>/<td> либо ARIA-роли role="grid"/"columnheader"/"gridcell". Классы .tc/.th — только оформление.
  • Сортировка — на ячейке шапки aria-sort="ascending|descending|none"; кнопка сортировки имеет aria-label.
  • Выбор строки — реальный чекбокс; состояние строки — aria-selected. Заголовочный чекбокс = «выбрать все».
  • Клавиатура — перемещение по ячейкам стрелками (grid pattern), фокус-ячейка получает видимую обводку (:focus-visible, inset 2px primary).
  • Усечённый текст — полное значение доступно через Tooltip (hover и focus) и title.
  • TreeNode — твисти: aria-expanded; строки дерева — role="row" + aria-level.
  • Иконка после значения и кнопки без текста — обязателен aria-label/title.
  • Skeleton — контейнер aria-busy="true"; шиммер отключается при prefers-reduced-motion.
  • Контраст — фоны состояний и текст соответствуют WCAG AA; статус не передаётся только цветом (дублируется текстом/иконкой).

Бэклог

Кейсы, которых недостаёт документации, — на существующих токенах и компонентах.

  • Инлайн-график в ячейке — мини-спарклайн как тип контента — частый кейс в финансовых таблицах.
  • Групповая шапка (colspan) — заголовок, объединяющий несколько колонок.
