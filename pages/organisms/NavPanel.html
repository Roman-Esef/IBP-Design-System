NavPanel · Панель навигации

Главная навигация приложения в левой части каждой страницы. Отображает все разделы продукта, сгруппированные по подразделениям и типам. Один компонент — три режима: свёрнутый Rail, развёрнутый-оверлей Drawer и закреплённый Fixed.

Конструктор

Переключите режим панели и опции пунктов. В режиме Rail наведите курсор на пункт — подпись всплывёт тултипом справа.

Использование

NavPanel — точка входа во все разделы продукта. Живёт в левой части лайаута на каждой странице, показывает главные страницы по разделам и подсвечивает текущую. Свёрнута по умолчанию (Rail), разворачивается в Drawer поверх контента или закрепляется как Fixed-панель.

Когда использовать

  • основная навигация по разделам всего приложения;
  • разделов много, они делятся на смысловые блоки;
  • нужен быстрый доступ к профилю и выходу из системы.

Когда не использовать

  • переключение вьюх одного экрана — это Tab или SegmentControl;
  • действия над объектом — Context Menu;
  • путь до текущей страницы в шапке — Breadcrumbs.

Рекомендации

Так
Разделы сгруппированы в блоки с заголовком (Origination, Pipeline, Отчеты). Заголовок блока виден в Drawer/Fixed и заменяется короткой чертой в Rail.

Блоки помогают ориентироваться в длинном списке из десятков пунктов.

Не так
Все пункты одним сплошным списком без блоков и заголовков — пользователь теряется среди 30 строк.

Без группировки Rail превращается в неразличимую колонку иконок.

Так
Счётчик сущностей — компонент Badge (accent) справа от подписи; в Rail он уходит на угол иконки.

Badge опционален и нужен только там, где число реально помогает.

Не так
Бейджи навешаны на каждый пункт «для симметрии», хотя чисел за ними нет.

Пустой или декоративный бейдж — шум, снижающий значимость реальных счётчиков.

Анатомия

Сверху — бургер (и пин в развёрнутых режимах). Ниже — прокручиваемый список: пункт «Главная», блоки-секции с заголовками и пунктами меню. Внизу закреплён футер с профилем пользователя и выходом.

  • Burger — IconButton (глиф left-menu), сворачивает/разворачивает панель
  • Pin / Unpin — закрепляет панель в режиме Fixed (только Drawer/Fixed)
  • Navigation_Item — пункт меню: иконка + подпись, опц. Badge
  • Navigation-block — блок разделов с заголовком-секцией
  • Заголовок блока — метка подраздела (в Rail — короткая черта)
  • Footer — Avatar + профиль + IconButton выхода

Варианты

Две оси конфигурации: режим отображения панели и набор элементов, из которых она собрана.

Режимы отображения

Rail (свёрнут), Drawer (развёрнут-оверлей с тенью) и Fixed (закреплён, шов справа). Переключение — бургером; закрепление — кнопкой пина.

Элементы панели

Каждый элемент существует в двух обликах — Rail и Drawer.

Размеры

Ширина зависит от режима; высота пункта фиксирована во всех режимах. Значения измерены на живом экземпляре.

Контент

Правила подписей, иконографики и переполнения:

  • Подпись пункта — короткое существительное или название раздела; длинная подпись усекается многоточием, полный текст — тултипом (в Rail подпись всегда во всплывающем тултипе).
  • Иконка — глиф 24×24 из раздела Icons (Menu). У каждого пункта своя иконка — именно она опознаёт раздел в Rail.
  • Заголовок блока — название подразделения или типа (Origination, Pipeline, Отчеты); в Rail заменяется короткой чертой-разделителем.
  • Badge — только целое число сущностей; опционален. Пустой или декоративный бейдж не допускается.
  • Профиль в футере — ФИО (одна строка с усечением), должность и организация с признаком «+N» дополнительных.

Поведение

Переключение режимов, тултипы Rail, закрепление и выход.

Rail — свёрнутый режим

Свёрнутое меню не отбрасывает тень. При наведении на пункт справа от него всплывает тултип с названием. Разворачивание в Drawer — по клику на иконку бургера.

Drawer — развёрнутый оверлей

Развёрнутое меню лежит поверх контента и отбрасывает тень Shadow4.0_modalform. Сворачивается по клику на бургер или по клику вне области меню. Клик по IconButton выхода деавторизует пользователя.

Fixed — закреплённая панель

По клику на кнопку в правом верхнем углу панель закрепляется на странице: контент подстраивается под новую ширину контентной области, иконка в кнопке меняется с pin-menu на unpin-menu. Повторный клик открепляет панель и возвращает её в режим Drawer.

Доступность

Панель — элемент <nav> с aria-label="Главное меню". Текущая страница помечена aria-current="page". Кнопки бургера, пина и выхода имеют aria-label; пин отражает состояние через aria-pressed. Заблокированный пункт — aria-disabled="true" и вне таб-порядка.

Клавиатура

TabПеремещение по пунктам и кнопкам панели по порядку
EnterПереход к разделу / нажатие бургера, пина, выхода
EscСворачивает открытый Drawer

Скрин-ридер

  • значение Badge включается в доступную подпись пункта («Задачи, 7»);
  • тултип Rail дублирует подпись пункта, скрытую визуально;
  • фокус виден кольцом --primary на пункте;
  • пин объявляет закреплённость через aria-pressed.

Типографика

Цвета

Только семантические токены. Фон панели — Main Menu; пункты используют табличные состояния строк.

Поведение · псевдокод (framework-agnostic)

Поведение · псевдокод (framework-agnostic) 
// Состояние: mode ∈ { rail, drawer, fixed }, current (id активного пункта).

// 1. Бургер:
//    rail   → drawer  (развернуть оверлей)
//    drawer → rail    (свернуть)
//    fixed  → бургер скрыт / не меняет режим (панель закреплена).

// 2. Пин (только в drawer/fixed):
//    drawer → fixed  : закрепить, контент ужать под ширину панели,
//                      иконка pin-menu → unpin-menu, aria-pressed=true
//    fixed  → drawer : открепить, вернуть оверлей, unpin-menu → pin-menu.

// 3. Drawer как оверлей:
//    показывать тень Shadow4.0_modalform;
//    клик вне области панели ИЛИ Esc → свернуть в rail;
//    fixed тени не имеет и клик-снаружи не сворачивает.

// 4. Rail-тултип:
//    hover/focus пункта → показать всплывающую подпись справа
//    (в drawer/fixed подпись видна в строке, тултип не нужен).

// 5. Выбор пункта: клик → навигация, current = id;
//    рендер: пункт с current → nav__item--selected + aria-current.

// 6. Выход: клик по logout → деавторизация пользователя.

Это алгоритм поверх вашего стека, не готовый сниппет. Разметку панели и режимы см. в nav-panel.page.js (функция buildNav).

Что предлагаем добавить

Кейсы, которых недостаёт документации, — на существующих токенах и компонентах.

  • Вложенные разделы — раскрывающийся под-список у пункта (аккордеон) для разделов с подстраницами.
  • Поиск по меню — поле поиска над списком при большом числе разделов.
  • Избранное / закреплённые — блок часто используемых пунктов наверху списка.
  • Состояние загрузки — скелетон пунктов, пока список разделов подгружается.
