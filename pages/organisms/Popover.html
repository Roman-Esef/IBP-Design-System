Popover

Popover — нон-модальный всплывающий контейнер, привязанный к триггеру. Показывает расширенный контекстный контент (текст с форматированием, интерактивные элементы, формы, легенды, списки). В отличие от Tooltip — требует клика для открытия и не закрывается при уходе курсора; в отличие от Modal — не блокирует страницу, не имеет скрима и закрывается кликом вне себя.

Конструктор

Соберите поповер: тип триггера, ширина (5 шагов шкалы), наличие шапки, состав подвала, тип контента и стрелка-указатель.

Использование

Поповер показывает расширенный контекстный контент, требующий клика (в отличие от Tooltip — только hover/focus, только строка текста) и не блокирующий страницу (в отличие от Modal). Используйте для легенды чарта, карточки предпросмотра сущности, быстрого inline-редактирования 1–3 полей, фильтра, подтверждения лёгкого действия. Для многошаговых сценариев, обязательных к заполнению форм и любого контента, которому не хватает ширины 560px или высоты со скроллом — Modal или отдельная страница.

Правило переключения: если контенту не хватает ширины свыше 560px или вертикальная прокрутка тела превышает разумный предел (см. «Размеры») — это сигнал перейти на Modal.

Правила Header и Footer — как у Modal, но с одним отличием: в Header не размещаются кнопки действий — только заголовок и ✕ (опционально — служебная метка/ссылка). Все действия живут в Footer.

Анатомия

2 обязательные и 3 опциональные части. Header и Footer — фиксированные зоны (flex: none), Body — единственная гибкая и прокручиваемая зона (flex: 1).

  • Trigger — вне поповера. Любой интерактивный элемент: кнопка, иконка, ссылка, чип. Обязателен
  • Header — заголовок + ✕, ничего больше. Опционален
  • Body — произвольный контент, единственная прокручиваемая зона. Обязателен
  • Footer — foot-left служебное/инфо, foot-right Secondary+Primary. Опционален
  • Arrow — указывает на триггер. Опциональна, по умолчанию выключена

Варианты Header

Заголовок и ✕ — обязательный минимум. Справа от заголовка можно разместить служебную метку состояния или ссылку — но не действие.

Размеры

Ширина — 5 фиксированных шагов, резиновой ширины нет. Высота растёт по контенту (auto) и ограничена сверху max-height тела — дальше тело прокручивается, шапка и подвал остаются на месте. Горизонтальная прокрутка запрещена.

Контент

Body — единственная обязательная зона; поповер не диктует состав. Header и Footer добавляются по контенту:

Тип контентаПримерыHeaderFooter
Текст + ссылкаПояснение к полю, help-текст, определение термина——
Список / легендаЛегенда чарта, набор статусовОпционально—
ФормаБыстрое редактирование тега, фильтр, inline-editДаДа
Карточка данныхПревью сделки, разбивка метрикиДаОпционально
ТаблицаКороткий список операций (без горизонтального скролла)ДаОпционально
  • Header добавляется, когда контенту нужна метка/группировка, форма содержит несколько полей, или ширина поповера больше 320px.
  • Footer добавляется, когда внутри есть форма или выбор, требующий подтверждения, либо деструктивное действие. Только информационный контент — без Footer.
  • Крестик размещается в правом верхнем углу Header (если Header есть); если Header нет — компактный ✕ можно разместить в правом верхнем углу Body.
  • Порядок кнопок Footer — как у Modal: справа Secondary + Primary (Primary крайний правый), слева — служебная информация/ссылки, никогда не действия.
  • Поповер — контейнер-примитив. Конкретные содержательные варианты (Popover/Легенда, Popover/Форма правки) стоит документировать отдельными паттернами, а не плодить новые компоненты.

Состояния

Открывается по клику/Enter/Space на триггере. Закрывается пятью равнозначно обязательными способами — ни один нельзя убрать.

Способ закрытияОбязательность
Кнопка ✕ внутри поповераОбязательно
Клик вне поповераОбязательно
EscОбязательно — фокус возвращается на триггер
Tab за последний интерактивный элементОбязательно — поповер не модальный, фокус не заперт внутри
Кнопка-действие в Footer, завершающая задачуКонтекстно

Запрещено: автозакрытие по таймеру и открытие сразу при загрузке страницы — пользователь теряет контроль (Nielsen, эвристика №3). Одновременно может быть открыт только один поповер — открытие нового закрывает предыдущий (демо ниже). Возврат фокуса на триггер при любом закрытии — требование WCAG 2.1.2.

Позиционирование

4 стороны × 3 выравнивания стрелки = 12 позиций — тот же приём, что у Tooltip. Стрелка (опциональна, см. «Анатомия») всегда направлена на триггер.

Авто-flip у края контейнераНовое

Если предпочтительному выравниванию не хватает места по горизонтали до края — поповер перевыравнивается на противоположный край (start ↔ end), стрелка остаётся направленной на триггер. Переключите положение триггера.

Рекомендации

Поповер — лёгкая контекстная надстройка над триггером, а не самостоятельный экран.

Не так
Кнопка «Сохранить» размещена в Header рядом с заголовком — пользователь не понимает, действие это или метка.

Header — только заголовок и ✕. Все действия — в Footer.

Так
Заголовок и ✕ в Header, кнопки «Отмена» и «Применить» — в Footer справа.

Однозначное разделение: контекст сверху, действия снизу.

Не так
Форма из 8 полей с обязательной валидацией — засунута в поповер шириной 400px с прокруткой на 3 экрана.

Обязательная многошаговая форма — это Modal, а не Popover.

Так
Поповер — быстрое редактирование 1–3 полей (тег, срок, статус); полноценное редактирование сущности — в Modal.

Poповер — для лёгких, необязательных к завершению действий.

Доступность

Триггер получает aria-haspopup="dialog", aria-expanded и aria-controls, указывающий на поповер. Контейнер — role="dialog" и aria-modal="false" (не блокирует страницу, в отличие от Modal), aria-labelledby — на заголовок, если Header есть. При открытии фокус переходит на первый интерактивный элемент (или на ✕, если тело нестатично из интерактивных элементов); фокус не заперт внутри — Tab с последнего элемента уводит дальше по странице и закрывает поповер (в отличие от Modal с focus trap).

Клавиатура

Enter / SpaceОткрывает поповер, когда фокус на триггере
Tab / Shift+TabПеремещение по фокусируемым элементам внутри; с последнего/первого — уходит из поповера и закрывает его (без focus trap)
EscЗакрывает поповер, фокус возвращается на триггер

Скрин-ридер

  • триггер объявляет своё состояние через aria-expanded;
  • заголовок Header связан с контейнером через aria-labelledby;
  • ошибка загрузки контента объявляется через role="alert";
  • контент страницы за поповером не получает aria-hidden/inert — в отличие от Modal, Popover не блокирует остальной интерфейс.

Цвета

Компонент использует только семантические токены. Поверхность и тень — те же, что у соседних floating-поверхностей (Menu, DropdownList): --bg-popup, --radius-popup, тень --elevation-2.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки.

Поведение · псевдокод (framework-agnostic) 
// 1. Открытие: единственный открытый поповер в приложении — глобальный реестр.
//    Перед показом нового — закрыть предыдущий открытый (если есть).
let openPopover = null;
function open(popEl, triggerEl):
  if (openPopover && openPopover !== popEl) close(openPopover.trigger)
  position(popEl, triggerEl)             // см. п.2
  popEl.classList.add('is-open')
  triggerEl.setAttribute('aria-expanded', 'true')
  focusFirstFocusable(popEl) or focusCloseButton(popEl)
  openPopover = { pop: popEl, trigger: triggerEl }

// 2. Позиционирование (пересчитывать на показе, resize, scroll):
//    взять getBoundingClientRect() триггера и поповера;
//    top/bottom — y = край триггера ± gap(8px); left/right — аналогично по X;
//    align center/start/end сдвигает вторую ось (start/end — отступ под стрелку:
//    18px для top/bottom, 14px для left/right)

// 3. Авто-flip: если предпочтительному align не хватает места до края
//    контейнера/viewport — переключить на противоположный (start ↔ end);
//    если не хватает места самой предпочтительной СТОРОНЕ (placement) —
//    попробовать противоположную, затем перпендикулярные (как у Tooltip)

// 4. Закрытие — 5 равнозначных способов, ни один не убирать:
function onOutsideClick(e): if (!popEl.contains(e.target) && e.target !== triggerEl) close()
function onEsc(): close(); triggerEl.focus()
function onTabOut(): close()             // фокус НЕ заперт — Tab с последнего элемента уводит дальше
function onCloseButtonClick(): close()
// + опционально: клик по кнопке-действию в Footer, если она завершает задачу

function close():
  popEl.classList.remove('is-open')
  triggerEl.setAttribute('aria-expanded', 'false')
  openPopover = null

// 5. Тень у шапки/подвала — по scrollTop контентной зоны, идентично Modal:
onBodyScroll():
  head.classList.toggle('is-scrolled', body.scrollTop > 0)
  foot.classList.toggle('is-scrolled', body.scrollTop + body.clientHeight < body.scrollHeight - 1)

// 6. prefers-reduced-motion: анимация появления/исчезновения отключается —
//    поповер появляется/скрывается мгновенно (см. styles/popover.css)

Это не готовый сниппет для вставки — алгоритм поверх вашего стека. Рабочий референс на vanilla JS — функции buildPopover и placePop в popover.page.js.

Рекомендуемый API компонента (React)

Форма пропсов — предложение, а не обязательный контракт. Тело — просто children, компонент не диктует его структуру.

Popover.types.ts 
type PopoverWidth = 's' | 'm' | 'l' | 'xl' | 'max';
type PopoverPlacement = 'top' | 'bottom' | 'left' | 'right';
type PopoverAlign = 'start' | 'center' | 'end';

interface PopoverFooterAction {
  label: string;
  onClick(): void;
  disabled?: boolean;
}

interface PopoverProps {
  open: boolean;
  onOpenChange(open: boolean): void;
  trigger: React.ReactElement;          // клонируется с aria-haspopup/aria-expanded/aria-controls
  title?: string;                     // если задан — рендерит Header с заголовком и ✕
  width?: PopoverWidth;                 // по умолчанию 'm' (320px)
  placement?: PopoverPlacement;         // по умолчанию 'bottom'; авто-flip при нехватке места
  align?: PopoverAlign;                 // по умолчанию 'start'
  arrow?: boolean;                    // по умолчанию false — как у Menu/DropdownList
  offset?: number;                    // зазор от триггера в px, по умолчанию 8
  children: React.ReactNode;            // Body — произвольный контент
  primaryAction?: PopoverFooterAction;   // правый Primary — рендерит Footer
  secondaryAction?: PopoverFooterAction; // правый Secondary
  footerLeft?: React.ReactNode;         // служебная информация/ссылка слева — НЕ действие
  closeOnOutsideClick?: boolean;       // по умолчанию true, отключать запрещено
  closeOnEsc?: boolean;                // по умолчанию true, отключать запрещено
}
