Entity

Компонент для отображения любых объектов — компаний, людей, файлов, метрик — в тайлах, списках и формах. Собственного фона нет: тянется на всю ширину контейнера и выравнивается по левому краю.

Использование

Используется для отображения любых объектов — как в списках, так и по отдельности. Примеры объектов: пользователи, компании, метрики компании, файлы.

Анатомия

Entity — композитный компонент. Максимальная композиция (EntityUniversal) собирается из вложенных дизайнерских подкомпонентов, которые отдельно в продукте не публикуются.

  • Не имеет собственного фона — наследует фон контейнера (тайл, карточка, строка таблицы).
  • Ведущий элемент и текстовый блок выровнены по центру по вертикали; блок действий — по верху.

Не применимо: компонент существует в единственной конфигурации, отдельных вариантов нет.

Label не переносится в 3+ строки — усекайте, а подробности выносите в подзаголовок или Tooltip.

Corner cases

  • Нет значения — вместо Label показывается «—» (.entity--empty), цвет inactive.
  • Длинное значение — .entity__label--truncate усекает в одну строку; наведение открывает Tooltip с полным текстом.
  • Много подзаголовков — список клэмпится до 2 строк, остаток сворачивается в счётчик +N (max +99).
  • Объект удалён / недоступен — .entity--error: иконка в тоне error, Label зачёркнут.
  • Действий больше 2 — не допускается: сверните в кебаб-меню или пересмотрите паттерн.

Поведение

Не применимо: специфического поведения (позиционирование, анимация, адаптивность) у компонента нет.

Доступность

  • Кликабельный тайл (.entity--interactive) — role="button" или ссылка, tabindex="0", реакция на Enter/Space; фокус-обводка 2px solid var(--primary).
  • Выбор из списка — реальный чекбокс внутри .entity__lead; клик по всей строке переключает выбор, состояние — aria-selected/aria-checked.
  • Ведущая иконка / аватар — декоративны: aria-hidden="true"; смысл несёт текст Label.
  • Иконки-кнопки и закрытие — обязателен aria-label (кнопки без текста).
  • Усечённый Label — полное значение доступно через Tooltip и title/aria-label.
  • Skeleton — на контейнере aria-busy="true"; после загрузки заменяется реальным содержимым.
  • Счётчик +N — озвучивается как «и ещё N»; полный список доступен по клику/во всплывающем элементе.
  • prefers-reduced-motion — шиммер skeleton отключается.

Поведение (framework-agnostic)

// 1. Усечение Label + Tooltip // Label в одну строку (--truncate). Если scrollWidth > clientWidth — // показать Tooltip/title с полным значением. Иначе Tooltip не нужен. if (labelEl.scrollWidth > labelEl.clientWidth) attachTooltip(labelEl, fullText) // 2. Свёртка подзаголовков в +N // Рендерим все подзаголовки; список клэмпится CSS-ом до 2 строк. // Счётчик +N считается на данных, не по факту переполнения: hidden = subheaders.length - visibleCount // visibleCount по дизайну more = min(hidden, 99) // максимум +99 if (more > 0) render(+{more}) // 3. Кликабельный тайл // role=button|link, обрабатывать Enter/Space; клик по вложенным // кнопкам действий не должен всплывать до onSelect (stopPropagation). // 4. Выбор из списка // Клик по всей строке ⇢ toggle(checkbox); клавиатура — Space. // Состояние: aria-selected на строке + checked на input. // 5. Skeleton // Пока data == null → рендер .entity--skeleton (aria-busy). // Ширины плейсхолдеров — фиксированные (head 42%, label 68%, sub 88%). // 6. Действия (max 2) // Если действий > 2 — свернуть в кебаб-меню (ContextMenu). // Порядок кнопок = порядку в заголовке страницы.
