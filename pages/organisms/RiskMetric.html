RiskMetric

Компонент для обозначения уровня надёжности контрагентов: рейтинг (число) и зона проблемности (цвет). Собран из Chip (ReadOnly, S, pill) и Popover. Popover_RiskMetric — неотъемлемая часть компонента, а не опциональная зависимость: клик по иконке-информеру всегда открывает карточку с деталями расчёта. Собственных состояний наведения или клика по самому чипу нет.

Конструктор

Соберите метрику: зона проблемности, рейтинг контрагента и состояние поповера. Кликните по иконке-информеру в превью, чтобы открыть Popover_RiskMetric.

Использование

RiskMetric применяется в таблицах и списках контрагентов — компактная колонка с рейтингом и зоной, без необходимости открывать карточку контрагента для этих двух показателей. Клик по информеру раскрывает даты расчёта и текстовые риск-атрибуты, не уводя пользователя со страницы.

  • Popover_RiskMetric — неотъемлемая часть компонента. Это Popover шириной --pop-w-m (320px): Header — заголовок «Рейтинг и зона проблемности» + ✕; Body — два серых блока подкачки (Зона / Рейтинг), каждый с датой расчёта, и два текстовых поля (Риск-сегмент, Риск-профиль). Footer у RiskMetric нет — контент только информационный, подтверждать нечего.
  • Одновременно открыт только один поповер — как и в базовом Popover: открытие нового закрывает предыдущий (демо — раздел «Состояния»).
  • Компонент не диктует ширину колонки — чип растёт по контенту (число 1–2 цифры + иконка), высота фиксирована размером Chip S.

Компоненты подкачки поповера

Body поповера собран из переиспользуемых блоков. Блок «Зона» и блок «Рейтинг» — серые контейнеры из двух строк (значение + дата расчёта); текстовые поля «Риск-сегмент» и «Риск-профиль» — простые label-значение. Ниже — все варианты наполнения.

Текстовые поля

Анатомия

Чип не имеет собственных интерактивных состояний — единственный интерактивный элемент внутри него — кнопка-информер, открывающая Popover.

  • Chip — ReadOnly, S, pill (chip--rounded). Метка — рейтинг (число) или «—». Тон заливки — по зоне: зелёная/watchlist — светлые, красная/чёрная — solid (белый текст); без зоны — Outline дефолтного цвета
  • .chip__info — кнопка-информер (не span, а настоящий <button>): открывает Popover. Отсутствует, если нет ни рейтинга, ни зоны
  • Popover → Header — заголовок «Рейтинг и зона проблемности» + ✕
  • Popover → Body: блоки подкачки — два серых блока «Зона проблемности» и «Рейтинг контрагента», у каждого строка «Дата расчета». Значение зоны окрашено цветом зоны
  • Popover → Body: текстовые поля — «Риск-сегмент» и «Риск-профиль», разделены отступом (без Divider). Footer отсутствует

Варианты

Не применимо: компонент существует в единственной конфигурации, отдельных вариантов нет.

Размеры

У компонента нет пропса размера — используется единственный фиксированный размер Chip S (24px), подобранный для плотных таблиц контрагентов. Ширина растёт по контенту: число (1–2 цифры) + иконка-информер 16px.

ПараметрЗначение

Popover_RiskMetric фиксирован на шаге --pop-w-m (320px) — стандартная ширина для заголовка и коротких строк данных; см. шкалу размеров Popover.

Контент

Число отвечает за рейтинг контрагента: целое от 1 до 26. Цвет отвечает за зону проблемности — четыре зоны риска, от зелёной до чёрной.

  • Рейтинг — целое число 1–26. Если данных нет — на месте числа длинное тире «—».
  • Зона — если данных нет, чип не имеет заливки, только бордер дефолтного цвета (Outline, StSystem).
  • Ни рейтинга, ни зоны — чип с дефолтным бордером, тире вместо числа, иконка-информер отсутствует — раскрывать нечего, чип становится чисто декоративным.
  • Даты расчёта в Popover — формат ДД.ММ.ГГГГ, отдельно для зоны и для рейтинга (пересчитываются независимо).
  • Риск-сегмент и Риск-профиль — свободный текст без ограничения символов; перенос по словам, лимита строк нет (см. «Предлагаемые дополнения» — сворачивание длинного текста).

Поведение

Не применимо: специфического поведения (позиционирование, анимация, адаптивность) у компонента нет.

Состояния

Четыре комбинации наличия данных, каждая — по четырём зонам. Rate и None не зависят от зоны, поэтому показаны одной ячейкой.

Один поповер одновременно

Откройте информер у второй строки — поповер первой закроется автоматически (правило унаследовано от Popover).

Загрузка и ошибка

Данные зоны/рейтинга могут ещё не быть загружены к моменту клика по информеру, либо запрос может завершиться ошибкой — Body переходит в состояние шиммера или role="alert", Header (заголовок + ✕) остаётся на месте.

Предлагаемые дополнения

Чего не хватает в текущей спеке — три корневых случая, которые почти наверняка встретятся в проде. Собраны на существующих токенах, не требуют новых цветов.

Доступность

Сам чип не в таб-порядке (не несёт собственного взаимодействия) и получает описательный aria-label («Риск-метрика. Рейтинг 26, зона проблемности — красная.») — этого достаточно для скрин-ридера, даже если визуально видна только цифра и иконка. Единственный фокусируемый элемент — кнопка-информер.

Клавиатура

TabПереводит фокус на кнопку-информер (если она есть) — сам чип и его текст из таб-порядка исключены
Enter / SpaceОткрывает Popover_RiskMetric, когда фокус на информере
EscЗакрывает поповер, фокус возвращается на информер
Tab из поповераС кнопки ✕ (единственный интерактивный элемент внутри) уводит дальше по странице и закрывает поповер — без focus trap, как у Popover

Скрин-ридер и цвет

  • Кнопка-информер — aria-haspopup="dialog", aria-expanded, aria-controls;
  • заголовок Popover связан контейнером через aria-labelledby;
  • ошибка загрузки — role="alert", загрузка — aria-busy="true" на Body;
  • зона никогда не кодируется только цветом — рядом всегда текстовое название («Зелёная», «Watchlist»…), и в чипе (число), и в Popover;
  • текст на зонах (*-dark токены) подобран для контраста AA на соответствующей заливке (*-light).

Типографика

Весь текст поповера — гарнитура SB Sans Text; выделенное значение (рейтинг) — strong.

Цвета

Только токены Local (StGreen/StOrange/StRed/StGrey/StSystem, colors.css). Зелёная и Watchlist — светлая заливка (тон *-light + тёмный текст). Красная и чёрная — solid-заливка (базовый тон StRed/StGrey + белый текст --c-mgrey-50) для читаемости на насыщенном фоне. Без зоны — Outline (StSystem). Новых цветов не вводится.

Для разработчиков

Хендофф-пакет: точные размеры, эталонная структура DOM, поведение и рекомендуемый API. Не привязан к стеку — команда, которая собирает компонент на React, берёт отсюда нужное для вёрстки.

Точные размеры (redline) — измерено на живом экземпляре через getComputedStyle

// 1. Резолвинг вида чипа из данных:
function resolveChip(rating, zone):
  hasZone = zone != null
  hasInfo = hasZone || rating != null
  label = rating != null ? String(rating) : '—'
  tone  = hasZone ? ZONE_TONE[zone] : null      // null → chip--outline; красная/чёрная → *-solid
  return { label, tone, showInfoButton: hasInfo, rounded: true }

// 2. Открытие/позиционирование — идентично Popover (см. placePop в popover.page.js),
//    с одним отличием: координаты считаются от самого .pop-anchor (он тесно облегает
//    кнопку-информер), а не от внешнего контейнера — так позиционирование остаётся
//    корректным независимо от того, где чип оказался в таблице.
let open = null   // { pop, btn } — глобальный на страницу, как у Popover
function onInfoClick(btn, pop, anchor):
  if (open && open.pop !== pop) close(open)
  position(pop, anchor, btn)             // gap 8px, align 'start'
  pop.classList.add('is-open')
  btn.setAttribute('aria-expanded', 'true')
  open = { pop, btn }

// 3. Закрытие — 3 равнозначных способа (Footer с действиями отсутствует):
//    ✕ / клик вне / Esc. Фокус при Esc возвращается на кнопку-информер.

// 4. Состояния Body — взаимоисключающие:
//    loading  → aria-busy="true" + skeleton вместо блоков
//    error    → role="alert" + сообщение, Header остаётся кликабельным (✕ работает)
//    default  → 2 × rm-block (Зона / Рейтинг, каждый со своей датой расчёта)
//               + 2 × rm-field (Риск-сегмент / Риск-профиль), разделены отступом (без Divider)
