TableFilter

TableFilter — панель фильтра над таблицей. Кнопка «Фильтр» открывает модалку с параметрами фильтрации; применённые параметры возвращаются в бар отдельными чипами с возможностью снять каждый по отдельности или сбросить все разом. Когда чипов больше, чем помещается в ряд, лишние сворачиваются в счётчик «+N», а бар разворачивается в несколько рядов по шеврону.

Конструктор

Соберите бар: применён фильтр или нет, сколько параметров выбрано, свёрнут бар в один ряд или развёрнут. Модалку фильтра с сохранёнными пресетами можно открыть в разделе «Поведение».

Использование

TableFilter отвечает за фильтрацию содержимого таблицы по нескольким параметрам сразу. Панель живёт в шапке таблицы: слева — кнопка вызова модалки, справа от неё — применённые параметры. В отличие от инлайн-поиска (одно поле над колонкой) фильтр собирает сложное условие из многих полей и сохраняет его набором.

Когда использовать
  • таблица фильтруется сразу по нескольким параметрам (тип, деск, продукт, период);
  • условие фильтра нужно показывать и снимать по частям;
  • частые наборы условий имеет смысл сохранять как пресеты.
Когда не использовать
  • фильтр по одному полю — достаточно инлайн-поиска или Select в шапке колонки;
  • переключение вьюх, а не сужение данных — это Tab или SegmentControl;
  • список без табличной структуры — фильтр-панель привязана к таблице.

Рекомендации

Бар остаётся читаемым, пока чипов немного; за пределом одного ряда работает сворачивание.

Так
Чип-параметр объединяет категорию и её значения: «ТБ: ЦА, МБ, СРБ». Один чип — один параметр модалки.

Пользователь снимает целый параметр одним крестиком, а не по значению.

Не так
Каждое значение вынесено отдельным чипом («ЦА», «МБ», «СРБ», «СЗБ») — бар мгновенно переполняется и теряет структуру.

Дробить параметр на значения-чипы стоит, только если модалка позволяет снимать значения по отдельности.

Так
Кнопка сброса появляется только когда есть что сбрасывать — отдельной кнопкой рядом с «Фильтр».

Нет применённых параметров — нет кнопки сброса и чипов, только одиночная кнопка «Фильтр».

Не так
Бар растянут на 4 ряда чипов по умолчанию и занимает половину экрана над таблицей.

По умолчанию бар свёрнут в один ряд; развёрнутое состояние — по явному действию пользователя.

Анатомия

Слева — кластер-триггер (кнопка «Фильтр» и, при активном фильтре, отдельная соседняя кнопка сброса). Справа — чиплист применённых параметров, а при переполнении — хвост со счётчиком «+N» и шевроном-аккордеоном.

  • Кнопка «Фильтр» — Button Outline XS с ведущей воронкой; открывает модалку
  • Сброс фильтров — отдельная кнопка рядом с «Фильтр», снимает все параметры
  • Чиплист — контейнер применённых параметров с авто-переносом
  • Чип-параметр — Chip Edit S «Категория: значения» с крестиком удаления
  • Счётчик «+N» — Badge Text XS, число скрытых при сворачивании чипов
  • Шеврон-аккордеон — IconButton Neutral S, разворачивает/сворачивает бар

Варианты

Две оси конфигурации: облик кнопки-триггера и состояние бара в целом.

Кнопка фильтра

Фильтр не применён — одиночная кнопка. Применён — рядом появляется вторая, самостоятельная кнопка сброса (Button Outline S icon-only), не сросшаяся с первой.

Обе кнопки — обычный компонент Button (Outline XS), поставленные рядом с зазором 8px; углы каждой скруглены полностью, общего шва между ними нет.

Состояние бара

Три состояния из исходного компонента: без фильтра, с фильтром в один ряд (лишнее свёрнуто в «+N»), с фильтром в несколько рядов (развёрнуто).

Размеры

Бар наследует размеры от вложенных компонентов. Базовая плотность — Button/IconButton S и Chip S; высота бара определяется высотой ряда чипов.

ЭлементКомпонент · размерВысотаКомментарий

Значения высот измеряются на живом экземпляре (см. «Для разработчиков»). Зазор между чипами, а также между последним чипом и хвостом («+N» + шеврон) — 4px; кластер-триггер отделён от чиплиста дополнительным отступом 4px (итого 8px).

Контент

Правила наполнения чипов и счётчика:

  • Формат чипа — «Категория: значения через запятую» («Дески: RE, TMT, CND»). Категория совпадает с меткой поля в модалке.
  • Переполнение чипа — длинный текст обрезается многоточием по максимальной ширине чипа (320px из Chip), полный текст показывается тултипом при наведении.
  • Один параметр — один чип. Значения внутри параметра не выносятся в отдельные чипы, если модалка не позволяет снимать их по отдельности.
  • Счётчик «+N» — целое число скрытых при сворачивании чипов, без пробелов и знаков («+99»). Появляется только в свёрнутом состоянии при переполнении ряда.
  • Кнопка «Фильтр» — существительное без счётчика; количество активных параметров передают чипы, а не подпись кнопки.

Поведение

Открытие модалки, переполнение чиплиста, сброс, сохранённые фильтры и кнопка сброса.

Открытие модалки фильтра

Клик по «Фильтр» открывает модалку (Modal) с вертикальными табами-разделами слева и полями справа. Подвал: «Сохранить фильтр» слева, «Очистить» и «Применить» справа. «Применить» закрывает модалку и возвращает выбранные параметры в бар чипами.

Живой демо-мокап: вертикальные табы, поля, сохранённые пресеты и вложенные диалоги сохранения/удаления.

Переполнение чиплиста

По умолчанию бар свёрнут в один ряд. Скрипт измеряет, сколько чипов помещается, и прячет лишние; наружу выносится счётчик «+N» и шеврон вниз. Клик по шеврону разворачивает бар — чипы переносятся на несколько рядов, шеврон меняется на «вверх». Повторный клик сворачивает обратно.

Сброс фильтров

Крестик на чипе снимает один параметр (и обновляет данные таблицы). Кнопка сброса (отдельная кнопка рядом с «Фильтр») снимает все параметры разом и убирает чиплист — бар возвращается к одиночной кнопке «Фильтр». Кнопка сброса появляется только при наличии хотя бы одного применённого параметра.

Сохранённые фильтры (пресеты)

Текущий набор параметров сохраняется как именованный пресет (до 5 штук). Пресеты живут в отдельном табе модалки: применить, переименовать замену или удалить. Сохранение с занятым именем открывает вложенный диалог подтверждения замены; удаление — вложенный диалог подтверждения. Пустой список показывает плейсхолдер.

Кнопка сброса после применения

Как только применён хотя бы один параметр, рядом с «Фильтр» появляется вторая кнопка — самостоятельный Button Outline XS icon-only с иконкой filter-reset, сбрасывающий все параметры. Это не Split Button — кнопки не сращены, у каждой полное скругление и обычный зазор 8px.

Состояния

Интерактивные элементы бара наследуют состояния от Button, IconButton и Chip; у самого бара есть отдельное состояние заблокированности.

Доступность

Бар — role="group" с aria-label="Фильтр таблицы". Кнопка «Фильтр» — aria-haspopup="dialog" и aria-expanded, связана с модалкой. Шеврон-аккордеон — aria-expanded и aria-controls на чиплист. Каждый чип-параметр фокусируем, крестик удаления озвучивается как «Убрать {категория}» и доступен по Backspace/Delete.

Клавиатура

Enter на «Фильтр»Открывает модалку фильтра
TabПеремещение по чипам и кнопкам бара как по обычным фокусируемым элементам
Backspace / DeleteСнимает сфокусированный чип-параметр, фокус переходит на соседний
Enter на шевронеРазворачивает / сворачивает бар

Скрин-ридер

  • счётчик «+N» дублируется доступным текстом «ещё N параметров»;
  • кнопка сброса имеет aria-label="Сбросить фильтры";
  • шеврон-аккордеон объявляет состояние через aria-expanded;
  • снятие чипа объявляется, число оставшихся параметров пересчитывается.

Типографика

Собственной типографики бар не вводит — текст наследуется от вложенных компонентов.

Подпись кнопки «Фильтр»SB Sans Text — токены Button (S)
Текст чипа-параметраSB Sans Text — Body S (Chip S), letter-spacing 0
Счётчик «+N»SB Sans Text — Badge XS (12px), tabular-nums, цвет Text_Inactive

Цвета

Только семантические токены. Собственный цвет бара — тонирование ведущей воронки кнопки в --primary; всё остальное берётся из Button, IconButton и Chip.

Для разработчиков

Хендофф-пакет: точные размеры и поведение переполнения/сброса/пресетов. Не привязан к стеку — команда, собирающая компонент на React, берёт отсюда нужное для вёрстки.

// 1. Триггер: фильтр не применён → одиночная кнопка «Фильтр» (Outline XS).
//    Применён ≥1 параметр → рядом отдельная кнопка сброса (Outline XS icon-only, filter-reset).
//    Кнопки не сращены — каждая обычный Button с полным скруглением.

// 2. Применение фильтра (Применить в модалке):
//    params = [{ key, label, values[] }]
//    для каждого param → чип «label: values.join(', ')» с крестиком.
//    записать params в состояние таблицы, перезапросить данные, закрыть модалку.

// 3. Переполнение чиплиста (свёрнутый бар — один ряд):
measureOverflow():
  показать все чипы, снять nowrap
  rowTop = chips[0].offsetTop
  hidden = чипы, у которых offsetTop > rowTop  // не поместились в первый ряд
  если hidden.length > 0 и бар свёрнут:
    скрыть hidden, показать хвост «+{hidden.length}» + шеврон вниз
  иначе: скрыть хвост
  пересчитывать в ResizeObserver контейнера

// 4. Аккордеон: клик по шеврону → toggle expanded.
//    expanded=false → tfilter--collapsed (nowrap, overflow hidden), шеврон chevron-down
//    expanded=true  → перенос на N рядов, шеврон chevron-up, aria-expanded=true

// 5. Снятие параметра:
//    крестик чипа ИЛИ Backspace/Delete на сфокусированном чипе → убрать param,
//    перенести фокус на соседний чип, пересчитать overflow. Список пуст → скрыть
//    сброс и чиплист, вернуть одиночную кнопку «Фильтр».

// 6. Сброс всех: клик по сбросу → очистить params, свернуть бар,
//    вернуть одиночную кнопку.

// 7. Пресеты (до 5):
//    save(name): если name занято → nested confirm «Заменить пресет?»; иначе push.
//    apply(preset): params = preset.params → см. п.2.
//    remove(preset): nested confirm «Удалить пресет?» → удалить из списка.
//    список пуст → плейсхолдер.

Это алгоритм поверх вашего стека, не готовый сниппет. Рабочий референс на vanilla JS — функции buildBar и measureOverflow в table-filter.page.js.

Что предлагаем добавить

Сверяясь с Material Design и практиками других ДС, документации бара недостаёт ряда кейсов. Все собраны на существующих токенах и компонентах.

  • Быстрые фильтры — ряд чипов-тоглов (Chip selected) перед кнопкой «Фильтр» для 2–3 самых частых условий без открытия модалки.
  • Пустой результат — подсказка «Ничего не найдено» с ссылкой «Сбросить фильтры» под таблицей, когда условие ничего не вернуло.
  • Счётчик результатов — опциональная подпись «Найдено: N» справа от бара.
  • Индикатор пресета — если применённые параметры совпадают с сохранённым пресетом, показывать его имя в кнопке «Фильтр».
