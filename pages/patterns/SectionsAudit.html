Аудит: дизайн-система из коробки

Тип: Аудит / RFCОбновлено: 21.07.2026Покрытие: все компоненты ДС

Рабочая память инициативы: возвращаясь к вопросу, начинай с этой страницы. Цель — чтобы потребитель, подключив ДС, получал рабочий компонент (позиционирование, открытие, клавнавигация), а не мёртвую вёрстку. Сейчас переиспользуемое поведение и композиционные стили многих компонентов живут только на странице-витрине (в *.page.js внутри playground-обёрток и в inline-<style>), а не в глобальном рантайме/CSS.

Фаза 3

В чём проблема

Проверил все scripts/*.page.js и inline-стили страниц. Поведение витрины написано заново на каждой странице внутри DOMContentLoaded + playground-IIFE и никуда не экспортируется. Композиционные стили (как .rm-block у RiskMetric) лежат прямо в <style> страницы.

Симптомы

  • Экран-эталон 02-Deal-v2 подключает только icons-data.js + ds-icons.js + screens-chrome.js. Рантайма компонентов нет — все состояния (открытый поповер, свёрнутый фильтр) нарисованы руками.
  • Реальное поведение заперто в витрине. placePop() + авто-flip + single-open (Popover), createSplitPane() drag/min-max (Splitter), bindRailTooltips() (NavPanel) — рабочий код, но внутри *.page.js, недоступен потребителю.
  • Композиционный CSS копипастится. .rm-block / .rm-field RiskMetric нет в styles/ — они инлайн на странице и уже скопированы в экран 02. Любая правка расходится в двух местах.
  • Сборщик экрана не знает возможностей. В чит-шите нет раздела «умеет из коробки»: какой скрипт подключить и какие data-атрибуты включают поведение.

Образец, к которому идём

  • ds-icons.js — самоинициализация: <i data-icon="…"> → SVG в currentColor, вызов dsIcons.apply(). Подключил — работает.
  • ds-illustrations.js — тот же принцип для <span class="illu" data-illu>.
  • input-kit.js (window.DSInputKit) — общий рантайм-фабрика для всех Input*. Логика поля вынесена из витрины, страница лишь вызывает makeInput().
  • ds-datepicker.js (window.DSDatePicker) — рантайм календаря: makeCalendar / openPicker + автоподключение кнопки-календаря полей InputDate / InputDateRange. Образец Фазы 2 → сразу из коробки.

Итог для каждого компонента: поведение → общий scripts/ds-<comp>.js (инициализация по классам/data-атрибутам, без привязки к DOM витрины); композиция-стили → глобальный styles/<comp>.css.

Матрица «из коробки»

Каждый компонент ДС: нужен ли рантайм-поведение, где оно живёт сейчас, где живёт композиционный CSS, доступно ли это потребителю без до-сборки, и что вынести.

page.js поведение заперто в витрине inline CSS инлайн на странице нет рантайма нет вовсе общий ✓ уже вынесен ✗ не из коробки ◐ частично ✓ из коробки

План экстракции

По компоненту за раз, проверка каждого — на живом экране (мокапе), а не на витрине.

  1. Аудит. Выписать, у каких компонентов поведение / композиция-стили живут только на витрине (эта таблица).
  2. Экстракция поведения. Переиспользуемую логику из *.page.js — в scripts/ds-<comp>.js: самоинициализация по классам/data-атрибутам, публичный API там, где нужен императивный вызов (Toast, SnackBar, Modal). Страница-витрина после этого лишь подключает общий скрипт + свой playground-код поверх.
  3. Экстракция стилей. Композиционные классы (как .rm-block) — в глобальный styles/<comp>.css, добавить @import в ds.css и styles.css. Убрать инлайн-дубли со страниц и экранов.
  4. Документация. В specs/_cheatsheet.md каждому компоненту — блок «умеет из коробки»: какой скрипт, какие data-атрибуты, какое поведение включается.
  5. Проверка. Собрать/пересобрать реальный экран на общем рантайме — убедиться, что поведение работает без кода на странице.

Единый контракт рантайма

// scripts/ds-popover.js — образец: самоинициализация + императивный API
(function () {
  // авто: любой [data-popover] на странице получает поведение
  function bind(trigger) { /* open/close, placePop, flip, Esc, клик-вне */ }
  document.querySelectorAll('[data-popover]').forEach(bind);
  // императивно, где нужно строить динамически:
  window.DSPopover = { bind, place, closeAll };
})();

Порядок работ

Приоритет по частоте на экранах и по «разблокировке» зависимых компонентов.

P1
Фундамент. Разблокирует зависимые компоненты и чаще всего встречается на экранах.
