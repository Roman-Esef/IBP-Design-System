> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Чип — компактный интерактивный элемент для фильтрации и группировки данных. Применяется как тег, маркер и статус, входит в состав чиплиста и InputAutocomplete. Два типа — Edit и ReadOnly, два стиля — Fill и Outline.

## Ключевые правила (из разделов страницы)
- **Использование** — Чипы используются в составе чиплиста для отображения параметров фильтра, в InputAutocomplete при множественном выборе, а также как статус-метки в таблицах и карточках.
- **Анатомия** — Чип — это капсула-контейнер с текстом и опциональными слотами. Ширина зависит от контента, высота фиксирована по размеру. Максимальная ширина — 320px; при переполнении текст уходит в многоточие.
- **Варианты · Типы** — Компонент имеет два типа. Edit — интерактивный чип, реагирует на наведение и поддерживает удаление. ReadOnly — для показа значения без редактирования (например, в ReadOnlyField) и на наведение не реагирует.
- **Варианты · Стиль** — Fill (Border + Fill) — заливка плюс обводка, базовый стиль для чиплистов и инпутов. Outline (Border) — только обводка, для более лёгкого вида на плотных поверхностях; при наведении появляется заливка.
- **Размеры** — Четыре размера. L — крупный ReadOnly-показ (зарезервирован). M и S — базовые для чиплистов и фильтров. XS — для компактных мест (плотные таблицы, инлайн-метки, значения-чипы в ReadOnlyField).
- **Состояния** — Базовые состояния чипа и их цветовые токены. При наведении меняется цвет фона. Чип в состоянии Disabled на наведение не реагирует — кроме случаев с длинными текстовками, когда показываем тултип.
- **Варианты · Ведущие элементы** — Слева к тексту можно добавить дополнительный элемент. .Chip_Content — иконка-маркер; Иконка — смысловая пиктограмма, не завязанная на тон/статус; .Chip_Badge — мини-бейдж; аватар — для чипов-персон. Справа — действие: крестик удаления или шеврон выпадающего списка (.Chip_Action).
- **Тон и статусыПредложение** — Текущая ДС описывает только нейтральный системный тон (StSystem). Поскольку чипы используются и как статусы, предлагаем тональную шкалу на базе локальных статус-рамп: фон Light, текст Dark, иконка — базовый цвет статуса. Это даёт семантику «активно / в работе / ошибка» без отдельной иконки.
- **Тон — открытый список рамп** — Тон не ограничен ситуативными именами (success/info/warning/error) — это просто самые частые алиасы локальных рамп StGreen/StLBlue/StOrange/StRed. Чип может использовать любую рампу из «Local» (colors.css), включая **StGrey** — тон `chip--dark`, добавлен для нейтрально-«чёрной» семантики (например, Чёрная зона в RiskMetric), где ситуативных error/warning недостаточно.
- **Solid-тонаНовое** — Для критичных статусов, где светлой заливки недостаточно, есть серия solid-тонов: `chip--success-solid` / `chip--warning-solid` / `chip--error-solid` / `chip--dark-solid`. Фон = базовый тон статуса (StGreen/StOrange/StRed/StGrey), текст и иконка — белые (`--c-mgrey-50`), hover — *-dark. Использует RiskMetric (Красная/Чёрная зона).
- **Info-trigger (`.chip__info`)Новое** — Третий вариант trailing-действия (наряду с крестиком удаления и шевроном списка): кнопка-иконка, открывающая Popover с деталями по чипу (использует RiskMetric). В отличие от `.chip__remove`/`.chip__dropdown` — это настоящий `<button>`, самостоятельно фокусируемый даже внутри `chip--readonly` (сам чип при этом остаётся вне таб-порядка).
- **Поведение · Удаление** — Если нужно дать пользователю управлять набором чипов, добавляется крестик. Крестик — единственная интерактивная зона помимо самого чипа; на наведении у него появляется собственный стейт-слой. Удаление доступно мышью и с клавиатуры (Backspace / Delete на сфокусированном чипе).
- **Контент · Переполнение текста** — Размер чипа зависит от контента, максимальная ширина — 320px. Если длина контента превышает максимум, текст уходит в многоточие, а при наведении показываем тултип с полной версией. Это работает и для Disabled-чипа с длинной текстовкой.
- **Скругление (Rounded)Новое** — Новые чипы объединили с маркером и добавили свойство Rounded для включения полного скругления (капсула 999px). В дизайн-системе пока не перешли на них — будем переводить после внутренних тестов нового компонента.
- **Что предлагаем добавить** — Сверяясь с Material Design и практиками других ДС, в документации недостаёт ряда состояний и кейсов. Ниже — предлагаемые дополнения; все собраны на существующих токенах.
- **Типографика** — Текст чипа набирается гарнитурой SB Sans Text. Кегль зависит от размера и совпадает с токенами Body. Letter-spacing — 0.
- **Контент** — Текстовое содержимое чипа — метка, опциональный числовой суффикс и текст в альтернативах (title/aria-label). Краткий свод правил (подробно — в «Переполнение текста» и «Правила» выше/ниже):
- **Использование · Правила** — Чип несёт короткую метку или статус. Не превращайте его в кнопку и не помещайте длинные предложения — для этого есть другие компоненты.
- **Доступность** — Чип Edit — фокусируемый элемент (tabindex="0"); крестик удаления — вторая интерактивная зона внутри него. ReadOnly-чип неинтерактивен и из таб-порядка исключён.
- **Цвета** — Компонент использует только семантические и локальные токены статусов. Базовый системный тон — на рампе StSystem; статусные тона — на локальных рампах.

### Разметка · HTML (эталонная реализация ДС)

```
<!-- Edit-чип с маркером и крестиком удаления -->
<span class="chip chip--edit chip--m chip--rounded" tabindex="0">
  <span class="chip__marker"><svg aria-hidden="true">…</svg></span>
  <span class="chip__label">Активна</span>
  <span class="chip__remove" role="button" aria-label="Удалить Активна"><svg aria-hidden="true">…</svg></span>
</span>

<!-- ReadOnly-чип: без tabindex и без крестика -->
<span class="chip chip--readonly chip--m"><span class="chip__label">Договор</span></span>

<!-- чиплист: контейнер с авто-переносом и единым зазором -->
<div class="chiplist" role="group" aria-label="Фильтры">…</div>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Hover — только для Edit и только когда не disabled: чистый CSS (:hover)

// 2. Обрезка текста: когда scrollWidth элемента > clientWidth (текст обрезан
//    ellipsis) — добавить title с полным текстом. Работает и для Disabled (chip--has-tooltip возвращает pointer-events)

// 3. Удаление: клик по chip__remove ИЛИ Backspace/Delete на сфокусированном чипе →
//    анимация исчезновения (opacity+scale) → удалить из DOM → перенести фокус на соседний чип

// 4. Переполнение чиплиста (Предложение): измерить, сколько чипов вмещается в одну строку,
//    остальные свернуть в чип-счётчик «+N», открывающий полный список по клику

// 5. Клавиатура: Tab фокусирует чип (если Edit); :focus-visible → outline 2px --primary, offset 2px

// 6. Disabled: pointer-events: none, кроме chip--has-tooltip (тултип должен работать и в disabled)
```

### Chip.types.ts

```
type ChipType  = 'edit' | 'readonly';
type ChipStyle = 'fill' | 'outline';
type ChipSize  = 'l' | 'm' | 's' | 'xs';
type ChipTone  = 'system' | 'accent' | 'success' | 'info' | 'warning' | 'error' | 'dark'; // открытый список — любая Local-рампа

interface ChipProps {
  type?: ChipType;                 // по умолчанию 'edit'
  style?: ChipStyle;                // по умолчанию 'fill'
  size?: ChipSize;                  // по умолчанию 'm'; 'l' — только readonly
  tone?: ChipTone;                  // по умолчанию 'system'
  label: React.ReactNode;
  leading?: 'marker' | 'icon' | 'badge' | 'avatar';
  count?: React.ReactNode;          // числовой суффикс
  removable?: boolean;              // только для type='edit'
  onRemove?(): void;
  dropdown?: boolean;
  infoTrigger?: boolean;            // Предложение — кнопка ⓘ, открывает Popover с деталями
  onInfoClick?(): void;
  rounded?: boolean;
  selected?: boolean;              // Предложение — тоггл-фильтр
  loading?: boolean;                // Предложение
  invalid?: boolean;                // Предложение
  disabled?: boolean;
  tooltip?: string;                 // полный текст при обрезке
  'aria-label'?: string;           // передаётся крестику удаления как «Удалить {label}»
}

interface ChiplistProps {
  children: React.ReactNode;        // Chip[]
  size?: 'm' | 's';              // влияет на зазор между чипами
  'aria-label'?: string;
}
```
