# План: SnackBar

## 1. Назначение
SnackBar — стек уведомлений в **правом верхнем углу** экрана, **поверх всех слоёв** (выше модалок; модалка с ошибкой — выше остальных модалок, но всё равно под снэкбаром). Четыре тона: Info · Warning · Error · Success. В отличие от Toast (тёмная пилюля по центру сверху, индикатор процесса, без действий) — снек цветной, с заголовком, текстом и опциональными кнопками действий. Обязательно описать разграничение Toast ↔ SnackBar в разделе «Использование» и слои: ToastBar < SnackBar < ToastLoader (сверить со specs/Toast.md, раздел «Позиционирование»).

## 3. Анатомия (DOM, имена классов условны)
```
<div class="snackbar-layer">            <!-- fixed, правый верхний угол, поверх всего; отступ от краёв 24px, зазор между снеками 12px -->
  <div class="snack snack--error" role="alert">
    <span class="snack__icon"><i data-icon="alert-circle-filled"></i></span>  <!-- 20px, выровнен по первой строке заголовка -->
    <div class="snack__body">
      <div class="snack__title">Ошибка</div>          <!-- Body S, --text-primary -->
      <div class="snack__text">Плановая дата…</div>   <!-- Body XS, --text-secondary -->
      <div class="snack__buttons">…</div>             <!-- опционально: 1 кнопка / 2 кнопки / ссылка -->
    </div>
    <button class="snack__close"><i data-icon="close"></i></button>  <!-- IconButton xs transparent, цвет тона -->
  </div>
  …ещё снеки (новый — сверху)…
</div>
```
- Иконки тона (из specs/Icons.md): Info → `Info-circle-filled`, Warning → `alert-triangle-filled`, Error → `alert-circle-filled`, Success → `check-circle-filled`. На странице ДС — SVG доставать из icons-data.js через run_script (НЕ читать файл), либо `<i data-icon>` + ds-icons.js как на экранах.
- Кнопки — переиспользовать Button из ДС (specs/Buttons.md), размер **xs**, перекрашенные в situative-палитру через переопределение CSS-переменных кнопки в контексте `.snack--<тон>` (не копировать стили кнопки). Ссылка — компонент Link.
- Actions в референсе (chevron + close) — элементы управления сворачиванием стека, см. §6.

## 4. Токены
| Роль | Info | Warning | Error | Success |
|---|---|---|---|---|
| Фон снека | `--info-bg` | `--warning-bg` | `--error-bg-light` | `--success-bg` |
| Иконка, close, кнопки | `--info` | `--warning` | `--error` | `--success` |
| Hover кнопок/close | `--info-dark` | `--warning-dark` | `--error-dark` | `--success-dark` |

- Заголовок: `--type-body-s` (Regular 400), цвет `--text-primary`.
- Текст: `--type-body-xs`, цвет `--text-secondary`.
- Радиус: токен из specs/Radius.md (взять ближайший к 8px, сверить с существующими значениями, не хардкодить).
- Тень: Elevation из specs/Elevation.md — уровень как у всплывающих слоёв (dropdown/popover), взять готовый токен, не выдумывать.
- Ширина: фиксированная **320px**.
- Паддинги/зазоры: ориентир по референсу ~16px паддинг, 8px зазор иконка↔текст; точные значения показать в redline через getComputedStyle.

## 5. Варианты и состояния (раздел «Состояния» страницы)
- Тон: info / warning / error / success.
- Контент: только title+text · +1 кнопка · +2 кнопки (вторая — transparent) · +ссылка (Link).
- Состояния кнопок/close: default / hover / active / focus. Focus — 2px outline `var(--primary)` offset 2px, как везде в ДС (единообразие важнее тона).
- Снек сам по себе не hover-интерактивен, но hover по снеку ставит авто-скрытие на паузу (см. §6).

## 6. Поведение (описать на странице + псевдокод в «Для разработчиков»)
- Позиция: fixed, top-right, охранное поле 24px; новый снек добавляется **сверху** стека.
- Слои: поверх модалок и ToastBar; под ToastLoader — согласовать со specs/Toast.md и упомянуть в обеих спеках.
- Авто-скрытие (правило от пользователя): **снек с кнопками действий — требует реакции, висит до ручного закрытия**; снек без кнопок — авто-скрытие 5с независимо от тона. Hover/focus внутри снека ставит таймер на паузу; уход курсора — таймер заново.
- Переполнение стека: видно ≤5; шестой и далее сворачиваются в плашку «+N» внизу стека; chevron-контрол (референс .SnackBar_Actions) разворачивает/сворачивает, рядом close — «закрыть все». Свёрнутые не теряются: закрытие видимого поднимает следующий из очереди.
- Дедупликация: одинаковые снеки подряд (тон+заголовок+текст) схлопываются в один со счётчиком «×N» рядом с заголовком; повтор перезапускает таймер.
- Анимации: появление — слайд слева-сверху/фейд, уход — фейд; тайминги явно в таблице (по образцу Toast); prefers-reduced-motion → только opacity.
- Клавиатура: Esc закрывает верхний снек; кнопки в снеке — в порядке табуляции.
- A11y: error/warning → `role="alert"` (assertive), info/success → `role="status"` (polite); close — `aria-label="Закрыть"`; авто-скрытие не должно убивать снек с фокусом внутри.

## 7. Страница компонента — разделы строго по порядку (Шаг 2 «обнови и проверь»)
1. **Шапка** (masthead): Версия, Обновлено.
2. **Конструктор** (первый после шапки, на pg-kit: настройки 360px слева + Splitter + превью):
   - контролы: тон (SegmentControl 4), заголовок (text), текст (text), кнопки (select: нет/одна/две/ссылка), тумблер close, счётчик ×N (тумблер); авто-скрытие показывать как вычисляемое правило (есть кнопки → «до закрытия», нет → «5с»), не отдельным тумблером;
   - превью: макет угла экрана; кнопка «Показать снек» добавляет живой снек в стек, работает лимит и сворачивание — это и есть демо поведения;
   - неактуальные контролы скрывать `.is-off`.
3. **Использование** — когда снек, когда Toast, когда Modal; таблица выбора. Правила слоёв.
4. **Анатомия** — схема с выносками: Icon / Title / Text / Buttons / Close / слот Actions стека.
5. **Размеры** — ширина, паддинги, зазоры, размер иконки, отступы стека и от краёв экрана.
6. **Контент** — правила текстовки: заголовок 1–3 слова, суть в первой фразе, глагол действия на кнопке; максимум строк/переполнение длинным текстом (corner case — показать пример).
7. **Состояния** — сетка тонов × наполнений (как в референсе .Snack), hover/focus кнопок, стек 1/2/5, свёрнутый стек «+N», снек со счётчиком ×N.
8. **Доступность** — roles, live-regions, фокус, Esc, reduced-motion, контраст текстов на цветных фонах (проверить фактически).
9. **Цвета** — таблица токенов из §4, свотчи.
10. **Для разработчиков** — redline через getComputedStyle (НЕ getBoundingClientRect на скрытых), анатомия DOM, поведение псевдокодом (очередь, лимит, авто-скрытие, пауза по hover), React API (SnackbarProvider/useSnackbar — как предложение, не контракт), справочник классов/атрибутов, copy-кнопки на всех блоках кода.

- Переполнение стека: лимит 5, свёртка «+N», «закрыть все».
