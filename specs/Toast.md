> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Тёмная пилюля-уведомление, всплывающая у верхней границы рабочей области по центру, — индикатор запуска фонового или блокирующего процесса. Два режима: ToastBar (фоновый, авто-скрытие, стек до трёх) и ToastLoader (блокирующий, со спиннером и затемнением). Ведущий слот — спиннер или статус-иконка (взаимоисключающие); тоны Neutral · Success · Error · Info.

## Ключевые правила (из разделов страницы)
- **Использование** — Toast сообщает о запуске серверного процесса, результат которого отложен во времени, либо подтверждает инициированное пользователем действие. Он не требует реакции и не содержит действий — только информирует.
- **ToastBar и ToastLoader — в чём разница** — Один компонент, два режима. Выбор — по тому, блокирует ли процесс работу пользователя.
- **Анатомия** — Тост — тёмная пилюля с необязательным ведущим слотом (спиннер или статус-иконка — взаимоисключающие) и текстом сообщения. Ведущий слот выровнен по верхней строке текста — при переносе остаётся сверху слева.
- **Ширина и перенос текста** — Ширина зависит от контента, высота фиксирована для одной строки. Максимум — 60% от ширины рабочей области; при достижении предела текст переносится, а ведущий слот прижимается к верхней строке.
- **Ведущий слот** — Слева от текста — необязательный слот: спиннер, статус-иконка или ничего. Спиннер и иконка взаимоисключают друг друга — одновременно показывается только одно. Переход «процесс → результат» — это замена спиннера на иконку во времени (см. «Переход Loading → Success»), а не оба глифа сразу.
- **Состояния и тоны** — Тост не реагирует на наведение и фокус — он информативный, не интерактивный. «Состояние» здесь — это тон результата процесса. Neutral и Loading были в исходной документации; Success и Error добавлены как недостающие финальные состояния фонового процесса.
- **Стек и переполнение** — Тосты добавляются по одному — новый появляется сверху. Одновременно показывается не более трёх. Что происходит с четвёртым при лимите — в исходной документации не описано; ниже — предлагаемое поведение.
- **Позиционирование и затемнение** — Тост появляется в отдельном слое, горизонтально — по центру рабочей области, по вертикали — у верхней границы. Вокруг всего компонента — охранное поле 24 px (сверху и по бокам), чтобы тост не касался краёв. ToastBar живёт в слое под снэкбаром и не затемняет подложку; ToastLoader — в слое над снэкбаром с затемнением 25%.
- **Анимация** — Тосты добавляются в бар по одному — линейной анимацией сверху. Удаляются фейдом вниз. При prefers-reduced-motion: reduce остаётся только смена прозрачности, спиннер замедляется до 1.6 с.
- **Тайминги** — Явные значения длительностей и кривых — чтобы поведение было воспроизводимо в коде.
- **Текстовка** — Название короткое и понятное, однозначно описывает суть процесса. Формулируйте так, чтобы вид глагола отражал режим тоста.
- **Доступность** — Тост не фокусируется и не участвует в порядке табуляции — он появляется и исчезает сам. Чтобы скринридер озвучил сообщение, тост объявляется через ARIA live-region.
- **Цвета** — Пилюля — тёмный фон StGrey (CGrey_700, #333F48), текст белый (Text_OnDark). Тень — Elevation_2. Статус-иконки берут семантические токены раздела Situative (светлые рампы, читаемые на тёмном фоне): Success — Success_Light (LightGreen_400), Error — Error_Light (Red_200), Info — Info_Light (LightBlue_400). Затемнение ToastLoader — StGrey 25%.

### Поведение · псевдокод (framework-agnostic)

```
// 1. Очередь и лимит: одновременно видно не более MAX = 3 тостов.
//    ToastBar — стек; ToastLoader — единичный и замещает весь ToastBar.
MAX = 3

function pushToast(toast):
  if toast.kind == 'loader':            // блокирующий
    clearAllBars()                     // ToastLoader замещает весь ToastBar
    showScrim(0.25); showLoader(toast) // затемнение + слой над снэкбаром
    return
  // bar: новый добавляем СВЕРХУ; при переполнении вытесняем самый ранний (нижний) без лоадера
  visibleBars.unshift(toast); animateIn(toast)  // новый сверху, linear
  if visibleBars.length > MAX:
    dismiss(oldestDismissableBar())    // нижний уходит фейдом вниз, FIFO-окно из трёх

// 2. Авто-скрытие: тост БЕЗ лоадера — по таймауту 3с; тост С лоадером —
//    только по приходящему с сервера событию (успех / ошибка).
function onShown(toast):
  if !toast.hasLoader: setTimeout(() => dismiss(toast), 3000)

// 3. Loader → Success: по событию сервера спиннер меняется на галочку,
//    тост держится ~1с и уходит фейдом вниз.
function onServerDone(toast, result):
  toast.setTone(result.ok ? 'success' : 'error')
  toast.swapSpinnerToIcon()
  setTimeout(() => dismiss(toast), 1000)

// 4. dismiss = класс .toast--leave (фейд вниз), удаление узла по animationend.
// 5. reduced-motion: только opacity, без translateY; спиннер 1.6с.
```

### Toast.types.ts

```
type ToastKind = 'bar' | 'loader';      // фоновый | блокирующий
type ToastTone = 'neutral' | 'success' | 'error' | 'info';

interface ToastOptions {
  message: string;             // текст, вид глагола по kind/tone
  kind?: ToastKind;             // по умолчанию 'bar'
  tone?: ToastTone;             // по умолчанию 'neutral'
  loader?: boolean;            // показать спиннер (у kind='loader' — всегда)
  durationMs?: number | null;  // null = ждать событие сервера; bar без лоадера — 3000
}

// императивный контроллер очереди/стека (провайдер на уровне приложения)
interface ToastController {
  show(opts: ToastOptions): ToastHandle;   // поставить в очередь / показать
  update(h: ToastHandle, patch: Partial<ToastOptions>): void; // loader→success
  dismiss(h: ToastHandle): void;
  // invariants: одновременно ≤ 3 bar; loader единичен и замещает bar
}
```
