> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Выпадающий список — всплывающая поверхность со списком опций, которая раскрывается под полем Select или InputAutocomplete. Служит для выбора значения из перечня: одиночного (подсветка выбранной строки), множественного (ведущие чекбоксы) или с поиском по мере набора (подсветка совпадения). В отличие от Context Menu — это выбор данных, а не список команд.

## Ключевые правила (из разделов страницы)
- **Использование** — Триггер — поле Select или InputAutocomplete. По клику под полем раскрывается список; его ширина равна ширине поля. Нажмите на поле в каждом примере.
- **Использование** — Выпадающий список — способ выбрать значение из перечня, когда вариантов много или место ограничено. Используйте, когда нужно:
- **Анатомия** — Список — контейнер-поповер с вертикальным перечнем опций. Опция состоит из ведущего слота (иконка или чекбокс), тела (подпись + опциональный helper) и опциональной правой части. Опции можно группировать липким заголовком.
- **Варианты · Варианты опции Dropdown_Item** — Четыре варианта опции × пять состояний. Ведущий слот — либо иконка, либо чекбокс (для множественного выбора); helper-текст добавляет вторую строку. Матрица повторяет исходный компонент.
- **Размеры** — У опции один размер. Высота строки фиксирована для одной строки; helper-текст даёт вторую строку и увеличивает высоту. Ширина контейнера равна ширине поля-триггера (не меньше 200px).
- **Контент** — Подпись опции — существительное-значение, коротко и однозначно. Правила:
- **Состояния** — Опция меняет фон при наведении и фокусе, темнеет при нажатии. Выбранная (одиночный выбор) — залита фоном «выбранной строки» и усилена начертанием. Отключённая — приглушена и не реагирует на курсор. Фокус клавиатуры (активная опция) визуально отделён от hover.
- **Системные состояния Новое** — Когда показывать нечего, вместо списка опций выводится одна системная строка. Три случая — загрузка, пустой результат и ошибка — плюс корнер-кейс автокомплита со свободным вводом («создать»).
- **Позиционирование и авто-разворот Новое** — Список выравнивается по левому краю поля, его ширина равна ширине поля. Если снизу не хватает места, список разворачивается вверх. Перетащите поле к нижнему краю области.
- **Доступность** — Контейнеру присваивается role="listbox" (для множественного выбора — aria-multiselectable="true"), опциям — role="option" с aria-selected. Поле-триггер получает role="combobox", aria-expanded и aria-controls; активная опция помечается через aria-activedescendant без потери фокуса из поля. Список полностью управляется с клавиатуры и закрывается по Esc.
- **Типографика** — Подписи набираются гарнитурой SB Sans Text. Helper и заголовок группы — Body XS.
- **Цвета** — Компонент использует только семантические токены. Фон списка — --bg-popup, hover опции — --tertiary-light, выбранная строка — --bgtable-row-focus, заливка чекбокса — --primary.

## Для разработчиков (выжимка)

### Точные размеры (redline)

Таблица рендерится на странице через getComputedStyle. Точные значения — в CSS-файле компонента (см. `css:` в шапке).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- поле-триггер связано со списком через combobox / aria-controls -->
<div role="combobox" aria-expanded="true" aria-controls="ccy">…</div>

<!-- список: role=listbox; multiselect → aria-multiselectable -->
<div id="ccy" class="ddl ddl--floating ddl--scroll" role="listbox">
  <div class="ddl__group">Популярные</div>

  <!-- одиночный выбор: выбранная опция = aria-selected -->
  <button class="ddl__item" role="option" aria-selected="true">
    <span class="ddl__item-body">
      <span class="ddl__item-label">Доллар <span class="ddl__match">США</span></span>
      <span class="ds-helper">USD · 840</span>
    </span>
  </button>

  <!-- множественный выбор: ведущий чекбокс + aria-checked -->
  <button class="ddl__item ddl__item--checkbox" role="option" aria-checked="true">
    <span class="ddl__item-check"><span class="cb__box"><span class="cb__mark">…</span></span></span>
    <span class="ddl__item-body">…</span>
  </button>

  <button class="ddl__item" role="option" aria-disabled="true">Евро</button>

  <!-- системная строка вместо опций -->
  <div class="ddl__state ddl__state--empty">…Ничего не найдено</div>
</div>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Открытие: клик/стрелка вниз/ввод в поле → показать список, aria-expanded=true.
//    Фокус остаётся в поле; активная опция помечается aria-activedescendant.

// 2. Ширина и позиция: ширину списка приравнять к ширине поля; открыть под полем
//    (gap 6px), выровнять по левому краю. Не хватает места снизу → развернуть вверх.

// 3. Клавиатура: ↑/↓ двигают активную опцию (пропуская disabled), Home/End — край,
//    Enter выбирает (multiselect — переключает), Space — переключает чекбокс,
//    typeahead по первой букве, Esc закрывает и возвращает фокус в поле.

// 4. Одиночный выбор: клик по опции → aria-selected на ней (снять с прочих),
//    записать значение в поле, закрыть список.

// 5. Множественный выбор: клик переключает aria-checked, список НЕ закрывается;
//    в поле — сводка («N выбрано» / чипы). Закрытие по клику вне и Esc.

// 6. Автокомплит: по вводу фильтровать опции, подсвечивать совпавшую подстроку
//    (.ddl__match), дебаунсить запрос; показать loading → results / empty / error.

// 7. Скролл: при переполнении контейнер прокручивается; активная опция всегда
//    подкручивается в видимую область; заголовки групп липкие (sticky).
```

### DropdownList.types.ts

```
interface Option {
  value: string;
  label: string;
  helper?: string;               // вторая строка (код, ISO, счёт)
  icon?: React.ReactNode;           // ведущая иконка (single-select)
  group?: string;                // заголовок группы
  disabled?: boolean;
}

type ListStatus = 'idle' | 'loading' | 'empty' | 'error';

interface DropdownListProps {
  options: Option[];
  multiple?: boolean;               // ведущие чекбоксы, не закрывать при выборе
  value: string | string[];         // выбранное значение(-я)
  query?: string;                    // строка поиска — подсветка совпадения
  status?: ListStatus;              // системное состояние вместо списка
  emptyAction?: { label: string; onClick: () => void }; // «создать»
  maxHeight?: number;             // включает прокрутку (по умолч. ≈7 опций)
  onChange?: (value: string | string[]) => void;
}
```

### Справочник классов и атрибутов

| Класс / атрибут | На чём | Назначение |
|---|---|---|
| .ddl | div | Контейнер-список: фон, радиус 12px, тень, min-ширина |
| .ddl--scroll | div | Прокрутка при переполнении (--ddl-max-h) |
| .ddl--floating | div | JS-позиционирование под полем; показ через .is-open |
| .ddl__item | button | Опция: ведущий слот + тело + правая часть |
| .ddl__item--checkbox | button | Множественный выбор — ведущий чекбокс, состояние в aria-checked |
| .ddl__item--wrap | button | Перенос подписи вместо многоточия |
| .ddl__item--action | button | Строка «создать» / свободный ввод (акцентная) |
| .ddl__item-check / -icon / -body / -label / -trail · .ds-helper | span | Части опции: чекбокс, иконка, тело, подпись, правая часть; helper — компонент подкачки .ds-helper |
| .ddl__match | span | Подсветка совпавшей подстроки в автокомплите |
| .ddl__group | div | Липкий заголовок группы (Body XS uppercase) |
| .ddl__divider | hr | Разделитель между группами / перед action-строкой |
| .ddl__state (--loading / --empty / --error) | div | Системная строка вместо списка опций |
| .is-hover / .is-focus / .is-active | опция | Принудительные состояния (для доков/тестов) |
| role=listbox / option · aria-selected / -checked / -disabled / -activedescendant | список, опция | Доступность: семантика списка и выбора |
