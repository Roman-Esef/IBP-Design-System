> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Контекстное меню — всплывающий список действий над объектом. Возникает по нажатию на кнопку действий (kebab «три точки» или дропдаун) в заголовке таблицы, строке таблицы или карточке. Ширина зависит от контента, набор пунктов — от объекта.

## Ключевые правила (из разделов страницы)
- **Использование** — Триггер — кнопка действий рядом с объектом. Три типичных места вызова: заголовок таблицы (действия над всей таблицей), строка таблицы (действия над строкой) и карточка (действия над сущностью). Нажмите на kebab в каждом примере.
- **Когда использовать** — Контекстное меню группирует вторичные действия над объектом, чтобы не засорять интерфейс кнопками. Используйте, когда нужно:
- **Анатомия** — Меню — контейнер-поповер с вертикальным списком пунктов. Пункт состоит из опциональной иконки, подписи и опциональной правой части (горячая клавиша или каретка подменю). Пункты можно группировать заголовком и разделять линией.
- **Типы пунктов** — Пункт меню бывает текстовым или с ведущей иконкой; справа — горячая клавиша, каретка подменю или (для меню-выбора) галочка. Деструктивные действия окрашиваются в красный.
- **Горячие клавиши** — Горячая клавиша (шорткат) — сочетание клавиш, дублирующее действие пункта. Отображается серым текстом справа от подписи и служит подсказкой-напоминанием: пользователь видит, что то же действие доступно с клавиатуры без открытия меню.
- **Контент** — Подпись пункта — глагол в форме действия («Изменить», «Погасить», «Удалить»), коротко и однозначно. Правила:
- **Состояния** — Пункт меняет фон при наведении и фокусе, темнеет при нажатии. Выбранный пункт помечается галочкой и усиленным начертанием. Отключённый — приглушён и не реагирует на курсор.
- **Позиционирование и авто-разворотНовое** — Меню открывается под триггером и выравнивается по его краю (start / end). Если снизу не хватает места, оно разворачивается вверх, а по горизонтали смещается, чтобы не выходить за границы экрана. Перетащите триггер к краям области.
- **Подменю и переполнениеНовое** — Вложенное меню (flyout) раскрывается вправо по наведению на пункт с кареткой. Когда пунктов больше, чем помещается по высоте, контейнер прокручивается, сохраняя видимой шапку экрана — избегайте меню длиннее ~10 пунктов, группируйте или используйте подменю.
- **Доступность** — Контейнеру присваивается role="menu", пунктам — role="menuitem" (или menuitemradio / menuitemcheckbox для выбора). Триггер получает aria-haspopup="menu" и aria-expanded. Меню открывается по клику и Enter/Space, полностью управляется с клавиатуры и закрывается по Esc с возвратом фокуса на триггер.
- **Рекомендации** — Держите подписи короткими и в форме действия. Меню — список команд, а не место для пояснений.
- **Типографика** — Подписи пунктов набираются гарнитурой SB Sans Text. Размер зависит от размера меню.
- **Цвета** — Компонент использует только семантические токены. Фон меню — --bg-popup, hover/focus пункта — нейтральный серый --tertiary, деструктивные действия — ситуативные токены Error.

## Для разработчиков (выжимка)

### Точные размеры (redline)

Таблица рендерится на странице через getComputedStyle. Точные значения — в CSS-файле компонента (см. `css:` в шапке).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- триггер связан с меню через aria-haspopup / aria-expanded -->
<button type="button" class="kebab" aria-label="Действия"
        aria-haspopup="menu" aria-expanded="true">⋮</button>

<!-- меню: позиционируется JS относительно триггера, скрыто до открытия -->
<div class="menu menu--floating" role="menu">
  <div class="menu__label">Действия</div>

  <button type="button" class="menu__item" role="menuitem">
    <span class="menu__item-icon"><svg…></svg></span>
    <span class="menu__item-label">Изменить</span>
    <span class="menu__item-hint">⌘C</span>
  </button>

  <!-- выбор (radio): галочка + aria-checked -->
  <button class="menu__item" role="menuitemradio" aria-checked="true">…</button>

  <hr class="menu__divider">

  <!-- деструктивное действие + отключённый пункт -->
  <button class="menu__item menu__item--danger" role="menuitem">Удалить</button>
  <button class="menu__item" role="menuitem" aria-disabled="true">Архив</button>
</div>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Открытие: клик или Enter/Space на триггере → показать меню, aria-expanded=true,
//    перевести фокус на первый пункт (roving tabindex: активный пункт tabindex=0, остальные -1)

// 2. Позиционирование (при показе и resize): взять getBoundingClientRect() триггера и меню;
//    placement=bottom → y = низ триггера + gap(6px); align start/end → левый/правый край меню к краю триггера

// 3. Авто-разворот: не хватает места снизу → раскрыть вверх; по горизонтали зажать позицию
//    внутри границ viewport/контейнера, чтобы меню не выходило за экран

// 4. Клавиатура: ↑/↓ перемещают активный пункт (пропуская disabled), Home/End — край,
//    Enter/Space активируют, → открывает подменю, ←/Esc закрывают; typeahead по первой букве

// 5. Закрытие: клик по пункту, клик вне меню, Esc или blur → скрыть, вернуть фокус на триггер

// 6. Подменю: раскрывать по наведению (с задержкой закрытия ~160мс) и по стрелке →;
//    при нехватке места справа — раскрывать влево
```

### ContextMenu.types.ts

```
type MenuAlign = 'start' | 'end';

interface MenuItem {
  id: string;
  label: string;
  icon?: React.ReactNode;           // ведущая иконка (20px M / 18px S)
  hint?: string;                   // правая часть — горячая клавиша
  danger?: boolean;                // деструктивное действие (красный)
  disabled?: boolean;              // aria-disabled, не фокусируется активацией
  selected?: boolean;              // галочка + role=menuitemradio/checkbox
  items?: MenuItem[];               // вложенное подменю
  onSelect?: () => void;
}

interface ContextMenuProps {
  trigger: React.ReactElement;      // kebab / dropdown; получает ref и aria-*
  items: MenuItem[];                // плоский список; разделитель — { divider:true }
  align?: MenuAlign;                // по умолчанию 'start'; end — выравнивание по правому краю
  placement?: 'bottom' | 'top';   // предпочтительная сторона; авто-разворот при нехватке места
  maxHeight?: number;             // включает прокрутку при переполнении
}
```

### Справочник классов и атрибутов

| Класс / атрибут | На чём | Назначение |
|---|---|---|
| .menu | div | Контейнер: фон, радиус 12px, тень, min/max-ширина |
| .menu--scroll | div | Прокрутка при переполнении (--menu-max-h) |
| .menu--floating | div | JS-позиционирование; показ через .is-open |
| .menu__item | button / a | Пункт: иконка + подпись + правая часть |
| .menu__item--danger | button | Деструктивное действие (токены Error) |
| .menu__item--wrap | button | Перенос подписи вместо многоточия |
| .menu__item--sub | div | Пункт с подменю (каретка ›, .menu__sub внутри) |
| .menu__item-icon / -label / -hint / -caret / -check | span | Части пункта: иконка, подпись, горячая клавиша, каретка, галочка |
| .menu__label | div | Заголовок группы (Body XS uppercase) |
| .menu__divider | hr | Разделитель между группами |
| .is-hover / .is-focus / .is-active | пункт | Принудительные состояния (для доков/тестов) |
| role / aria-haspopup / aria-expanded / aria-checked / aria-disabled | меню, триггер, пункт | Доступность: семантика меню и его состояний |
