> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Функциональный разделитель контейнера: пользователь тащит сплиттер и регулирует размеры соседних панелей. При наведении курсор меняется на col-resize / ew-resize, а сам сплиттер переходит в состояние Hover. Перемещается по одной оси; в момент перетаскивания вид и курсор сохраняются.

## Ключевые правила (из разделов страницы)
- **Использование** — Используйте сплиттер, когда нужно:
- **Анатомия** — Сплиттер — это узкая зона захвата (11px) между двумя панелями. По центру проходит тонкая линия 1px, в середине — ручка-грип из точек, подсказывающая, что элемент можно тащить.
- **Размеры** — У сплиттера нет масштабной линейки размеров (M/S/…) — геометрия задана фиксированными токенами — одними и теми же для любой ориентации. Значения ниже измерены на живом экземпляре компонента (getComputedStyle).
- **СостоянияНовое** — Исходная пара Default / Hover дополнена состояниями Move (перетаскивание), Focus и Disabled. В Move сплиттер сохраняет вид Hover и курсор col-resize на всём документе, пока кнопка мыши зажата.
- **Поведение · Поведение и ограничения** — Сплиттер перемещается только по своей оси. Размеры панелей ограничены минимумом и максимумом (здесь 25% / 75%), чтобы ни одна панель не схлопнулась полностью. Перетащите разделитель — соотношение обновляется в реальном времени.
- **ОриентацияНовое** — Вертикальный сплиттер регулирует ширину колонок (курсор col-resize) — основной сценарий. Горизонтальный регулирует высоту строк (курсор row-resize) — например, делит контент и панель деталей по высоте. Грип разворачивается вдоль оси перетаскивания.
- **Использование · Пример использования** — Раздел «Пользователи»: слева — таблица-мастер, справа — карточка деталей. Сплиттер позволяет расширить нужную сторону. Потяните разделитель внутри макета.
- **Контент** — У сплиттера нет видимого текста — единственный «контент» это описание в aria-label, которое скрин-ридер озвучивает вместо визуальной подписи:
- **ДоступностьНовое** — Сплиттеру присваивается role="separator", aria-orientation и значения aria-valuemin / aria-valuemax / aria-valuenow (в процентах). Он фокусируется по Tab; стрелки ←/→ (или ↑/↓ для горизонтального) сдвигают границу с шагом 2%, с Shift — 10%, Home / End — в минимум / максимум. Сфокусируйте сплиттер в примерах выше и подвигайте стрелками.
- **Использование · Рекомендации** — Сплиттер должен читаться как перетаскиваемый: оставляйте ручку-грип и состояние Hover. Невидимый разделитель без аффорданса пользователь не найдёт.
- **Цвета · Токены** — Компонент настраивается через CSS-переменные. Значения по умолчанию ссылаются на семантические токены дизайн-системы.
- **Цвета** — Компонент использует только семантические токены границ, текста и подсветки. Собственных цветов нет.

### Разметка · HTML (эталонная реализация ДС)

```
<!-- вертикальный сплиттер между двумя панелями -->
<div class="splitpane">
  <div class="splitpane__panel splitpane__a">…</div>

  <div class="spl" role="separator" aria-orientation="vertical"
       aria-valuemin="20" aria-valuemax="80" aria-valuenow="50"
       aria-label="Изменить ширину панелей" tabindex="0">
    <span class="spl__grip"><i></i><i></i><i></i><i></i><i></i><i></i></span>
  </div>

  <div class="splitpane__panel splitpane__b">…</div>
</div>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. pointerdown на .spl → dragging = true, setPointerCapture, курсор фиксируется
//    на body (body.spl-dragging { cursor: col-resize !important }) — работает
//    поверх любых элементов, оказавшихся под курсором во время перетаскивания

// 2. pointermove (пока dragging) →
pct = clamp( (позиция_курсора - начало_контейнера) / длина_контейнера * 100, min, max )
//    применить flex-basis панели A = pct%, обновить aria-valuenow = round(pct)

// 3. pointerup / pointercancel → dragging = false, снять курсор с body

// 4. Клавиатура на самом .spl (role="separator", tabindex="0"):
//    ArrowLeft/ArrowRight (vert) или ArrowUp/ArrowDown (horiz) → шаг 2%, Shift → шаг 10%
//    Home / End → к минимуму / максимуму

// 5. Disabled: pointer-events: none; роль и aria-value* остаются — значение
//    по-прежнему осмысленно для чтения, меняться просто нельзя
```

### Splitter.types.ts

```
type SplitterOrientation = 'vertical' | 'horizontal';

interface SplitterProps {
  orientation?: SplitterOrientation;    // по умолчанию 'vertical'
  value: number;                       // % размера панели A — controlled
  onChange: (value: number) => void;
  min?: number;                        // %, по умолчанию 20
  max?: number;                        // %, по умолчанию 80
  step?: number;                       // шаг стрелками, по умолчанию 2
  bigStep?: number;                    // шаг со Shift, по умолчанию 10
  disabled?: boolean;
  'aria-label': string;             // обязателен — что именно меняется в размере
}
```
