> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Бейдж — небольшой дочерний элемент, который накладывается на родительский компонент или ставится рядом с ним. Тип Counter показывает количество (счётчик), тип Dot — статус без числа. Пять размеров — M / S / XS / XXS / Dot — и семантические тона.

## Ключевые правила (из разделов страницы)
- **Использование** — Встраивается в компонент в качестве дочернего:
- **Анатомия** — Бейдж — это контейнер-капсула со значением внутри. Ширина зависит от контента, высота фиксирована по размеру. При наложении на родителя бейдж позиционируется в правом верхнем углу и выступает за его границу.
- **Типы** — Два типа по содержимому. Counter несёт число — точное количество непрочитанного, элементов в табе, товаров в корзине. Dot не несёт числа — это компактный индикатор наличия события или статуса.
- **Размеры** — Четыре размера счётчика и фиксированный Dot. Размер выбирается по габаритам родителя и плотности интерфейса. Однозначное число даёт круг (min-width = высота), многозначное — капсулу с горизонтальными отступами.
- **Тон** — Тон передаёт смысл, а не просто цвет. Neutral — нейтральный счётчик без эмоциональной окраски (количество в табах, списках). Accent — привлекает внимание к новому (непрочитанное). Статусные тона — для индикации состояния родителя.
- **Переполнение и форматНовое** — У счётчика есть максимум (по умолчанию 99). Если значение его превышает, бейдж показывает max+ — например «99+», чтобы капсула не растягивалась бесконтрольно. В компактных местах (счётчик скрытых элементов, «ещё N») используется безфоновый текстовый вариант badge--text в формате «+99».
- **ПозиционированиеНовое** — Counter накладывается на правый верхний угол родителя и центрируется на его границе. Dot для статуса ставится в правый нижний угол (например, индикатор «онлайн» на аватаре). На цветном или фото-фоне у бейджа появляется белая обводка (badge--bordered), чтобы он читался отдельно от подложки.
- **Состояния и поведениеНовое** — Поведение бейджа определяется значением и состоянием родителя.
- **Контент** — Содержимое бейджа — число (Counter) или его отсутствие (Dot). Краткий свод правил (подробно — в «Переполнение и формат» выше):
- **Правила** — Бейдж дополняет родителя, а не заменяет его. Не помещайте в счётчик длинные строки и не используйте его как самостоятельную метку-«пилюлю».
- **Доступность** — Бейдж — неинтерактивный визуальный элемент: он сам по себе не получает фокус и не имеет собственной семантики вне контекста родителя. За доступность отвечает компонент, к которому бейдж прикреплён.
- **Типографика** — Значение набирается гарнитурой SB Sans Text начертанием Semibold с моноширинными цифрами (font-variant-numeric: tabular-nums), чтобы счётчик не «прыгал» при смене цифр. Кегль зависит от размера.
- **Цвета** — Компонент использует только семантические токены. Текст на цветных тонах — --text-on-dark; на amber для контраста — тёмный --c-cgrey-800.

## Для разработчиков (выжимка)

### Точные размеры (redline)

Таблица рендерится на странице через getComputedStyle. Точные значения — в CSS-файле компонента (см. `css:` в шапке).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- counter, наложенный на родителя (иконка) -->
<span class="badge-anchor" aria-label="Уведомления, 7">
  <span class="navcell"><svg aria-hidden="true">…</svg></span>
  <span class="badge badge--xs badge--accent badge--bordered" aria-hidden="true">7</span>
</span>

<!-- dot-статус на аватаре: без числа, описание — через aria-label родителя -->
<span class="badge-anchor" aria-label="Иван Белов, онлайн">
  <span class="avatar">ИБ</span>
  <span class="badge badge--dot badge--success badge--bordered" aria-hidden="true"></span>
</span>

<!-- безфоновый текстовый вариант «+N» -->
<span class="badge badge--m badge--neutral badge--text">+99</span>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Значение count = 0 и нет явного флага showZero → бейдж не рендерится (display:none)

// 2. Переполнение: если count > max → текст = max + "+"; полное значение — в title/aria-label

// 3. Позиционирование при наложении на родителя (требует position:relative на родителе):
//    Counter → правый верхний угол, translate(45%, -45%) от собственного размера бейджа;
//    Dot (статус) → правый нижний угол по умолчанию; модификатор → правый верхний, если нужен тот же угол, что и у counter

// 4. Bordered: добавляется автоматически, когда родитель — фото/цветная подложка (аватар, обложка обложки)

// 5. Muted: синхронизируется с disabled-состоянием родителя, не управляется независимо

// 6. Бейдж не интерактивен: нет собственного focus/hover/keyboard — все взаимодействие через родителя
```

### Badge.types.ts

```
type BadgeType = 'counter' | 'dot';
type BadgeSize = 'm' | 's' | 'xs' | 'xxs';
type BadgeTone = 'neutral' | 'accent' | 'success' | 'info' | 'warning' | 'error';

interface BadgeProps {
  type?: BadgeType;               // по умолчанию 'counter'
  tone?: BadgeTone;                // по умолчанию 'neutral'
  size?: BadgeSize;                // по умолчанию 'm'; игнорируется для type='dot'
  value?: number;              // только для counter
  max?: number;                  // порог переполнения, по умолчанию 99
  showZero?: boolean;            // показывать при value === 0 (по умолчанию скрыт)
  text?: boolean;                // безфоновый вариант «+N»
  bordered?: boolean;            // белая обводка для наложения на фото/цвет
  muted?: boolean;               // следует за disabled-состоянием родителя
  'aria-hidden'?: boolean;        // true, когда значение уже есть в aria-label родителя
}

interface BadgeAnchorProps {
  children: [React.ReactNode, React.ReactNode]; // [родитель, Badge]
  dotPosition?: 'top-right' | 'bottom-right'; // по умолчанию 'bottom-right' для dot
  'aria-label'?: string;         // описание для скрин-ридера, включая значение бейджа
}
```
