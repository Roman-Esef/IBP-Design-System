> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Свитч мгновенно переключает состояние — например, включает или отключает определённую опцию. Действие применяется сразу, без подтверждения. Два состояния — Off / On — и интерактивные состояния Default / Hover / Focus / Pressed / Disabled / Loading.

## Ключевые правила (из разделов страницы)
- **Использование** — Используйте свитч, когда нужно:
- **Использование · Описание работы** — Клик по названию свитча или по самому контролу приводит к его переключению. Повторное нажатие переключает свитч обратно. Поиграйте с примером:
- **Анатомия** — Контрол свитча имеет размер 20×36 px: трек 20px высотой (равен интерлиньяжу Body M) с круглым ползунком 16px и отступом 2px со всех сторон. В группе сверху может быть заголовок.
- **Размеры** — У свитча один размер. Трек высотой 20px равен интерлиньяжу Body M, чтобы контрол центрировался по первой строке названия. Значения ниже измерены на живом экземпляре.
- **Состояния** — Два состояния по вертикали — Off / On — и интерактивные состояния по горизонтали. Состояния Hover, Focus, Pressed и Loading добавлены к исходной матрице Default / Disabled.Новое
- **Варианты · Состав** — Вспомогательные части — название и хелпер — опциональны. Свитч рендерится без названия (например, на карточке), с названием и с хелпером. Контрол всегда выровнен по первой строке текста.
- **Загрузка и асинхронное действиеНовое** — Свитч выполняет действие мгновенно. Но если переключение запускает запрос к серверу, который может занять время, покажите промежуточное состояние Loading: в ползунке появляется индикатор, контрол временно недоступен для повторного нажатия. Когда ответ получен — свитч фиксируется в новом состоянии; при ошибке возвращается в исходное.
- **Свитч в строке настроекНовое** — Частый паттерн: название слева, контрол прижат к правому краю строки. Вся строка — активная область. Так свитчи легко сканировать в списке настроек, а контролы выровнены по правому краю.
- **Контент** — Текстовый контент свитча — название и опциональный хелпер. Краткий свод правил (подробно — в «Название свитча» ниже):
- **Контент · Название свитча** — Название должно быть коротким и понятным. Название должно описывать, что делает свитч в активном (On) состоянии, и пишется в положительном ключе — без отрицаний и без указания текущего состояния.
- **Область клика и доступностьНовое** — Видимый контрол — 20×36px, но кликабельная область шире: она включает название и хелпер. В строке настроек активная область вытягивается на всю строку. Управление с клавиатуры: Tab ставит фокус на свитч, Space / Enter переключают его, фокус показывается обводкой --primary. Контролу присваивается role="switch" с aria-checked.
- **Типографика** — Название набирается гарнитурой SB Sans Text токеном --type-body-m, хелпер — --type-body-xs, заголовок группы — --type-h6-strong.
- **Цвета** — Компонент использует только семантические токены. Производные оттенки (hover) и state-слои вычисляются из --primary через color-mix.

## Для разработчиков (выжимка)

### Точные размеры (redline)

Таблица рендерится на странице через getComputedStyle. Точные значения — в CSS-файле компонента (см. `css:` в шапке).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- свитч с названием и хелпером; контрол и текст — внутри <label> -->
<label class="sw sw--on">
  <input type="checkbox" class="sw__input" role="switch" aria-checked="true" checked>
  <span class="sw__control"><span class="sw__thumb"></span></span>
  <span class="sw__content">
    <span class="sw__label">Тёмная тема</span>
    <span class="ds-helper ds-helper--left">Применяется ко всему интерфейсу</span>
  </span>
</label>

<!-- loading: класс .sw--loading + спиннер внутри ползунка, ввод блокируется -->
<label class="sw sw--off sw--loading">
  <input type="checkbox" class="sw__input" role="switch" disabled>
  <span class="sw__control"><span class="sw__thumb"><span class="sw__spinner"></span></span></span>
</label>

<!-- строка настроек: название слева, свитч прижат вправо -->
<div class="sw-row">
  <label class="sw sw--on">…</label> <!-- .sw-row разворачивает .sw в row-reverse -->
</div>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Состояние — бинарное на нативном input (checked=off/on),
//    отражается также в aria-checked для ассистивных технологий

// 2. Действие применяется МГНОВЕННО по клику, без подтверждения

// 3. Loading (асинхронное действие):
//    клик → оптимистично показать .sw--loading (спиннер вместо точки, ввод заблокирован)
//    → дождаться ответа сервера → успех: зафиксировать новое состояние;
//    ошибка: откатиться к предыдущему состоянию, снять .sw--loading

// 4. Клик по названию ИЛИ контролу переключает (<label> оборачивает input)

// 5. Интерактивные состояния — чистый CSS (:hover / :active / :focus-visible);
//    hover: halo 5px; pressed: halo 7px и ползунок расширяется до 19px
//    классы .sw--hover / .sw--focus / .sw--pressed — только для спецификации в документации

// 6. Клавиатура: Tab фокусирует input, Space или Enter переключают;
//    :focus-visible → outline 2px --primary, offset 2px

// 7. Disabled: input.disabled + .sw--disabled (pointer-events: none)
```

### Switch.types.ts

```
interface SwitchProps extends Omit<React.InputHTMLAttributes<HTMLInputElement>, 'type' | 'onChange'> {
  checked: boolean;              // Off / On
  onChange(next: boolean): void | Promise<void>;
  label?: React.ReactNode;        // название справа от контрола
  helper?: React.ReactNode;       // подпись Body XS под названием
  loading?: boolean;             // асинхронное действие в процессе — спиннер в ползунке, ввод заблокирован
  disabled?: boolean;
  settingsRow?: boolean;          // паттерн «название слева, контрол справа» (.sw-row)
}
```

### Справочник классов и атрибутов

| Класс / атрибут | На чём | Назначение |
|---|---|---|
| .sw | label | Корень: inline-flex, зазор 10px, выравнивание по верхней строке |
| .sw--off / --on | label | Состояние контрола |
| .sw--hover / --focus / --pressed | label | Форс-состояния — только для спецификации в документации, не для продакшена |
| .sw--loading | label | Асинхронное действие: pointer-events: none, спиннер в ползунке (.sw__spinner) |
| .sw--disabled | label | Отключено: pointer-events: none, текст/трек в --disabled |
| .sw--no-content | label | Контрол без названия/хелпера (например, на карточке) |
| .sw__input | input | Визуально скрытый input[type=checkbox] с role="switch" и aria-checked |
| .sw__control | span | Трек 36×20px |
| .sw__thumb | span | Ползунок 16px; все state-слои и анимация перемещения на нём |
| .sw__spinner | span | Индикатор загрузки внутри ползунка (только loading) |
| .sw__content | span | Колонка название + хелпер (зазор 2px) |
| .sw__label | span | Название (Body M) |
| .ds-helper.ds-helper--left | span | Хелпер (Body XS) — общий компонент, см. Label / Helper |
| .sw-row | div | Строка настроек: .sw внутри в row-reverse, вся строка — hit area |
| .sw-group | div | Контейнер списка свитчей (колонка) |
| .sw-group__title | p | Заголовок группы (H6 Strong) |
| role="switch" / aria-checked | input | Доступность: сообщает ассистивным технологиям текущее состояние как switch, не checkbox |
