> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Всплывающая подсказка при наведении или фокусе на объект — иконку, IconButton, обрезанный текст или поле с ошибкой. Контент — строка, тултип не может быть пустым. Хинт может быть со стрелкой или без неё, в любом из 12 краевых положений.

## Ключевые правила (из разделов страницы)
- **Использование** — Три типичных сценария — пояснение иконки, полное значение обрезанного текста, сообщение об ошибке рядом с полем. Наведите курсор на каждый пример.
- **Использование** — Используйте тултип, когда нужно:
- **Анатомия** — Тултип состоит из контейнера с фоном, текстового контента и опциональной стрелки, направленной на объект подсказки.
- **Размеры** — В общем случае размер тултипа определяется размером контента. Задаётся максимальная ширина контейнера (--tip-max = 280px). Если ширина контента с паддингами меньше максимальной — тултип сжимается до обтекания контента.
- **Контент** — Текст тултипа — единственное содержимое компонента; пустым он не бывает. Правила:
- **Варианты · Типы** — Виды тултипа отличаются между собой цветом фона. Main — нейтральные подсказки, Error — сообщения об ошибке рядом с полем или контролом.
- **Поведение · Размещение и стрелка** — Тултип размещается с одной из 4 сторон объекта (top · bottom · left · right), а стрелка выравнивается вдоль края по одному из 3 положений (start · center · end) — всего 12 комбинаций. Стрелка всегда направлена на объект подсказки.
- **Авто-разворот и смещениеНовое** — Если у предпочтительной стороны не хватает места, тултип разворачивается на противоположную сторону, а вдоль края смещается, чтобы не выходить за границы экрана — при этом стрелка остаётся направленной на объект. Перетащите цель к краям области, чтобы увидеть поведение.
- **Триггеры и таймингиНовое** — Тултип появляется при наведении курсора и при фокусе с клавиатуры, скрывается при уходе курсора, потере фокуса или нажатии Esc. Чтобы случайные пересечения курсора не вызывали мелькания, появление задерживается на ~400 мс, скрытие — мгновенное. На тач-устройствах тултип вызывается долгим нажатием.
- **ДоступностьНовое** — Контейнеру присваивается role="tooltip" и уникальный id, а объект-триггер ссылается на него через aria-describedby. Триггер должен быть фокусируемым (кнопка, ссылка или элемент с tabindex="0"). Тултип не перехватывает фокус и закрывается по Esc.
- **Использование · Рекомендации** — Держите текст коротким и по делу. Тултип поясняет объект, а не дублирует видимую подпись и не содержит длинных инструкций.
- **Типографика** — Текст тултипа набирается гарнитурой SB Sans Text токеном --type-body-s.
- **Цвета** — Компонент использует только семантические токены. Стрелка наследует фон контейнера через переменную --tip-bg.

## Для разработчиков (выжимка)

### Точные размеры (redline)

Таблица рендерится на странице через getComputedStyle. Точные значения — в CSS-файле компонента (см. `css:` в шапке).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- триггер ссылается на тултип через aria-describedby -->
<button type="button" class="iconbtn" aria-label="Удалить" aria-describedby="tip-1">…</button>

<!-- сам тултип: позиционируется JS относительно триггера, скрыт до показа -->
<span class="tip tip--main tip--top tip--center tip--floating" role="tooltip" id="tip-1">
  Удалить
  <span class="tip__arrow"></span>
</span>

<!-- без стрелки / с переносом -->
<span class="tip tip--error tip--bottom tip--start tip--no-arrow tip--multiline" role="tooltip">…</span>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Триггеры: hover (задержка ≈400мс перед показом, чтобы случайное пересечение курсора
//    не вызывало мелькание), focus (мгновенно, без задержки); скрытие — мгновенно по mouseleave/blur/Esc

// 2. Позиционирование (вызывается при показе и при resize):
//    взять getBoundingClientRect() цели и тултипа относительно позиционирующего родителя;
//    top/bottom — y = край цели ± gap(8px); left/right — аналогично по X;
//    align center/start/end сдвигает вторую ось относительно центра цели (start/end — отступ 13px для стрелки)

// 3. Авто-разворот: проверить свободное место по предпочтительной стороне относительно viewport/контейнера;
//    не хватает — пробовать противоположную, затем перпендикулярные стороны; выбрать первую подходящую;
//    затем зажать позицию внутри границ контейнера и пересчитать позицию стрелки, чтобы она всё ещё указывала на цель

// 4. Перенос: если ширина одной строки > --tip-max — включить white-space:normal (.tip--multiline)

// 5. Доступность: триггер должен быть фокусируем элементом; тултип не перехватывает фокус и закрывается по Esc без сброса фокуса с триггера
```

### Tooltip.types.ts

```
type TooltipType = 'main' | 'error';
type TooltipPlacement = 'top' | 'bottom' | 'left' | 'right';
type TooltipAlign = 'start' | 'center' | 'end';

interface TooltipProps {
  content: React.ReactNode;         // обязательно; тултип не рендерится без содержимого
  children: React.ReactElement;     // триггер — должен принимать ref и быть фокусируемым
  type?: TooltipType;               // по умолчанию 'main'
  placement?: TooltipPlacement;     // по умолчанию 'top'; авто-разворачивается при нехватке места
  align?: TooltipAlign;             // по умолчанию 'center'
  arrow?: boolean;                // по умолчанию true
  delay?: number;                 // задержка появления при hover в мс, по умолчанию 400
  disabled?: boolean;             // полностью отключить тултип
}
```

### Справочник классов и атрибутов

| Класс / атрибут | На чём | Назначение |
|---|---|---|
| .tip | span | Контейнер: фон, паддинги 8/12, тень |
| .tip--main / --error | span | Тип (цвет фона) |
| .tip--top / --bottom / --left / --right | span | Размещение относительно объекта |
| .tip--start / --center / --end | span | Выравнивание стрелки вдоль края |
| .tip--no-arrow | span | Скрывает .tip__arrow, отступает от объекта на 8px |
| .tip--multiline | span | Перенос текста (white-space: normal) в границах --tip-max |
| .tip--floating | span | JS-позиционирование относительно триггера; показ/скрытие через .is-visible |
| .tip__arrow | span | Треугольная стрелка (clip-path), наследует фон через --tip-bg |
| .tip-anchor | span | position:relative обёртка для floating-тултипа |
| role="tooltip" / aria-describedby | span / триггер | Доступность: связь тултипа с объектом-триггером |
