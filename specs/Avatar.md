> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Аватар — компактное визуальное представление пользователя или сущности. Содержит инициалы (Text), запасную иконку (Icon) или фотографию (Image). Две формы — Circular и Rounded — четыре размера и опциональный бейдж: счётчик или индикатор статуса.

## Ключевые правила (из разделов страницы)
- **Использование** — Может использоваться как самостоятельный компонент, так и в составе другого — например, в сущности (Entity): строка пользователя, автор комментария, участник в ячейке таблицы. Группа аватаров показывает несколько участников компактным стеком.
- **Анатомия** — Аватар — это фигура-контейнер фиксированного размера с содержимым по центру и опциональным бейджем поверх. В группе у каждого аватара появляется белое кольцо-разделитель.
- **Типы содержимого** — Тип определяется доступными данными. Приоритет — изображение; если его нет, показываем инициалы; если нет и имени — нейтральную иконку.
- **Форма** — Circular — для людей (инициалы, фото). Rounded (скруглённый квадрат) — для сущностей, организаций, ботов. Радиус скругления у Rounded пропорционален размеру: 28 / 12 / 10 / 8 px для XL / L / M / S.
- **Размеры** — Четыре размера. Выбирайте по контексту: XL — карточка профиля, L — строки списков и шапки, M — таблицы и навигация, S — плотные перечисления и инлайн-упоминания.
- **Бейдж на аватареНовое** — Аватар может нести дочерний Badge. Counter ставится в правый верхний угол — непрочитанные сообщения, количество задач. Dot ставится в правый нижний угол как индикатор присутствия. На фото у бейджа всегда белая обводка (badge--bordered).
- **Группа аватаровНовое** — Несколько участников показываются перекрывающимся стеком: левый аватар лежит поверх следующего, между ними — белое кольцо. Если участников больше лимита, остаток сворачивается в нейтральный чип «+N». Рекомендуемый лимит — 3–5 видимых аватаров.
- **Заглушки и фолбэкНовое** — Содержимое выбирается по цепочке деградации: если есть фотография — показываем её; если фото нет или не загрузилось — инициалы; если нет и имени — нейтральную иконку. Это гарантирует, что аватар никогда не остаётся пустым.
- **Контент** — Содержимое аватара определяется доступными данными, а не выбирается произвольно — см. цепочку деградации в разделе «Заглушки и фолбэк» выше. Правила текста:
- **СостоянияНовое** — Аватар может быть интерактивным — вести на профиль. В этом случае у него есть состояния Hover, Focus (обводка --primary) и Pressed. Неактивный аватар приглушается.
- **Доступность** — Сам по себе аватар — визуальный, декоративный элемент. Доступность определяется контекстом использования.
- **Типографика** — Инициалы набираются гарнитурой SB Sans Text начертанием Semibold. Кегль пропорционален размеру аватара (около ⅓ стороны). Используются 1–2 заглавные буквы.
- **Цвета** — Аватар по умолчанию использует нейтральную «болотную» подложку. Для сущностей доступен акцентный тон. Индикаторы статуса наследуют семантические токены из Badge.

### Разметка · HTML (эталонная реализация ДС)

```
<!-- самостоятельный аватар с инициалами -->
<span class="av av--circular av--l">
  <span class="av__text">ИБ</span>
</span>

<!-- с бейджем: обёртка .av-stack даёт position:relative для абсолютного позиционирования -->
<span class="av-stack" aria-label="Мария А., в сети">
  <span class="av av--circular av--xl" aria-hidden="true">…</span>
  <span class="badge badge--dot badge--success badge--bordered" aria-hidden="true"></span>
</span>

<!-- группа участников с переполнением -->
<span class="av-group av-group--m" role="group" aria-label="Участники: ИБ, ВБ, МА, КГ и ещё 3">
  <span class="av av--circular av--m" aria-hidden="true">…</span>
  <!-- ещё 3–4 <span class="av"> — каждый смещается на -–av-overlap -->
  <span class="av-group__more" aria-hidden="true">+3</span>
</span>

<!-- интерактивный аватар-ссылка на профиль -->
<button type="button" class="av av--circular av--l av--button" aria-label="Открыть профиль Игоря Белова">…</button>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Фолбэк-цепочка содержимого (при монтировании и при ошибке загрузки фото):
//    есть URL изображения и оно загрузилось → content = 'image'
//    иначе, есть имя/название → content = 'text' (вычислить инициалы)
//    иначе → content = 'icon' (нейтральный глиф)
//    обработать <img>.onerror — при ошибке загрузки переключиться на следующее звено цепочки

// 2. Инициалы: разбить полное имя на слова, взять первый символ первого
//    и первого символа последнего слова, привести к верхнему регистру; одно слово → 1 буква

// 3. Группа аватаров с переполнением:
//    рендерить первые limit участников (рекомендуемо 3–5);
//    остаток = total - limit; если > 0 — добавить .av-group__more с текстом '+'+остаток;
//    z-index каждого аватара = (общее число видимых узлов − индекс), чтобы левый лежал сверху

// 4. Бейдж позиционируется через CSS относительно .av-stack — отдельного JS не требуется;
//    Counter — правый верхний угол (translate 40%,-40%); Dot — правый нижний (7% / 7%)

// 5. Disabled: класс .av--disabled (opacity .5 + grayscale + pointer-events: none)
```

### Avatar.types.ts

```
type AvatarShape = 'circular' | 'rounded';
type AvatarSize  = 'xl' | 'l' | 'm' | 's';

interface AvatarProps {
  shape?: AvatarShape;             // по умолчанию 'circular'
  size?: AvatarSize;               // по умолчанию 'm'
  name?: string;                 // источник для авто-инициалов и aria-label
  src?: string;                  // URL фото; при отсутствии/ошибке — фолбэк на инициалы/иконку
  icon?: React.ReactNode;          // переопределяет глиф фолбэка (иначе — нейтральный user)
  tone?: 'accent';                // акцентная подложка для Entity
  badge?: { type: 'counter' | 'dot'; value?: number; tone?: string };
  interactive?: boolean;         // рендерится как <button>, требует aria-label
  disabled?: boolean;
  decorative?: boolean;          // true — рядом уже есть видимое имя → aria-hidden
}

interface AvatarGroupProps {
  items: { name: string; src?: string }[];
  size?: 'l' | 'm' | 's';
  shape?: AvatarShape;
  limit?: number;                // сколько аватаров показать явно; остаток → «+N»
  'aria-label'?: string;
}
```
