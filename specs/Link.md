> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Текстовая ссылка для навигации внутри лайаута и модальных окон, а также в Хлебных крошках. Тоны Accent / Muted / Neutral / Info / Warning / Error / Success, размеры M / S (S также используется в таблицах, где весь текстовый контент набирается размером S), состояния Default и Hover = Focus (подчёркивание) — кроме тона Neutral (Breadcrumbs_2): у него только состояние Default, ссылка не кликабельна. Цветовое решение едино для всех видов и размеров.

## Ключевые правила (из разделов страницы)
- **Использование** — Используйте ссылку, когда нужно:
- **Анатомия** — Ссылка — это текстовая метка, окрашенная в цвет тона. Подчёркивание работает как маркер интерактивности: у standalone-ссылки оно появляется на Hover / Focus, у инлайн-ссылки включено всегда. Опциональная иконка масштабируется относительно текста (1em). При навигации с клавиатуры добавляется обводка фокуса.
- **Варианты · Ссылки лайаута и модалок** — Основной тип ссылок (тон Accent). Цветовое решение едино для всех видов и размеров. В покое — без подчёркивания, на Hover и Focus появляется подчёркивание.
- **Варианты · Ссылки Хлебных крошек** — Ссылки используются в Хлебных крошках (шапка страницы). Есть два цветовых решения: Breadcrumbs_1 (тон Muted — родительские страницы) — кликабельна, подчёркивается на Hover и Focus; и Breadcrumbs_2 (тон Neutral — выделенная текущая страница) — не кликабельна, доступно только состояние Default, без подчёркивания. При длинной цепочке средние крошки сворачиваются в кнопку «…»: клик по ней открывает выпадающий список с названиями скрытых страниц (сами крошки не разворачиваются). Полная спецификация компонента (анатомия, авто-схлопывание, доступность) — на странице Breadcrumbs.
- **Размеры** — Два размера, привязанных к токенам текста: M — --type-body-m (16/20), S — --type-body-s (14/16). Инлайн-ссылка наследует размер окружающего текста. Размер S также используется в таблицах — весь текстовый контент внутри таблицы набирается размером S.
- **Состояния** — Hover и Focus оформлены одинаково (подчёркивание) — как в спецификации. Дополнительно добавлены Pressed и Disabled, а на клавиатурный фокус показывается обводка для доступности. Тон Neutral (Breadcrumbs_2) не кликабелен — для него доступно только состояние Default.Новое
- **Ссылка внутри текстаНовое** — Инлайн-ссылка живёт в потоке текста и наследует его размер. Чтобы её можно было отличить не только по цвету, инлайн-ссылка всегда подчёркнута (мягким подчёркиванием), а на Hover подчёркивание становится насыщенным. Это требование доступности (WCAG 1.4.1).
- **С иконкой и внешние ссылкиНовое** — Иконка может стоять слева (например, ссылка-скачивание) или справа. Внешняя ссылка ведёт на другой ресурс — она открывается в новой вкладке (target="_blank", rel="noopener") и помечается иконкой со стрелкой. Иконка масштабируется относительно текста (1em).
- **Перенос и обрезкаНовое** — Длинная ссылка по умолчанию переносится по словам вместе с текстом. В узких контейнерах (ячейка таблицы, чип) ссылку можно обрезать многоточием — полное значение остаётся в title / тултипе.
- **Контент** — Контент ссылки — её текст (и опциональная иконка). Текст должен считываться вне контекста:
- **Доступность** — Не полагайтесь только на цвет: смысл ссылки должен считываться и без него. В потоке текста ссылка всегда подчёркнута; standalone-ссылки подчёркиваются на Hover / Focus. Клавиатурный фокус оформлен точно так же, как Hover, без отдельной обводки — единое оформление для мыши и клавиатуры. Для внешних ссылок добавляйте подпись «открывается в новой вкладке» в aria-label.
- **Типографика** — Ссылка набирается гарнитурой SB Sans Text. Размер задаётся токеном текста; начертание — Regular.
- **Цвета** — Компонент использует только семантические токены. Hover берёт затемнённый оттенок того же цвета.

## Для разработчиков (выжимка)

### Точные размеры (redline)

Таблица рендерится на странице через getComputedStyle. Точные значения — в CSS-файле компонента (см. `css:` в шапке).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- standalone-ссылка (лайаут / модалка), тон Accent -->
<a class="link link--accent link--m" href="/deals/4521">Открыть договор</a>

<!-- инлайн-ссылка в тексте: подчёркнута всегда -->
… подробнее в <a class="link link--accent link--inline" href="/help">справке</a> …

<!-- иконка слева (скачивание) -->
<a class="link link--accent link--m link--with-icon" href="/report.pdf">
  <span class="link__icon"><svg aria-hidden="true">…</svg></span>
  <span class="link__text">Скачать PDF</span>
</a>

<!-- внешняя ссылка: новая вкладка + иконка-стрелка справа -->
<a class="link link--accent link--m link--with-icon" href="https://…"
   target="_blank" rel="noopener" aria-label="Открыть в реестре (открывается в новой вкладке)">
  <span class="link__text">Открыть в реестре</span>
  <span class="link__icon"><svg aria-hidden="true">…</svg></span>
</a>

<!-- хлебные крошки: родители — ссылки Muted, текущая — не ссылка; полная анатомия — см. Breadcrumbs.html -->
<nav class="crumbs" aria-label="Хлебные крошки">
  <li class="crumbs__item"><a class="link link--muted" href="/">Главная</a></li>
  <!-- разделитель «/» — CSS ::after на .crumbs__item, не отдельный элемент -->
  <li class="crumbs__item crumbs__item--current">
    <span class="crumbs__current" aria-current="page">Договор №4521</span>
  </li>
</nav>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Подчёркивание — маркер интерактивности (отдельного DOM-слоя нет):
//    standalone (link--m/s): в покое без подчёркивания;
//    :hover / :focus-visible / .is-hover / .is-focus → text-decoration: underline
//    inline (link--inline): подчёркнута ВСЕГДА — 45% currentColor,
//    на hover/focus подчёркивание → 100% currentColor

// 2. Цвет задаёт тон:
//    accent  → --link, hover/focus --link-dark, pressed --c-emerald-900
//    muted   → --text-inactive, hover --text-secondary (Breadcrumbs_1)
//    neutral → --text-primary (Breadcrumbs_2); только Default, не кликабельна (pointer-events: none)
//    info/warning/error/success → --<тон>, hover --<тон>-dark, pressed на ступень темнее (ссылки в Alert/SnackBar)

// 3. Фокус = Hover: :focus-visible применяет ТЕ ЖЕ правила, что и :hover
//    (цвет + подчёркивание), без отдельной обводки — оформление одинаковое для мыши и клавиатуры

// 4. Disabled: aria-disabled="true" (или .link--disabled) →
//    цвет --text-inactive, pointer-events: none, подчёркивание снято

// 5. Внешняя ссылка: target="_blank" + rel="noopener";
//    aria-label дополняется «(открывается в новой вкладке)»; справа иконка-стрелка 1em

// 6. Обрезка: .link--truncate = inline-block + ellipsis; полное значение — в title/тултипе

// 7. Хлебные крошки: текущая страница — НЕ ссылка (span[aria-current=page]);
//    средние звенья схлопываются в кнопку «…», клик открывает выпадающий
//    список с названиями скрытых страниц (сами крошки не разворачиваются).
//    Форс-состояния .is-hover/.is-pressed/.is-focus — только для документации.
//    Полная спецификация компонента Хлебные крошки (авто-схлопывание по ширине,
//    обрезка текущей крошки, доступность) — см. Breadcrumbs.html
```

### Link.types.ts

```
type LinkTone = 'accent' | 'muted' | 'neutral' | 'info' | 'warning' | 'error' | 'success';
type LinkSize = 'm' | 's';

interface LinkProps extends React.AnchorHTMLAttributes<HTMLAnchorElement> {
  children: React.ReactNode;      // текст ссылки — называет цель перехода
  tone?: LinkTone;                // по умолчанию 'accent'
  size?: LinkSize;                // по умолчанию 'm'; игнорируется при inline
  inline?: boolean;              // в потоке текста → font: inherit + всегда underline
  icon?: React.ReactNode;         // иконка слева, 1em
  external?: boolean;            // target=_blank + rel=noopener + иконка-стрелка
  truncate?: boolean;            // обрезка многоточием (нужен title)
  disabled?: boolean;            // aria-disabled + pointer-events: none
}
```

### Справочник классов и атрибутов

| Класс / атрибут | На чём | Назначение |
|---|---|---|
| .link | a | Базовый класс: цвет по тону, переходы; фокус оформлен тем же правилом, что hover — без отдельного focus-ring |
| .link--accent / --muted / --neutral | a | Тон: Accent (лайаут/модалки) и Muted (Breadcrumbs_1) — задают цвет и оттенки hover/pressed; Neutral (Breadcrumbs_2) — только цвет Default, без hover/pressed, pointer-events: none (не кликабельна) |
| .link--m / --s | a | Размер standalone: 16/20 · 14/16. При inline не используется |
| .link--inline | a | В потоке текста: font: inherit + постоянное подчёркивание |
| .link--with-icon | a | inline-flex + зазор для иконки; парой идут .link__icon / .link__text |
| .link--truncate | a | Обрезка многоточием (inline-block + ellipsis); полное значение — в title |
| .link--disabled / aria-disabled="true" | a | Отключено: цвет --text-inactive, pointer-events: none, без подчёркивания |
| .is-hover / .is-pressed / .is-focus | a | Форс-состояния — только для спецификаций в документации, не для продакшена |
| .link__icon / .link__text | span | Слоты иконки (1em) и текста внутри ссылки с иконкой |
| target="_blank" · rel="noopener" | a | Обязательны на внешних ссылках; aria-label дополняется пометкой о новой вкладке |
| .crumbs | nav > ol | Контейнер хлебных крошек — единственный размер S, crumbs--m не существует; полная спецификация — страница Breadcrumbs |
| разделитель «/» | li::after | CSS-псевдоэлемент между звеньями, цвет как у Breadcrumbs_1; не часть DOM — не читается скринридером |
| .crumbs__current + aria-current="page" | span | Текущая страница — не ссылка, --text-primary; начертание такое же, как у родительских звеньев |
| .crumbs__more | button | Свёрнутое звено «…»; раскрывает попап со скрытыми страницами (см. Breadcrumbs.html) |
