> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Хлебные крошки — трейл ссылок в шапке страницы, показывающий путь до текущего раздела. Состоят из ссылок Link (Breadcrumbs_1 — родительские страницы, Breadcrumbs_2 — текущая, некликабельная) и разделителя «/». Показываются на всех страницах системы, кроме Дашборда; отступ до контента страницы — 0.

## Ключевые правила (из разделов страницы)
- **Использование** — Показывайте хлебные крошки, когда нужно:
- **Анатомия** — Крошки — это список ссылок (Breadcrumbs_1) с текущей неинтерактивной крошкой в конце (Breadcrumbs_2). Между звеньями — разделитель «/» того же цвета, что и родительские ссылки. Когда трейл не помещается по ширине, срединные звенья схлопываются в кликабельное «…».
- **Размеры** — У компонента единственный размер — S (--type-body-s, 14/16). Второго размера нет: крошки — вспомогательная навигация над заголовком страницы и не должны конкурировать с ним по весу, поэтому размер не варьируется.
- **Контент** — Правила именования и корнер-кейсы:
- **Состояния** — Кроме базового состояния из спецификации, документированы наведение/фокус на родительской ссылке, авто-схлопывание по ширине и обрезка текущей крошки — с раскрывающимся списком скрытых страниц.Новое
- **Поведение · Схлопывание** — Ширина измеряется по контейнеру (ResizeObserver). Если трейл не помещается и звеньев больше двух — звенья со 2 по предпоследнее заменяются на «…», первое и текущее звено остаются видимыми всегда; если даже так текущая крошка не помещается — обрезается многоточием с тултипом. Пересчитывается при каждом ресайзе, не только при первом рендере.
- **Доступность** — Контейнер — ориентир <nav aria-label="Хлебные крошки"> со списком <ol>/<li> внутри (паттерн WAI-ARIA APG для breadcrumb). Текущая страница — не ссылка и не фокусируется, помечена aria-current="page". Разделитель «/» — CSS-псевдоэлемент (::after), поэтому не входит в дерево доступности и не зачитывается скринридером. Focus у ссылок оформлен идентично Hover — единое цветовое решение для мыши и клавиатуры, без отдельной обводки. Кнопка «…» — обычная <button> с aria-haspopup / aria-expanded, раскрывающееся контекстное меню закрывается по Esc и клику вне.
- **Цвета** — Компонент использует только семантические токены — те же, что определены для тонов Muted и Neutral компонента Link.

## Для разработчиков (выжимка)

### Точные размеры (redline)

Таблица рендерится на странице через getComputedStyle. Точные значения — в CSS-файле компонента (см. `css:` в шапке).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- обычный трейл, всё помещается -->
<nav class="crumbs" aria-label="Хлебные крошки">
  <li class="crumbs__item"><a class="link link--muted" href="/">Главная</a></li>
  <li class="crumbs__item"><a class="link link--muted" href="/deals">Сделки</a></li>
  <li class="crumbs__item crumbs__item--current">
    <span class="crumbs__current" aria-current="page">Договор №4521</span>
  </li>
</nav>

<!-- трейл не помещается: срединные звенья схлопнуты в «…» -->
<nav class="crumbs" aria-label="Хлебные крошки">
  <li class="crumbs__item"><a class="link link--muted" href="/">Главная</a></li>
  <li class="crumbs__item">
    <button class="crumbs__more" aria-haspopup="true" aria-expanded="false">…</button>
    <!-- .menu.menu--floating (Контекстное меню), при открытии добавляется в конец <body> (position:fixed) -->
  </li>
  <li class="crumbs__item crumbs__item--current">
    <span class="crumbs__current" aria-current="page" title="Полное название страницы">Полное название страницы</span>
  </li>
</nav>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Рендер полного трейла: все звенья, кроме последнего, — ссылки (Breadcrumbs_1),
//    последнее — текст с aria-current="page" (Breadcrumbs_2).

// 2. Измерить: scrollWidth контейнера > clientWidth?
//    Если да и звеньев больше 2 — заменить звенья [1 .. N-2] на одно
//    звено-кнопку «…» (оставляя первое и текущее звено видимыми).
//    Измерение лучше делать в скрытом клоне той же ширины — тогда
//    пользователь не увидит промежуточного «полного» кадра.

// 3. Текущая крошка ВСЕГДА однострочная и обрезается многоточием по CSS
//    (overflow:hidden + text-overflow:ellipsis + white-space:nowrap) —
//    независимо от того, схлопнулись ли срединные звенья. Если это
//    реально обрезало текст (scrollWidth > clientWidth) — показать
//    полное название в тултипе по hover/focus.

// 4. Клик по «…»: открыть контекстное меню (см. Context Menu) со списком скрытых
//    страниц; клик по строке — переход; Esc/клик вне — закрыть.
//    aria-expanded кнопки синхронизируется с открыт/закрыт.

// 5. Пересчитывать схлопывание при ресайзе контейнера (ResizeObserver)
//    или ширины окна — не только при первом рендере.

// 6. Если у страницы нет родителей — компонент не рендерится вовсе.
```

### Breadcrumbs.types.ts

```
interface BreadcrumbItem {
  label: string;   // текст звена — совпадает с заголовком страницы
  href?: string;   // не указывается у последнего элемента (текущая страница)
}

interface BreadcrumbsProps {
  items: BreadcrumbItem[];        // от корня к текущей странице; <2 элементов → не рендерить
  onNavigate?: (href: string) => void; // клик по звену/по пункту меню «…»
  // единственный размер — S; size-пропа нет
}
```

### Справочник классов и атрибутов

| Класс / атрибут | На чём | Назначение |
|---|---|---|
| .crumbs | nav > ol | Контейнер трейла: flex-строка, разделитель через ::after; единственный размер — S (--type-body-s) |
| .crumbs__item | li | Звено трейла; у всех, кроме последнего, генерируется разделитель «/» |
| .crumbs__item--current | li | Звено с текущей страницей — последнее в списке |
| .link.link--muted | a | Родительское звено — Breadcrumbs_1 (см. компонент Link) |
| .crumbs__current + aria-current="page" | span | Текущая страница — Breadcrumbs_2, не ссылка, --text-primary; начертание то же, что у Breadcrumbs_1 — отличие только в цвете |
| .crumbs__more + aria-haspopup / aria-expanded | button | Свёрнутое звено «…»; раскрывает попап со скрытыми страницами |
| .menu.menu--floating.crumbs__popup | div | Контекстное меню скрытых звеньев (пункты — .menu__item): добавляется в конец <body> с position:fixed, чтобы его не обрезал overflow:hidden трейла |
| .tip.tip--floating | span | Тултип с полным названием обрезанной текущей крошки (см. компонент Tooltip) |
