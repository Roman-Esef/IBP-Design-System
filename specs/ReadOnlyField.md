> Автоспека для быстрого контекста. Источник истины — CSS-файл и страница компонента. При изменении компонента обновляй эту спеку.

## Назначение
Поле для отображения данных без возможности редактирования. Показывает значение — текст, чипы или ссылку — с опциональными лейблом и хэлпером (см. Label / Helper), иконками слева/справа и префиксом/постфиксом. Не имеет собственного контейнера: без рамки и фона, это пара «подпись → значение» для карточек сущностей и детальных панелей.

## Ключевые правила (из разделов страницы)
- **Использование** — ReadOnlyField используется исключительно для отображения данных — в карточках сущностей и детальных панелях, когда значение приходит из системы и не редактируется пользователем напрямую в этом месте экрана.
- **Анатомия** — Поле — вертикальный стек из трёх зон: подпись, строка значения и пояснение. Строка значения может нести до пяти слотов одновременно; всё, кроме самого значения, — опционально.
- **Размеры** — У ReadOnlyField нет собственной шкалы размеров (M/S/…) — типографика каждого слота фиксирована токенами: Label и Helper — Body XS, значение — Body M. Единственная зависимая переменная — размер вложенного Chip для значений-чипов — всегда XS. Значения ниже измерены на живом экземпляре.
- **Типы значения** — Три типа значения. Текст — произвольная строка, может обрезаться или переноситься. Чипы — список меток размера XS (см. компонент Chip, тип ReadOnly). Ссылка — значение-переход, набирается компонентом Link.
- **Опции** — Дополнительные слоты включаются независимо друг от друга и сочетаются в любой комбинации: иконка слева, префикс/постфикс, иконка справа (любой глиф из библиотеки иконок; действие — копирование, тултип или без действия), скрытие лейбла, выравнивание значения.
- **Переполнение и перенос** — Тип «Текст» может течь свободно, обрезаться до одной строки или до N строк — во всех случаях обрезанный текст уходит в многоточие, а при наведении курсора показывается тултип с полным содержанием. Тип «Чипы» переносится по-другому: чипы как целый список уходят на следующую строку, а при ограниченной высоте лишние могут сворачиваться в счётчик.
- **Цвет значения** — Текст значения может быть окрашен для передачи динамики — например, положительного (зелёный) или отрицательного (красный) изменения показателя. Остальные слоты (лейбл, хэлпер, аффиксы) цвет не наследуют.
- **СостоянияДополнено** — Кроме обычного заполненного значения, компоненту нужны состояния отсутствия данных и ожидания ответа от системы — в материалах ДС они не описаны, добавляем на существующих токенах.
- **Типографика** — Label и Helper набираются токеном --type-body-xs (как в компоненте Label/Helper), значение — --type-body-m. Letter-spacing — 0.
- **Контент** — Текстовое содержимое компонента — четыре независимых слота: подпись, значение, пояснение и опциональные аффиксы.
- **Правила** — ReadOnlyField сообщает, что значение принципиально недоступно для правки в этом месте интерфейса — в отличие от задизейбленного инпута, который лишь временно не активен.
- **Доступность** — ReadOnlyField — пара «подпись → значение» из статичного текста; сам по себе он не интерактивен и не участвует в таб-порядке, кроме собственных точечных интерактивных элементов (иконка-копирование, значение-ссылка).
- **Цвета** — Компонент использует только семантические токены текста; для положительной/отрицательной динамики — ситуативные токены Success/Error.
- **Что предлагаем добавить** — Материалы ДС описывают только структуру и типы ReadOnlyField, без состояний отсутствия/загрузки данных и без деталей интерактивных иконок. Ниже — предложения, собранные на существующих токенах и по аналогии с другими компонентами (Chip, Label/Helper).

### Разметка · HTML (эталонная реализация ДС)

```
<!-- значение-текст, с иконкой слева и копированием справа -->
<div class="rof">

  <div class="rof__row">
    <span class="rof__icon"><svg aria-hidden="true">…</svg></span>
    <span class="rof__value">123456789012</span>
    <span class="rof__icon rof__icon--interactive" role="button" tabindex="0" aria-label="Скопировать значение"><svg aria-hidden="true">…</svg></span>
  </div>
</div>

<!-- значение-чипы -->
<div class="rof">
  <span class="ds-label"><span class="ds-label__text">Регионы присутствия</span></span>
  <div class="rof__row"><div class="rof__value rof__value--chips"><div class="chiplist chiplist--s">…</div></div></div>
</div>
```

### Поведение · псевдокод (framework-agnostic)

```
// 1. Обрезка/тултип: когда scrollWidth/scrollHeight элемента значения > clientWidth/clientHeight
//    (текст обрезан ellipsis или line-clamp) — на hover/focus показать tooltip с полным текстом.
//    Проверка выполняется заново при каждом наведении — обрезка зависит от текущей ширины layout.

// 2. Копирование: клик/Enter/Space по rof__icon--interactive → navigator.clipboard.writeText(value) →
//    иконка временно меняется на «check», всплывает тултип «Скопировано» (~1.3с), затем возврат

// 3. Переполнение чип-листа (chipsMaxRows): после layout измерить offsetTop чипов, сгруппировать
//    по строкам; если строк больше лимита — скрыть лишние чипы и добавить чип-счётчик «+N»
//    с тултипом, перечисляющим скрытые значения

// 4. Empty: если данных нет — показать value = «—» приглушённым цветом вместо пустой строки

// 5. Loading: aria-busy="true" на корне; вместо label/value — шиммер-плейсхолдеры фиксированной высоты
```

### ReadOnlyField.types.ts

```
type ROFValueType = 'text' | 'chips' | 'link';
type ROFTone      = 'default' | 'positive' | 'negative' | 'empty';

interface ReadOnlyFieldProps {
  label: React.ReactNode;
  showLabel?: boolean;                // по умолчанию true
  helper?: React.ReactNode;
  helperStatus?: 'default' | 'error';
  type?: ROFValueType;                     // по умолчанию 'text'
  value?: React.ReactNode;                 // text / link
  chips?: string[];                     // type='chips'
  chipsMaxRows?: number;                // свёртка в «+N» (Предложение)
  linkHref?: string;                    // type='link'
  iconLeft?: React.ReactNode;
  prefix?: string;
  postfix?: string;
  iconRight?: React.ReactNode;
  iconRightAction?: 'none' | 'copy' | 'tooltip';
  iconRightTip?: string;                 // обязателен при iconRightAction='tooltip'
  tone?: ROFTone;                           // 'empty' — плейсхолдер «—» (Предложение)
  clampMode?: 'none' | '1' | '2' | '3';    // число строк перед line-clamp
  align?: 'left' | 'right';
  state?: 'default' | 'loading';      // (Предложение)
}
```
