## Ключевые уроки (не наступать снова)
1. **Судить рендер только на ширине макета (1920), 1:1.** Мелкий zoom прячет кегли; узкое превью харнеса (~900px) ≠ 1920.
2. **Мокап = фиксированный канвас** (`.scr__main{min-width:1664px}`), НЕ адаптивное схлопывание. Выдуманный `@media`-брейкпоинт показывает пользователю деградацию.
3. **Проверять то, что файл рендерит сам**, без временных `!important`-override в скриншот-коде.
4. **Компоненты ДС публикуют только CSS.** Поведение и стили-композиции живут в `scripts/<comp>.page.js` и inline-`<style>` страниц-витрин, НЕ в `ds.css`/бандле. Поэтому «из коробки» их взять нельзя — приходится до-собирать (отсюда были баги NavPanel/RiskMetric). Стили `rm-block` пришлось скопировать локально; поведение NavPanel — переписать в `screens-chrome.js`.
5. **Чипы режутся**: `.chip__label{min-width:0}` схлопывается как flex-элемент → `min-width:max-content` на лейбле.

### B. Инициатива «ДС из коробки» (подтверждена пользователем)
Цель: любой компонент работает у потребителя без до-сборки; в чит-шите сразу видно, что он умеет. План:
1. **Аудит**: пройти `scripts/*.page.js` и inline-`<style>` страниц; выписать, у каких компонентов поведение/стили-композиции только на витрине. Уже видно: NavPanel, RiskMetric. Проверить: Popover, Tooltip, ContextMenu, Modal, TableFilter, Tabs, Splitter, SegmentControl, Toast, Snackbar, Pagination, Breadcrumbs (more-меню).
2. **Экстракция** на каждый: поведение → `scripts/ds-<comp>.js` (init по data-атрибутам/классам, без привязки к DOM витрины); стили-композиции → глобальный `styles/<comp>.css`.
3. **Чит-шит**: у каждого компонента раздел «умеет из коробки» (какой скрипт, какие data-атрибуты, какое поведение).
4. Проверка на живом экране, не на витрине.
**Не решено пользователем**: приоритет — начать с NavPanel+RiskMetric (довести до общего рантайма ДС, а не только screens-chrome), затем Popover/Tooltip/ContextMenu/Modal/TableFilter; ИЛИ сперва полный аудит-таблица. Спросить в начале.

## Правила проекта (напоминание)
Русский; ванильный HTML/CSS без React; иконки только `<i data-icon>` (имена — `specs/Icons.md`, не читать `icons-data.js`); токены/классы ДС, ничего не выдумывать; точечные правки (`str_replace_edit`/`run_script`); при новом компоненте — синхронно `index.html` + `ds-nav.js` + спека + чит-шит; `check_design_system` чист → `ready_for_verification`.
